**Date:** 2026-07-04 · **Slug:** `somatic-vaf-plausibility` · **Aspect:** `vaf-verdict`
**PRD:** `docs/planning/somatic-vaf-plausibility/prd.md` · **Dig:** `../understanding.md`
**Branch:** `feat/somatic-vaf-plausibility/aliz`

Single cohesive verify-stage slice. Strict TDD (RED → GREEN → REFACTOR) per phase.
Everything is a pure function of VCF bytes + one runner gate. No detector/model/persisted-
record change. No real nf-core/sarek run in CI — synthetic VCF fixtures only.

---

## Design decisions resolved here (the two tech-plan open questions)

**D1 — Q1: no Mutect2 VCF found.** The runner gate globs `*.vcf.gz` under the run dir and
selects the first candidate whose **path contains `mutect2`** (case-insensitive).
- **No `*.vcf.gz` at all** → the somatic gate **skips silently** (emits nothing). Rationale:
  structural QC already FAILs a missing required output; a second "missing output" signal
  here would be redundant noise. Mirrors germline's `if vcfs:` skip (`runner.py:66-68`).
- **`*.vcf.gz` exist but none under a `mutect2` path** → emit a **single UNVERIFIED**
  `QCResult` (`check="somatic_vaf_plausibility"`, `value=None`, `kind="metric"`, message
  "no Mutect2 somatic VCF found to assess VAF"). Rationale: we had outputs to assess and
  honestly could not — that is UNVERIFIED, never a silent pass.

**D2 — swapped-pair smell test: DEFERRED (documented residual risk), not implemented.**
Tumor identification relies on Mutect2's `##tumor_sample=<name>` header mapped to the
`#CHROM` column. If that header is absent/unparseable → UNVERIFIED (never guess a column;
D-edge below). A full cross-column smell test (compare tumor vs normal median VAF to catch
a *mislabeled* pair whose header is present but wrong) adds real complexity for a
low-probability failure and is **out of scope for this slice** — noted in Edge Cases and in
the CHANGELOG deferrals so it is not mistaken for covered. The header-based ID + UNVERIFIED
fallback is the honest floor this slice commits to.

**RED — write these tests first** (mirror `test_variant_metrics.py` fixture style; new

Fixture helpers to add to the test file:
```python
_TUMOR = "TUMOR"
_NORMAL = "NORMAL"

def _header(tumor=_TUMOR, normal=_NORMAL, tumor_line=True):
    lines = ["##fileformat=VCFv4.2"]
    if tumor_line:
        lines.append(f"##tumor_sample={tumor}")
    # column order: NORMAL then TUMOR, to prove we select by name not position
    lines.append(
        "#CHROM\tPOS\tID\tREF\tALT\tQUAL\tFILTER\tINFO\tFORMAT\t"
        f"{normal}\t{tumor}"
    )
    return "\n".join(lines) + "\n"

def _rec(chrom, pos, ref, alt, tumor_fmt, normal_fmt="0/0:0.0:10,0:10", fmt="GT:AF:AD:DP"):
    # tumor_fmt like "0/1:0.30:14,6:20"; column order NORMAL then TUMOR
    return f"{chrom}\t{pos}\t.\t{ref}\t{alt}\t.\tPASS\t.\t{fmt}\t{normal_fmt}\t{tumor_fmt}\n"

def _write(path, header, recs):
    path.write_text(header + "".join(recs)); return path
```

Test cases (each asserts on `somatic_metrics(path)` → a `SomaticMetrics(median_vaf, variant_count)`):
1. `test_median_vaf_from_af` — two recs, tumor AF 0.20 and 0.40 → `median_vaf == 0.30`
   (even count → mean of the two middles), `variant_count == 2`. Proves the **TUMOR**
   column (not the position-0 NORMAL) is read, since NORMAL AF is 0.0.
2. `test_vaf_ad_dp_fallback_when_no_af` — FORMAT `GT:AD:DP` (no AF); tumor `0/1:14,6:20`
   → VAF 6/20 = 0.30.
3. `test_af_preferred_over_ad_dp` — both present, AF=0.25 while AD/DP would give 0.50 →
   uses 0.25.
4. `test_multiallelic_excluded` — a rec with `A>G,T` (comma in ALT) is excluded from the
   VAF list and from `variant_count`; a sibling biallelic rec still counts.
5. `test_indel_included` — `AT>A` biallelic indel with a tumor AF contributes a VAF.
6. `test_dp_zero_guarded` — `GT:AD:DP` tumor `0/1:0,0:0` (DP=0, no AF) → contributes no
   VAF (no divide-by-zero); if it's the only rec, `median_vaf is None`.
7. `test_median_vaf_none_when_no_vaf` — a rec whose tumor has neither AF nor usable AD/DP
   → `median_vaf is None`, `variant_count` reflects considered biallelic recs.
8. `test_missing_tumor_header_yields_no_vafs` — `_header(tumor_line=False)` → tumor column
   unidentifiable → `median_vaf is None` (never read a guessed column).
9. `test_gzip_supported` — same via `gzip.open(...,'wt')`, results equal.

**GREEN — module API:**
```python
@dataclass(frozen=True)
class SomaticMetrics:
    median_vaf: float | None
    variant_count: int

def somatic_metrics(vcf_path) -> SomaticMetrics: ...
```
Internals:
- `_open_text` — reuse the gzip-transparent idiom from `concordance.py:79-84` (copy, do
  not import a private; a 5-line helper is fine and keeps the module self-contained).
- `_tumor_column_index(header_lines) -> int | None` — parse `##tumor_sample=NAME`, then
  find NAME's position in the `#CHROM` line's sample columns (index ≥ 9). None if either
  is missing.
- `_biallelic(ref, alt) -> bool` — `"," not in alt` (multiallelic excluded); indels allowed.
- `_vaf_from_sample(fmt_keys, sample_fields) -> float | None` — locate `AF` → float(first
  comma-split value); else `AD` (`alt = second value`) and `DP` → `alt/DP` when `DP>0`;
  else None. Malformed → None (never crash).
- `somatic_metrics`: stream lines; capture header lines until `#CHROM`; resolve tumor col;
  for each biallelic data rec, read the tumor sample field, compute VAF, collect; count
  considered biallelic recs. `median_vaf = statistics.median(vafs) if vafs else None`.

**Validation:** `uv run pytest tests/verification/test_somatic_plausibility.py -q`
**Commit:** `test+feat(somatic): tumor-aware VAF + variant-count metrics from the somatic VCF`

**GREEN:**
- Add to `rule_pack.py` near `RNASEQ_PLAUSIBILITY_PACK:237-250` (WARN-cap: only

```python

    {   # tumor VAF distribution: a somatic set spans low subclonal to ~0.5 clonal-het;
        # a median pinned near 1.0 (germline leakage) or ~0.5 (mis-paired normal) is
        # suspicious. Uncalibrated engineering defaults, WARN-capped.

        "metric": "median_vaf",
        "warn_below": 0.05,
        "warn_above": 0.95,

        # because target type (panel/WES/WGS) varies by orders of magnitude.
        "check": "somatic_variant_count",
        "metric": "somatic_variant_count",
```

## 4. Testing strategy

- **Unit** (`test_somatic_plausibility.py`): VAF derivation matrix (AF / AD-DP / neither /
  DP0 / multiallelic / indel / missing-header / gzip / even-count median), the three
  WARN-capped checks (pass/warn-never-fail/unverified), PON three-state matrix.
- **Integration** (`test_run_qc.py`): positive + negative assay gate; D1 skip vs UNVERIFIED.
- **Full suite** `uv run pytest -q` after Phase 4 and Phase 5 — the germline/rnaseq
  plausibility tests, structural tests, and the shipped-corpus detector guard must be
  unaffected (this slice adds verification only).

## 5. Environment & reproducibility

- stdlib only; no `uv add`. Deterministic pure functions of VCF bytes (no network, no tool
  exec, no randomness) — reproducible by construction. No raw-read egress (operates on the
  VCF on the user's compute).

## 6. Edge cases & error handling

- Missing `##tumor_sample=` or tumor name absent from `#CHROM` → tumor col None → no VAFs →
  `median_vaf` UNVERIFIED; sample label falls back to `"sample"`.
- `DP==0`, malformed `AD`/`AF`, non-numeric fields → that record contributes no VAF (never
  crash); if all records fail, `median_vaf` UNVERIFIED.
- Multiallelic (comma in ALT) excluded from both `median_vaf` and `somatic_variant_count`;
  indels included.
- Even VAF count → `statistics.median` (mean of the two middles), deterministic.
- No Mutect2 VCF but other VCFs present → single UNVERIFIED (D1). No VCF → silent skip.
- **Residual (deferred, D2):** a header-present-but-mislabeled tumor/normal pair could
  yield a plausible-looking PASS; the cross-column smell test that would catch it is out of
  scope and named in the CHANGELOG deferrals.
