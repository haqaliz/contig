# Implementation plan: concordance-autorun / verify-autorun (2026-06-27)

Source: `../prd.md` + `spec.md`. Branch `feat/concordance-autorun/aliz`. Strict TDD,
suite kept green (`uv run pytest`, 757 at branch point). bcftools is NEVER executed
in tests (injected fake / argv-only assertions). Two sequential phases plus a manual
gate.

## Conventions
- `VariantCaller = Callable[[str, str, str], str]`: `(bam, ref, out_dir) -> vcf_path`.
- The default caller runs `bcftools mpileup -f <ref> <bam>` piped to
  `bcftools call -mv -Oz -o <out_dir>/second.vcf.gz`, then returns that path.
- Honesty: a missing binary, missing BAM/ref, or empty output never yields PASS.
- Reuses `evaluate_concordance` and `kind="concordance"`; no report/dashboard change.

## Phase 1: the injectable second-caller seam

**Goal:** the caller module, testable in CI without bcftools.
**Files:** create `src/contig/verification/second_caller.py`,
`tests/verification/test_second_caller.py`.

**RED (write first):**
1. `test_bcftools_command_builds_expected_argv`, `bcftools_command(bam, ref, out)`
   returns the expected argv list (the mpileup and call stages; assert binary names,
   `-f ref`, the bam, `-mv`, `-Oz`, `-o out`). Pure, no execution.
2. `test_run_bcftools_caller_missing_binary_is_clear_error`, with PATH emptied (or
   a bogus binary name injected), `run_bcftools_caller` raises a clear, named error
   (e.g. `SecondCallerError`) mentioning bcftools, not a bare FileNotFoundError.
3. `test_run_bcftools_caller_missing_inputs_errors`, a non-existent BAM or ref
   raises a clear error before any spawn.

**GREEN:**
- `bcftools_command(bam, ref, out) -> list[str]` (or a small structure for the two
  piped stages) as a pure function.
- `run_bcftools_caller(bam, ref, out_dir) -> str`: validate bam/ref exist; build the
  command; run it (subprocess, like `runner.default_executor`); on
  FileNotFoundError for the binary, raise `SecondCallerError("bcftools not found
  ...")`; return the out path. The actual subprocess success path is exercised only
  by the manual gate, not CI.
- Define `VariantCaller` and `SecondCallerError`.

**Validation:** `uv run pytest tests/verification/test_second_caller.py -q`.
**Commit:** `feat(concordance-autorun): injectable bcftools second-caller seam`.

## Phase 2: the verify --concordance-auto CLI path

**Goal:** the user-facing command, with the caller injected so CI never runs bcftools.
**Prerequisites:** Phase 1.
**Files:** modify `src/contig/cli.py`; add tests to `tests/test_cli.py`.

**RED:**
1. `test_verify_concordance_auto_emits_checks`, a germline run (primary VCF on
   disk) + `--concordance-auto --bam <bam> --ref <ref>`, with the module caller
   monkeypatched to a fake that writes a recorded second VCF, prints the concordance
   checks; a concordant fake -> PASS-ish, a divergent fake -> WARN.
2. `test_verify_concordance_auto_at_most_warn_exit`, a divergent fake caller does
   not make verify exit non-zero.
3. `test_verify_concordance_auto_json_includes_results`, `--json` includes the
   concordance results.
4. `test_verify_concordance_auto_missing_bam_or_ref_skips`, a non-existent `--bam`
   or `--ref` prints a clear note and emits no concordance, exit unaffected.
5. `test_verify_concordance_auto_caller_failure_skips`, a fake caller that raises
   `SecondCallerError` -> clear note, no PASS, no crash.
6. `test_verify_concordance_auto_non_germline_skips`, an rnaseq run -> skip note.

**GREEN:**
1. Add `concordance_auto: bool = typer.Option(False, "--concordance-auto", ...)`,
   `bam: str = typer.Option(None, "--bam", ...)`, `ref: str = typer.Option(None,
   "--ref", ...)` to `verify`.
2. Add `_evaluate_run_concordance_auto(record, runs_dir, run_id, bam, ref, caller=
   None)`: gate to `variant_calling` (skip note otherwise); resolve the primary VCF
   (reuse the existing manifest-glob resolution from `_evaluate_run_concordance`);
   validate bam/ref exist (clear note + `[]` if not); resolve `actual = caller or
   run_bcftools_caller` (module-level name so tests can monkeypatch
   `contig.cli.run_bcftools_caller`); call it inside try/except `SecondCallerError`
   (clear note + `[]` on failure); call `evaluate_concordance(primary, second,
   "variant_calling")` and return it.
3. Wire into `verify`: when `concordance_auto`, compute via the auto helper (mutually
   exclusive with `--concordance-vcf`; if both given, prefer explicit vcf or error
   clearly). Surface like the existing concordance path (text + JSON), never
   changing the exit code.

**Validation:** `uv run pytest tests/test_cli.py -q` then full `uv run pytest -q`.
**Commit:** `feat(concordance-autorun): contig verify --concordance-auto`.

## Phase M (manual gate, before the PR is final, NOT CI)

Run the real bcftools once on a real germline BAM + reference (the local toolchain
has Docker/Nextflow; bcftools via a container or a local install). Confirm
`contig verify <run> --concordance-auto --bam <bam> --ref <ref>` produces a
concordance result. If the default command needs adjusting (ploidy, regions), fix
`bcftools_command` and note it. Record the outcome in the PR description.

## Testing strategy
- Unit: argv builder, missing-binary/missing-input errors (no execution).
- Integration: the CLI auto path with an injected fake caller (concordant, divergent,
  failure, missing inputs, non-germline).
- Determinism: fixtures are tiny inline VCFs; the fake caller writes a recorded VCF;
  bcftools never runs in CI.
- Regression: full `uv run pytest` green; no report/dashboard change.

## Edge cases
- Missing binary -> SecondCallerError -> clear skip note, no PASS.
- Missing/unreadable BAM or ref -> clear note before spawn.
- Caller returns an empty/garbage VCF -> evaluate_concordance yields unverified
  (no shared sites), never PASS.
- `--concordance-auto` and `--concordance-vcf` both given -> clear error or a
  defined precedence (decide in Phase 2; document).

## Agent execution notes
