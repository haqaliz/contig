## 0. Design decisions locked (resolving PRD open questions)

| # | Decision | Rationale |
|---|---|---|
| Fix target | **plain uncompressed `.fa`** | universal downstream acceptance; mirrors rnaseq's GUNZIP; one step. |
| FailureClass | **new `reference_not_bgzf`** | file isn't missing ‚Äî wrong compression; clean corpus/routing. |
| Patch | `kind="reference"`, `operation={"recompress_reference": True}`, `risk="needs_confirmation"` | mirrors the sibling `build_index` reference patch; gated path; tests pass `auto_approve=True`. (Tradeoff noted: `safe` would raise unattended-completion further, but `needs_confirmation` matches the sibling and lets a human veto a reference rewrite. Keep `needs_confirmation`.) |
| **Decompression mechanism (R5)** | **stdlib `gzip` streaming, NOT the `IndexBuilder` seam** | the seam exists to keep *external binaries* out of CI; stdlib gzip is pure-Python, inherently streaming (`shutil.copyfileobj` with a bounded buffer), always available, and runs for real in CI on a tiny fixture. Deviates from understanding.md's "via IndexBuilder" on purpose ‚Äî simpler + more testable. `index_builder` is **not** threaded into the recompress helper. |
| Fasta source | **`params["fasta"]`** (not diagnosis evidence) | sarek faidx's exactly that path; matches `_build_star_index`'s `params.get("fasta")`. |
| Detector token (R2 over-match) | anchor on faidx-specific **`cannot index files compressed with gzip`** | tabix/bcftools also say "please use bgzip" for *VCFs* ‚Äî a different fix. `fai_build3` / "could not build fai index" as secondary evidence. |
| Outcome labels (R4) | success `recompressed_reference_and_retried`; give-ups `reference_recompress_unresolvable` (no/again fasta, or not-plain-gzip) and `reference_recompress_failed` (decompress error) | outcomes are free-form `RepairStep.outcome` strings (no Literal); precise names aid corpus/telemetry. Honest FAIL on every give-up, never a false pass. |

### Phase 3 ‚Äî `_recompress_reference` helper + magic-byte guard (the correctness core)
- **Goal:** a pure-ish helper that decides + performs the recompress into scratch, honestly.
- **Files:** `src/contig/self_heal.py` (new `_recompress_reference` + a small pure
  `_gzip_kind(path) -> "plain_gzip"|"bgzf"|"not_gzip"` classifier); `tests/test_self_heal.py`.
- **RED (unit):**
  - `_gzip_kind` on a plain-gzip fixture ‚Üí `"plain_gzip"`; on a BGZF fixture (bytes
    `1f 8b 08 04` + `BC` extra subfield) ‚Üí `"bgzf"`; on an uncompressed / short / non-gzip
    file ‚Üí `"not_gzip"`. **(R2 ‚Äî the top correctness test: a valid BGZF ref must be
    recognized and left untouched.)**
  - `_recompress_reference` with `params={"fasta": <plain-gzip fixture>}` ‚Üí returns
    `recompressed_reference_and_retried`, writes `<run_dir>/healed_reference/<stem>.fa`
    (uncompressed, byte-equal to the original), sets `params["fasta"]` to that scratch path,
    adds original+scratch to `built_paths`, `continue_=True`.
  - give-ups (all `continue_=False`, honest detail, params["fasta"] unchanged or absent):
    no `fasta` key ‚Üí `reference_recompress_unresolvable`; BGZF fixture ‚Üí `reference_recompress_unresolvable` ("already BGZF/not plain-gzip"); already in `built_paths` ‚Üí `reference_recompress_unresolvable` ("already recompressed; failure persists").
- **GREEN:** implement, modeled on `_build_star_index` (self_heal.py:566-673) but:
  - resolve `fasta = params.get("fasta")`; missing ‚Üí give up.
  - `built_paths` one-per-run guard (add both original + scratch).
  - `_gzip_kind(fasta)`: only `"plain_gzip"` proceeds; else give up honestly.
  - scratch `Path(run_dir)/"healed_reference"`; wipe+mkdir (mirror STAR L630-631);
    target `scratch/<basename minus a trailing .gz>` (`ref.fa.gz`‚Üí`ref.fa`).
  - **stream-decompress:** `with gzip.open(fasta,"rb") as src, open(target,"wb") as dst: shutil.copyfileobj(src, dst, length=1<<20)`; wrap in try/except ‚Üí `reference_recompress_failed` on `OSError`/`gzip.BadGzipFile` (honest FAIL, never a false pass); on failure do NOT redirect.
  - on success set `params["fasta"] = str(target)`, `RepairStep`-detail names original+scratch, return `recompressed_reference_and_retried, detail, True`.
  - signature: `(target, params, *, run_dir, built_paths)` ‚Äî no `index_builder` (see ¬ß0).
- **Validation:** `uv run pytest tests/test_self_heal.py -k recompress`.
- **Commit:** `feat(heal): _recompress_reference ‚Äî stdlib-gzip decompress + BGZF-safe guard [C2]`

## 4. Testing strategy

- **Unit:** `_gzip_kind` (plain/bgzf/not-gzip ‚Äî R2), detector classification, repair proposal.
- **Integration:** the full self-heal loop with an injected executor + a **real** tiny
  plain-gzip fixture (stdlib gzip runs for real; STAR/samtools never do). BGzf fixture built
  with the real `bgzip` available in this env for the fixture, OR hand-crafted bytes
  (`1f 8b 08 04 ‚Ä¶` + `BC` subfield) so the test needs no external tool.
- **Guards:** `eval-detector` (24/24), `eval-guard` (refreeze), `heal-guard` (unchanged unless P7).
- **Commands:** `uv run pytest`; `uv run contig eval-detector`; `uv run contig eval-guard`.

## 5. Environment & reproducibility

- No env vars, no new deps. Determinism: stdlib gzip is deterministic; scratch path never
  persisted; launch.json keeps the original fasta (Phase 5 proves it). No raw-read egress.

## 6. Edge cases & error handling

- Valid BGZF reference misfiled as the failure ‚Üí left untouched, honest give-up (R2, AC3).
- Truncated/corrupt gzip ‚Üí `reference_recompress_failed`, honest FAIL (never a false pass).
- No `params["fasta"]` (e.g. `--genome KEY` path) ‚Üí give up; iGenomes never hits this.
- Orphaned user-built `.fai`/`.dict` beside the `.gz` (R6) ‚Üí harmless; sarek rebuilds from the
  redirected fasta. Note in the changelog, no code needed.
- Second identical failure after a recompress ‚Üí one-per-run guard gives up (AC5).
