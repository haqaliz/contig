## Phase 2 — Single-cell MatrixMarket loader + `scrnaseq` entry point

**Goal:** a new stdlib module that loads a `.mtx` triplet into a per-gene pseudobulk dict,
sniff-routes the second matrix, and evaluates concordance gated to `scrnaseq` — reusing
Phase 1's seam.
**Prerequisites:** Phase 1.
**Files:** create `src/contig/verification/sc_count_concordance.py`; create
`tests/verification/test_sc_count_concordance.py`.

**Module surface:**
```python
class ScMatrixError(Exception): ...   # unparseable / ambiguous / missing sibling

_SC_CONCORDANCE_ASSAYS = {"scrnaseq"}
_SC_MATRIX_GLOB = "*matrix.mtx*"      # exported for the CLI resolver

def load_mtx_pseudobulk(mtx_path) -> dict[str, float]: ...
def load_sc_matrix(path) -> dict[str, float]: ...       # sniff: .mtx* -> mtx, else parse_count_matrix
def evaluate_sc_count_concordance(primary_mtx, second, assay) -> list[QCResult]: ...
```

**`load_mtx_pseudobulk(mtx_path)`:**
1. Resolve siblings in the same dir: `features.tsv`|`features.tsv.gz` and
   `barcodes.tsv`|`barcodes.tsv.gz` (prefer non-gz if both, else gz). Missing either →
   `ScMatrixError("features.tsv/barcodes.tsv not found beside <mtx>")`.
2. `features = [line.split("\t")[0] for line in _open_text(features_path) if line.strip()]`
   — **column 1 = gene id** (10x id,name,type); a single-column file yields the sole token.
   `n_barcodes = <count non-blank lines of barcodes>`.
3. Read the `.mtx` via `_open_text`: skip the `%%MatrixMarket` banner and any `%`/blank
   line; the **first non-comment line** is the dimension line `nrows ncols nnz`.
4. **Orientation** by matching dims to `len(features)` / `n_barcodes`:
   - `nrows==len(features) and ncols==n_barcodes` → genes are **rows**; gene index = `r`.
   - `nrows==n_barcodes and ncols==len(features)` → genes are **cols**; gene index = `c`.
   - both dims equal (genes==cells) **and** ambiguous, or neither matches →
     `ScMatrixError("cannot determine gene axis: mtx dims (r,c) vs features N, barcodes M")`.
5. Accumulate: for each remaining `r c val` triplet (1-based, tolerant of int/float val),
   `totals[features[gene_index-1]] += float(val)`. Ignore blank/short lines. A gene axis
   index out of range → `ScMatrixError`. Zero usable genes → `ScMatrixError`.
6. Return `{gene_id: pseudobulk_total}` (repeated gene ids in features.tsv accumulate — same
   posture as `parse_count_matrix`).

**`load_sc_matrix(path)`:** if `Path(path).name` ends with `.mtx` or `.mtx.gz` →
`load_mtx_pseudobulk(path)`; else `parse_count_matrix(path)` (dense pseudobulk TSV, may raise
via open()).

**`evaluate_sc_count_concordance(primary_mtx, second, assay)`:**
- `assay not in _SC_CONCORDANCE_ASSAYS` → `[]`.
- Try `a = load_mtx_pseudobulk(primary_mtx)`; `b = load_sc_matrix(second)`. On `ScMatrixError`
  or `OSError`/`ValueError` from either → return **one** explicit UNVERIFIED via `_concordance`
  (import from `count_concordance`): `check="sc_count_concordance"`, `status="unverified"`,
  message names the file and the reason (located-but-unparseable, never a false pass), `value=None`.
- Else `return results_from_counts(a, b, Path(primary_mtx).name, Path(second).name)`.

**TDD steps (write RED first, one test per branch — critique gap 2):**
1. `load_mtx_pseudobulk` — genes-as-rows fixture (3 genes × 2 cells) sums per gene correctly.
2. genes-as-cols (transposed) fixture → same totals (orientation handled).
3. gzipped `matrix.mtx.gz` + `features.tsv.gz` + `barcodes.tsv.gz` → parsed.
4. `%%MatrixMarket` banner + `%` comment lines skipped; dimension line respected.
5. multi-col `features.tsv` (id,name,type) → keyed by **column 1 (id)**; single-col → sole token.
6. missing `barcodes.tsv` sibling → `ScMatrixError`.
7. ambiguous orientation (dims match neither / square-ambiguous) → `ScMatrixError`.
8. `load_sc_matrix` sniff: `.mtx` path → loader; `.tsv` path → `parse_count_matrix` (assert a
   dense pseudobulk TSV yields the expected dict).
9. `evaluate_sc_count_concordance`: concordant pair (≥10 genes) → `spearman_concordance` PASS;
   divergent → WARN with the value; `<10` shared → UNVERIFIED (`value=None`).
10. non-`scrnaseq` assay → `[]`.
11. primary `.mtx` present but sibling missing → **one** `sc_count_concordance` UNVERIFIED
    (not `[]`, not a crash).

**Fixture idiom:** mirror `tests/verification/test_count_concordance.py` — write real files
under `tmp_path`. Helper `_write_triplet(dir, genes, cells, counts)` renders `matrix.mtx`
(`%%MatrixMarket matrix coordinate integer general`, dim line, triplets), `features.tsv`,
`barcodes.tsv`. Build ≥12 genes for the concordant/divergent cases (> the 10-gene floor).

**Validation:** `uv run pytest tests/verification/test_sc_count_concordance.py` green; full
suite green; `grep -rn "scipy\|numpy\|anndata\|h5py" src/contig/verification/sc_count_concordance.py`
returns nothing.

## Edge cases & error handling

- Primary `.mtx` found, sibling missing/malformed, or ambiguous orientation → one explicit
  `sc_count_concordance` UNVERIFIED (never a silent `[]`, never a crash).
- No `matrix.mtx` at all (e.g. `.h5ad`-only simpleaf run) → CLI skip note + `[]`.
- `< 10` shared genes → core UNVERIFIED (`value=None`).
- id-vs-symbol keyed matrices → low `gene_overlap` (informational, always PASS); a genuinely
  disjoint pair → UNVERIFIED, never a false WARN/PASS.
- Second matrix `.mtx` with missing siblings → UNVERIFIED (same path as primary).
