## Phase 2 — Reducer ignores informational results

- **Goal:** R3/A2–A6. The invariant.
- **Prerequisites:** P1.
- **Files:** `src/contig/models.py`; `tests/test_models.py`.
- **Steps:**
  1. **RED:** A2 `overall_verdict([informational pass])` → `"unverified"`.
  2. **RED:** A3 `[informational pass, asserting pass]` → `"pass"`.
  3. **RED:** A4 `[informational pass, unverified]` → `"unverified"`.
  4. **RED:** A6 `[informational pass, asserting fail]` → `"fail"`; same for `warn`.
  5. **RED:** A5 `overall_verdict([])` still raises `ValueError` — **must not regress**.
  6. **GREEN:** in `overall_verdict` (`models.py:78-96`), keep the empty-list guard on the
     **full** list, then reduce over `severe = [r for r in results if not r.informational]`:
     `fail` → `warn` → `pass` → else `"unverified"`. An all-informational list is *not*
     empty; it must reduce to `"unverified"`, never raise.
  7. Update the docstring: informational results carry no severity and cannot produce a
     `pass`. State the invariant explicitly.
- **Decision (spec R3 sub-question):** informational results contribute **no severity at
  all**. None of the four can currently emit fail/warn, so this is the simplest honest rule.
  Do not add a "may an informational check warn" pathway.
- **Validation:** `uv run pytest tests/test_models.py -q`.
- **Commit:** `feat(verify): informational results carry no severity in overall_verdict`

## Phase 4 — Mark band-less rule-pack rules + the guard

- **Goal:** R2/A7/A8 (config mechanism) + R7.
- **Prerequisites:** P1.
- **Files:** `src/contig/verification/rule_pack.py` (`evaluate` ~:569-588, and a new
  band-less predicate); `tests/verification/test_rule_pack.py`.
- **Steps:**
  1. **RED — A8, the trap.** A rule with **only `fail_below`** and no `warn_*` must be
     **not** informational and must still FAIL below its floor. Write this test FIRST; it is
     the guardrail against the `expected_range` mistake.
  2. **RED:** a rule with none of the four bound keys yields `informational=True`.
  3. **RED:** a rule with any bound yields `informational=False`.
  4. **GREEN:** add a predicate, e.g.
     `_is_band_less(check) -> bool: return not any(k in check for k in _BOUND_KEYS)`
     with `_BOUND_KEYS = ("fail_below", "fail_above", "warn_below", "warn_above")`.
     Reuse the same tuple in `_status_for`/`_expected_range` if it tidies without
     behaviour change; otherwise leave them alone.
  5. **GREEN:** in `evaluate` (`rule_pack.py:579-587`), pass
     `informational=_is_band_less(check)` into the `QCResult`.
  6. **MUST NOT** key off `_expected_range(check) is None` — it inspects only the two
     `warn_*` keys, so a `fail_below`-only rule also returns `None` yet can FAIL. Add a
     comment at the predicate saying exactly this, so the next reader doesn't "simplify" it.
  7. **R7/A7 — enumeration guard.** One test asserting the informational set is exactly
     `{duplication_rate, gene_symbol_concordance, x_het_ratio, gene_overlap}`, sourced by
     scanning the rule packs for band-less rules plus a literal list of the three
     code-marked ones. A fifth landing without updating this test must fail.
- **Validation:** `uv run pytest tests/verification/test_rule_pack.py -q`
- **Commit:** `feat(verify): band-less rules are informational; guard the informational set`

## Environment & reproducibility

No env vars, no external tools, no new deps. Determinism unaffected: no manifest/pin change,
no re-run behaviour change. The reproduce bundle gains one additive per-result boolean;
`RunRecord.verdict` remains a `@computed_field` (`models.py:357-369`) re-derived from
**stored** results — so old bundles keep their verdicts (R6) and history is stable.

## Edge cases & error handling

| Case | Expected |
|---|---|
| All results informational | `"unverified"` — never `"pass"`, never raise |
| Empty result list | `ValueError` — unchanged (A5) |
| Informational + one asserting pass | `"pass"` (A3) |
| Informational + asserting fail | `"fail"` (A6) |
| Rule with only `fail_below` | NOT informational; still FAILs (A8) |
| Old bundle, no `informational` key | defaults `False`; verdict unchanged (A9) |
| `x_het_ratio` with no callable X sites | `"unverified"`, not a pass (A11) |
| Dashboard reads a record with `informational` | ignored by un-updated code; additive/safe (`runs.ts:58` casts without validation) |
