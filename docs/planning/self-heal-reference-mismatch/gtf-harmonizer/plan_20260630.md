## Phase 1 — Pure harmonization decision (`plan_harmonization`)

- **Goal:** decide whether a *safe* uniform `chr` transform on the GTF resolves a
  detected mismatch, and in which direction. Pure, no I/O beyond reading contig sets.
- **Files:** create `src/contig/reference_harmonize.py`; create
  `tests/test_reference_harmonize.py`.
- **TDD steps:**
  1. **RED** — write tests first (mirror `tests/test_reference_check.py` style: real
     `tmp_path` FASTA/GTF via `_write`/`_write_gz` helpers, no mocks):
     - chr-prefixed FASTA + bare GTF, disjoint → plan with `direction="add_chr"`.
     - bare FASTA + chr-prefixed GTF, disjoint → plan with `direction="strip_chr"`.
     - any shared contig (overlap/subset) → `None` (detector already passes; nothing to do).
     - both bare but genuinely different assemblies (disjoint, neither chr-prefixed) →
       `None` (not an asymmetry).
     - chr-asymmetric BUT post-transform still disjoint (true wrong-assembly dressed as
       a prefix diff, e.g. FASTA `{chrA,chrB}` vs GTF `{1,2}`) → `None` (refuse).
     - empty/unparseable either side → `None`.
     - determinism: same inputs → identical plan.
  2. **GREEN** — implement:
     ```python
     HarmonizationDirection = Literal["add_chr", "strip_chr"]

     @dataclass(frozen=True)
     class HarmonizationPlan:
         direction: HarmonizationDirection
         fasta_sample: str   # sorted first-3, for the user note
         gtf_sample: str

     def plan_harmonization(fasta_path, gtf_path) -> HarmonizationPlan | None:
         if not check_reference_consistency(fasta_path, gtf_path):
             return None                      # no mismatch → nothing to do
         fa = fasta_contigs(fasta_path); gt = gtf_contigs(gtf_path)
         if _all_chr_prefixed(fa) and not _all_chr_prefixed(gt):
             direction = "add_chr"; transformed = {f"chr{n}" for n in gt}
         elif _all_chr_prefixed(gt) and not _all_chr_prefixed(fa):
             direction = "strip_chr"
             transformed = {n[3:] if n.startswith("chr") else n for n in gt}
         else:
             return None                      # not an unambiguous asymmetry
         if not (transformed & fa):
             return None                      # transform does NOT resolve it → refuse
         return HarmonizationPlan(direction, _sample(fa), _sample(gt))
     ```
     (Import `_sample`, `_all_chr_prefixed`, `fasta_contigs`, `gtf_contigs`,
     `check_reference_consistency` from `reference_check`.)
- **Validation:** `uv run pytest tests/test_reference_harmonize.py -q` green; full suite green.
- **Commit:** `feat(self-heal): decide safe chr-prefix GTF harmonization (pure, refuses wrong-assembly)`

## Phase 2 — GTF stream rewrite (`harmonize_gtf`)

- **Goal:** rewrite the GTF's column-1 seqnames applying the plan's direction, to a
  scratch path, byte-faithful elsewhere, gzip-transparent. Closed-loop: the result
  re-passes `check_reference_consistency`.
- **Files:** add to `src/contig/reference_harmonize.py`; extend
  `tests/test_reference_harmonize.py`.
- **TDD steps:**
  1. **RED** — tests:
     - **Closed loop (M7):** harmonize a chr-asymmetric GTF, then assert
       `check_reference_consistency(fasta, out) == []`.
     - **Column fidelity (M8):** a GTF row with 9 tab-separated columns + attributes →
       only column 1 changes; columns 2-9 byte-identical (compare the post-tab remainder).
     - **Pass-through:** `#`-comment, `track`, and `browser` lines unchanged; blank lines
       unchanged; trailing whitespace and `\r\n` vs `\n` preserved.
     - **gzip:** a `.gz` input yields a `.gz` output that decompresses to the harmonized text.
     - **add_chr / strip_chr** both directions; a seqname already in target form is left
       as-is (idempotent per-line under the direction).
  2. **GREEN** — implement:
     ```python
     def harmonize_gtf(gtf_path, direction, out_path) -> Path:
         opener = gzip.open if str(gtf_path).endswith(".gz") else open
         writer = gzip.open if str(out_path).endswith(".gz") else open
         with opener(gtf_path, "rt") as fin, writer(out_path, "wt") as fout:
             for line in fin:                      # keepends → preserves \n / \r\n / EOF
                 if line.startswith("#") or not line.strip():
                     fout.write(line); continue
                 seqname, _, rest = line.partition("\t")   # rest includes the tab? no
                 # use split("\t", 1) to MATCH gtf_contigs tokenization exactly:
                 head, sep, tail = line.partition("\t")
                 new = _apply(head, direction)
                 fout.write(new + sep + tail if sep else _apply(line.rstrip("\n"), direction) + "\n")
         return Path(out_path)
     ```
     - `_apply(name, direction)`: `f"chr{name}"` for `add_chr` if not already
       chr-prefixed; `name[3:]` for `strip_chr` if chr-prefixed; else unchanged.
     - **Pin the tokenization to `split("\t", 1)`** so column-1 boundary matches
       `gtf_contigs` (`reference_check.py:32`). Preserve the exact original line ending —
       iterate with `for line in fin` (keeps `\n`); detect/keep `\r\n`. Do NOT
       `.strip()` the whole line. (Agents: write the fidelity test first and let it drive
       the exact byte handling — the snippet above is indicative, not final.)
  3. **REFACTOR** — extract `_apply`; ensure no double-`chr` and no over-strip.
- **Validation:** `uv run pytest tests/test_reference_harmonize.py -q` green; full suite green.
- **Commit:** `feat(self-heal): stream-rewrite GTF seqnames to harmonize chr-prefix (gzip-transparent, col-1 only)`

## Edge cases & error handling

- Post-transform still disjoint → `None` → refuse (no false harmonization). [P1]
- Non-asymmetric disjoint (both bare / both chr) → `None` → existing refusal. [P1]
- Overlap/subset references → `None` (detector already passes). [P1]
- chrM/MT, scaffolds (`GL…`) → uniform transform only; unmatched residue tolerated as
  long as sets intersect post-transform. No per-contig mapping. [P1]
- `.gz` fidelity, CRLF, comments, 9-column rows → byte-faithful. [P2]
- Legacy manifests / run records without the new fields → default cleanly. [P3]
- iGenomes runs (`--genome`, no fasta/gtf) → harmonizer never reached (gate guarded by
  `"fasta" in params and "gtf" in params`). [P4]
