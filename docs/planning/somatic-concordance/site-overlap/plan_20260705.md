## Phase 1 — The pure concordance module (`somatic_concordance.py`)

**Goal:** a deterministic, FILTER-aware PASS-site-overlap evaluator over two call sets,
emitting one `kind="concordance"` QCResult, WARN-capped, UNVERIFIED below a floor.

**Prerequisites:** none.

**Files:** create `src/contig/verification/somatic_concordance.py` and
`tests/verification/test_somatic_concordance.py`.

**Design (mirror `concordance.py` shape; do NOT reuse its GT parser):**

- Constants (documented engineering defaults, not clinical claims):
  - `_OVERLAP_WARN_BELOW = 0.90` — WARN below this Jaccard (mirrors `concordance.py:30-31`).
  - `_MIN_SHARED_SITES = 10` — below this many **union** PASS sites, UNVERIFIED (mirrors
    `count_concordance.py:_MIN_SHARED_GENES=10`; a Jaccard over a handful of sites is
    meaningless). Rationale one-liner in a comment.
- `SiteKey = tuple[str, str, str, str]` (CHROM, POS, REF, ALT), same as `concordance.py:40`.
- `_open_text(path)` — gzip-transparent open (copy idiom from `concordance.py:79-84`).
- `_PASS_FILTERS = {"PASS", "."}` and `parse_pass_sites(path) -> set[SiteKey]`:
  stream lines, skip `#` headers and blanks, `cols = line.split("\t")`, require
  `len(cols) >= 7`, keep the record **only when `cols[6].strip() in _PASS_FILTERS`**
  (FILTER is column 6 — net-new vs germline which ignores it), add `(cols[0], cols[1],
  cols[3], cols[4])`. Reads no sample columns (sample-agnostic).
- `read_caller_sites(paths: Iterable[Path]) -> set[SiteKey]`: union of `parse_pass_sites`
  over one caller's file(s) (Strelka's split SNV+indel files → one set).
- `_overlap(a: set, b: set) -> tuple[int, int, float]`: returns
  `(shared=|a∩b|, union=|a∪b|, jaccard = shared/union if union else 0.0)`.
- `evaluate_somatic_concordance(mutect2_paths, strelka_paths, *, label_a="mutect2",
  label_b="strelka") -> list[QCResult]`:
  1. `a = read_caller_sites(mutect2_paths)`, `b = read_caller_sites(strelka_paths)`.
  2. `shared, union, jac = _overlap(a, b)`.
  3. If `union < _MIN_SHARED_SITES`: one `somatic_site_overlap` **unverified** result
     (value `None`, message: too few PASS sites to corroborate), `kind="concordance"`.
  4. Else: `status = "warn" if jac < _OVERLAP_WARN_BELOW else "pass"`; one
     `somatic_site_overlap` result, `value=round(jac, 4)`,
     `expected_range=">= {_OVERLAP_WARN_BELOW}"`, message names both callers and
     `shared`/`union`, states FILTER policy (PASS-only), `kind="concordance"`.
  - Never FAIL. A QCResult builder `_concordance(...)` local to the module (copy
    `concordance.py:43-58`) keeps `kind="concordance"`.

**Implementation steps (TDD):**
1. RED — write `tests/verification/test_somatic_concordance.py` with helpers `_header()`
   (a minimal `##fileformat` + `#CHROM…` line) and `_rec(chrom,pos,ref,alt,filt="PASS")`,
   plus `_write_vcf(path, records)` and a gzip variant (mirror `test_concordance.py:16-38`).
   Cases → assert on `evaluate_somatic_concordance`:
   - identical PASS sites (≥10) → one result, `status="pass"`, `value` ≈ 1.0.
   - half-disjoint (≥10 union) → `status="warn"`, `value < 0.90`.
   - union < 10 → `status="unverified"`, `value is None`.
   - a `filt="weak_evidence"` (Strelka) / `filt="clustered_events"` (Mutect2) record is
     excluded from its set (prove FILTER filtering changes the overlap).
   - `"."` FILTER is treated as PASS.
   - Strelka split: two files (snvs + indels) unioned via `read_caller_sites`.
   - gzipped inputs give the same result as plaintext.
   - every returned result has `kind == "concordance"` and `status != "fail"`.
2. GREEN — implement the module until the file's tests pass.
3. REFACTOR — dedupe, docstring the module (posture: corroboration, WARN-cap, PASS-only,
   sample-agnostic, the literal-site-key limitation à la `concordance.py:14-17`).

**Validation:** `uv run pytest tests/verification/test_somatic_concordance.py -q` green.

**Commit:** `test+feat(somatic): PASS-site-overlap concordance module (Strelka2 vs Mutect2)`

## Phase 2 — Auto-wire into the somatic verdict (`_discover_qc`)

**Goal:** a somatic run with both callers' VCFs automatically gets the concordance check;
deterministic selection; UNVERIFIED on an ambiguous multi-pair layout; exit-code-neutral.

**Prerequisites:** Phase 1 module.

**Files:** modify `src/contig/runner.py` (somatic branch inside
`if assay == "somatic_variant_calling":`, after the VAF-plausibility block at
`runner.py:78-107`); add tests to `tests/verification/test_run_qc.py`.

**Design — a selection helper in the module (unit-testable without the runner):**
Add to `somatic_concordance.py`:
- `select_caller_vcfs(run_dir: Path, vcfs: list[Path]) -> tuple[list[Path], list[Path], str | None]`
  returning `(mutect2_paths, strelka_paths, ambiguity_reason)`:
  - `parts = {part.lower() for part in p.relative_to(run_dir).parts}` per VCF (mirror the
    existing Mutect2 path-component match at `runner.py:86-93`).
  - `mutect2 = [p for p in vcfs if "mutect2" in parts(p)]`, `strelka = [… "strelka" …]`.
  - Distinct pair dirs per caller = `{p.parent.name for p in caller_list}`. If either
    caller has **> 1** distinct pair-dir name → return `([], [], reason)` where reason
    describes the ambiguous multi-pair layout.
  - Else return `(mutect2, strelka, None)`.
- `evaluate_somatic_concordance_from_run(run_dir, vcfs) -> list[QCResult]`:
  - `m, s, reason = select_caller_vcfs(...)`.
  - If `reason` → one `somatic_site_overlap` **unverified** result (multiple tumor–normal
    pairs; not computed for an ambiguous pair).
  - If not `m` or not `s` → return `[]` (clean skip: one caller absent; VAF plausibility
    already handles a missing Mutect2 separately).
  - Else → `evaluate_somatic_concordance(m, s)`.

**Wiring step (runner.py):** inside the existing `if vcfs:` of the somatic branch, AFTER
the plausibility mutect2/else block, append:
```python
results.extend(evaluate_somatic_concordance_from_run(run_dir, vcfs))
```
Import `evaluate_somatic_concordance_from_run` alongside the other verification imports
(`runner.py:31-36`). Do **not** touch the plausibility block — the two are independent and
each self-contained.

**Implementation steps (TDD):**
1. RED — in `tests/verification/test_somatic_concordance.py` add `select_caller_vcfs` /
   `evaluate_somatic_concordance_from_run` unit tests using a `tmp_path` sarek-shaped tree:
   `variant_calling/mutect2/T_vs_N/…vcf.gz` and `variant_calling/strelka/T_vs_N/
   T_vs_N.strelka.somatic_{snvs,indels}.vcf.gz` (paths from `test_somatic_end_to_end.py:36-40`).
   Cases: both present → a concordance result; strelka-only / mutect2-only → `[]`; two pair
   dirs (`T1_vs_N`, `T2_vs_N`) → one UNVERIFIED.
2. RED — in `tests/verification/test_run_qc.py` (mirror `test_run_qc.py:119-139,172-225`):
   `_discover_qc(run_dir, assay="somatic_variant_calling")` on a both-callers tree includes a
   `kind="concordance"` result; a divergent pair yields WARN but the overall run's structural
   pass/fail (and any exit path) is unchanged; a germline/rnaseq run yields no somatic
   concordance result.
3. GREEN — add the helpers + the one-line `results.extend(...)` wiring.
4. REFACTOR — keep the somatic branch readable; comment why concordance is appended after
   (and independent of) plausibility.

**Validation:** `uv run pytest tests/verification/test_run_qc.py tests/verification/test_somatic_concordance.py -q`,

## 4. Testing Strategy

- **Unit** (Phase 1): overlap math, FILTER filtering, floor→unverified, split-file union,
  gzip parity, kind/never-fail.
- **Unit** (Phase 2): `select_caller_vcfs` determinism + multi-pair UNVERIFIED; skip on a
  single caller.
- **Integration**: `_discover_qc` includes the result for a both-callers somatic tree,
  excludes it for other assays, and never changes exit/verdict severity beyond WARN.
- No integration test runs nf-core/sarek; all fixtures are synthetic VCF strings via
  `tmp_path`. Commands: `uv run pytest` (targeted then full).

## 5. Environment & Reproducibility

- No env vars, no external tools, no network. Deterministic set operations over local VCF
  bytes; gzip-transparent. `uv sync` already done in the worktree.

## 6. Edge Cases & Error Handling

- Missing/unreadable a caller's VCF → clean skip (`[]`), no crash, no false pass.
- All records filtered out (empty PASS sets) → `union == 0 < floor` → UNVERIFIED.
- `"."` FILTER counted as PASS; multiallelic ALT with a comma keys as its literal string
  (site-key limitation, disclosed — not split in slice 1).
- Multi-pair / tumor-only sarek layout → UNVERIFIED, never an arbitrary pair compare.
- Concordance is WARN-capped and `kind="concordance"`, so it can never drive the exit code.
