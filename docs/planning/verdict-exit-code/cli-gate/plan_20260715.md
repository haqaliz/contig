**Source:** `docs/planning/verdict-exit-code/prd.md` + `_card/understanding.md`

Add an opt-in `--fail-on-verdict` boolean flag (default `False`) to `contig run` and
`contig verify` so a **FAIL** reduced verdict exits `1`. WARN/UNVERIFIED/PASS stay `0`.
Default (no flag) behavior is byte-identical to today. On the verdict-driven exit, print
a one-line reason to stderr. Existing codebase â€” integration only, no scaffolding.

## Assumptions & locked decisions

- Flag name: `--fail-on-verdict` (same on both commands). Exit code: `1`. Severity:
  FAIL-only. Slice covers both `run` and `verify`.
- Reason line **is in scope** (user call at the review gate): a single
  `typer.echo(..., err=True)` before the verdict-driven `raise typer.Exit(code=1)`.
- `--json` payloads are **unchanged** (N1 deferred). The exit code still applies under
  `--json`.
- `rerun`/`resume` do **not** get the flag in slice 1 â€” `_dispatch_run` gains a
  keyword-only `fail_on_verdict: bool = False`, so those two callers inherit the default
  and are unaffected.
- Single source of truth: `RunRecord.verdict` (`models.py:357-369`); no new model field,
  no recomputation.

## Edge Cases & Error Handling

- **verify no-checksums `return 0` path (R1):** must convert to a verdict-aware exit â€”
  covered by Phase 2 test (1). This is the single most likely miss; the consolidated
  decision + explicit test guard it.
- **Incomplete run + flag:** already exits via `.succeeded`; verdict is `"fail"` too, so
  behavior is consistent (Phase 1 test 6). No double message â€” the `.succeeded` gate
  returns before the verdict gate.
- **UNVERIFIED never fails (G3/over-claim guard):** `record.verdict == "fail"` is a strict
  equality, so `"unverified"` never trips it. Explicit tests in both phases.
- **`--json` + flag:** exit code applies; payload identical (tests assert key set).
- **`rerun`/`resume`:** inherit `fail_on_verdict=False`; assert unaffected (their existing
  tests must stay green â€” no new assertion needed, but do run them).
