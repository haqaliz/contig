## Implementation Phases

### Phase 1 — `db_version` field + honest header parsing (provenance foundation)

- **Goal:** Capture the annotation cache/build identifier into `AnnotationProvenance`.
- **Prerequisites:** none.
- **Files:**
  - `src/contig/models.py` — `AnnotationProvenance` (206-216): add
    `db_version: str | None = None`. No validator change (existing
    `_normalize_annotation_identity` `mode="before"` at 310-324 + Pydantic default cover
    pre-M5 load).
  - `src/contig/bundle.py` — `_parse_annotation_header` (113-126):
    - VEP branch: after version parse, scan the same `##VEP=` line for a `cache="..."`
      token; store `os.path.basename(cache_path.rstrip("/"))` (e.g. `110_GRCh38`) as
      `db_version`. Absent → `None`.
    - SnpEff branch: today only `##SnpEffVersion=` is read and returns immediately. Change
      so SnpEff DB is pulled from `##SnpEffCmd=` / `##SnpEffGenomeVersion=` lines in the
      same header block — i.e. scan all `header_lines` for the genome token (commit to a
      spelling: `##SnpEffCmd="SnpEff  GRCh38.105 ..."` → first token matching an assembly
      like `GRCh38.105`; `##SnpEffGenomeVersion=GRCh38.105` → value). Keep returning the
      `SnpEff` provenance with `version` from `##SnpEffVersion=` and `db_version` from the
      genome token. Absent → `None`. **Never fabricate.**
  - `tests/test_annotation_provenance.py` — extend fixtures: VEP fixture already has
    `cache="/vep/homo_sapiens/110_GRCh38"` (assert `db_version=="110_GRCh38"`); **add a
    SnpEff `##SnpEffCmd`/`##SnpEffGenomeVersion` line** to the SnpEff fixtures (none exists
    today) and assert its `db_version`; add a header-without-cache/genome case asserting
    `db_version is None`.
- **Steps (TDD):**
  1. RED: write the three parse assertions above (VEP cache, SnpEff genome, absent→None).
  2. GREEN: add the field, then the VEP cache-token parse and SnpEff genome scan.
  3. REFACTOR: factor a small `_extract_vep_cache(line)` / `_extract_snpeff_db(lines)` if it
     clarifies; keep null-safe.
- **Validation:** `uv run pytest tests/test_annotation_provenance.py`.
- **Commit:** `feat(verify): capture annotation cache/build into AnnotationProvenance [C7 M5]`

### Phase 4 — Shared corroboration helper (read, never recompute)

- **Goal:** One pure helper producing the "corroborated by" line or `None`.
- **Prerequisites:** none (independent of Phases 1-3; can run in parallel).
- **Files:**
  - `src/contig/verification/annotation_surface.py` (new):
    `corroborated_by_line(record: RunRecord) -> str | None`.
    - Find `consequence_concordance` and `gene_symbol_concordance` in
      `record.qc_results` (kind `"concordance"`, exact check names).
    - If `consequence_concordance` is absent or its `value is None` → return `None`
      (PRD D2). No recompute.
    - Annotator names from `record.annotation_identity` (fallback: none → still render using
      "VEP and SnpEff" only if both concordance metrics exist; prefer the identity list).
    - Format: `"Corroborated by {A} and {B}: {cons_matches}/{cons_total} consequences agree
      ({0.XX}); gene symbols {gs_matches}/{gs_total} ({0.XX}, informational)."` Parse
      matches/total from the `QCResult.message` counts OR carry them via `value` — since the
      message already contains `"{matches}/{total} shared site(s)"`, extract with a small
      regex; the parenthetical fraction is the rounded `value`. Mark gene-symbol
      **informational** (PRD D3/S-1). If gene-symbol is absent/`None`, render only the
      consequence half.
  - `tests/verification/test_annotation_surface.py` (new): dual-annotated → full line;
    consequence-only → half line; consequence `value None` → `None`; single-annotator
    fixture (concordance UNVERIFIED) → `None`.
- **Steps (TDD):** RED the four cases → GREEN. Keep it pure (no I/O, no recompute).
- **Validation:** `uv run pytest tests/verification/test_annotation_surface.py`.
- **Commit:** `feat(verify): corroborated-by line helper over M4 concordance results [C7 M5]`

## Environment & Reproducibility

- No env vars, no external tools, no network. `db_version` is derived from the output VCF
  header (reproduce-safe, like the rest of `compute_annotation_identity`), never a stored
  scratch path. Determinism preserved: parse is pure string handling; the helper is pure.

## Edge Cases & Error Handling

- Only one annotator / annotation absent / below shared-site floor → `consequence_concordance`
  value `None` → helper returns `None` → line omitted everywhere (PRD D2).
- Header carries tool but no cache/genome token → `db_version=None`, render tool+version
  only, no orphan "cache/build" label.
- VEP `cache` path with trailing slash / `file://` → basename after strip; be null-safe.
- Pre-M5 bundle (no `db_version` key) → loads, defaults `None`.
- Gene-symbol concordance low but informational → line marks it informational; never reads
  as a failure (S-1).
