### Phase 1 — `detect_missing_module` (pure, reproduce-local)

- **Goal:** extract the missing top-level package name from captured error text.
- **Prerequisites:** Phase 0.
- **Files:** `verification/reproduce.py`; `models.py` (`FailureClass`); new tests in
  `tests/test_reproduce_env_resurrection.py`.
- **Steps (TDD):**
  1. RED: write unit tests for `detect_missing_module(output: str) -> str | None`:
     - `"ModuleNotFoundError: No module named 'numpy'"` → `"numpy"`.
     - `"ModuleNotFoundError: No module named 'sklearn.utils'"` → `"sklearn"` (top-level only).
     - `"ImportError: cannot import name 'x' from 'pandas'"` → `"pandas"` (or `None` if we scope to
       `No module named` only — **decide in the test; recommended: match `No module named 'X'`
       first, and only that form this slice** to avoid false extractions from generic ImportErrors).
     - case-insensitive needle match; multi-line output with the error mid-stream.
     - no match → `None`; unsafe token (fails `^[A-Za-z0-9._-]+$`) → `None`.
  2. GREEN: implement with a compiled regex, e.g. `re.search(r"No module named ['\"]([^'\"]+)['\"]",
     output, re.IGNORECASE)`; take `group(1).split(".")[0]`; validate against
     `^[A-Za-z0-9._-]+$`; return `None` otherwise. Pure, no I/O — mirror
     `self_heal._parse_missing_index` discipline.
  3. GREEN: add `"missing_dependency"` to the `FailureClass` literal (`models.py:262-280`). Add a
     one-line test that the literal is accepted by `Diagnosis(failure_class="missing_dependency",
     …)`. **Do not** add a `diagnose_failure` branch or any corpus/holdout case for it (keeps the C6
     eval-guard baseline green — this literal stays reproduce-local, like `no_progress` is unreached).
- **Validation:** `uv run pytest tests/test_reproduce_env_resurrection.py`; run
  `uv run contig eval-guard` (or the CI step) to confirm the held-out baseline is unaffected.
- **Commit:** `feat(reproduce): detect_missing_module + missing_dependency class [C8 slice 2]`

### Phase 3 — Engine: opt-in, bounded install-retry loop + repair record

- **Goal:** the heal loop and its record.
- **Prerequisites:** Phases 0–2.
- **Files:** `models.py` (`ReproduceRecord`), `verification/reproduce.py`; tests.
- **Steps (TDD):**
  1. RED: add `repair_history: list[RepairStep] = []` to `ReproduceRecord` (`models.py:661-677`).
     Test: a `ReproduceRecord` built without it defaults to `[]`; JSON round-trip of a legacy dict
     (no key) loads with `[]` (back-compat, mirror the additive-field pattern).
  2. RED: write the engine tests (scripted executor + scripted installer) in
     `tests/test_reproduce_env_resurrection.py`:
     - **heals:** executor call 1 → `(1, "ModuleNotFoundError: No module named 'numpy'")`; installer
       records the module + returns 0; executor call 2 → `(0, "")` and writes `results.json`.
       Assert: claims classify (not all-unverified), `record.exit_code == 0`, `len(repair_history)
       == 1`, the step names `numpy`, installer called **exactly once**, executor called **twice**.
     - **flag off:** same executor, `allow_install=False` → installer never called, all-unverified,
       `exit_code == 1`, `repair_history == []`.
     - **no module:** executor 1 → `(1, "Segmentation fault")` → all-unverified, no install.
     - **install fails:** installer returns 1 → all-unverified, `repair_history` has one step with a
       give-up outcome, executor called **once** (no retry).
     - **retry still fails:** executor 2 → `(1, "…")` → all-unverified, `exit_code == 1`, one step.
     - **bounded:** even if the retried output names another missing module, installer is called at
       most once (single install + single retry).
     - **stale results guard:** pre-place a `results.json` with wrong numbers in the repo; the failed
       first run must NOT classify against it — only the retried run's freshly-written file counts
       (executor call 2 overwrites it). Assert the classified values are the retried ones.
  3. GREEN: extend `run_reproduction` signature (kw-only): `allow_install: bool = False`,
     `installer: Installer = default_installer`. Rewrite the non-zero branch (321-342): on
     `exit_code != 0`:
     - if `not allow_install` → existing all-unverified short-circuit (with `exit_code`).
     - else `module = detect_missing_module(run_output)`; if `module is None` → all-unverified.
     - else (one-shot; no loop needed since capped at one): `install_rc = installer(
       _pip_install_argv(module), repo_path)`. Record a `RepairStep(attempt=1,
       diagnosis=Diagnosis(failure_class="missing_dependency", root_cause=…, evidence=[matched
       line], confidence=…), patch=Patch(kind="env", operation={"install": module}, rationale=…,
       risk="needs_confirmation", expected_signal="run exits 0 after install"), outcome=…,
       detail=…)`.
     - if `install_rc != 0` → all-unverified, outcome `"install_failed"`, record it.
     - else re-invoke `exit_code, run_output = executor(shlex.split(run_command), repo_path)`
       **once**; if still non-zero → all-unverified, outcome `"retry_failed"`; else fall through to
       the normal classification path (344+), outcome `"installed_and_retried"`.
     - Thread `repair_history` into **every** `ReproduceRecord(...)` return (including the healed
       path at the end) and set `exit_code` to the final run's exit.
  4. REFACTOR: extract a small `_resurrect(...) -> tuple[int, str, list[RepairStep]]` helper if the
     branch gets dense, keeping the happy path readable.
- **Validation:** `uv run pytest tests/test_reproduce_env_resurrection.py tests/test_reproduce.py`
  green; full suite green.
- **Commit:** `feat(reproduce): opt-in bounded install-retry env resurrection [C8 slice 2]`

## 6. Edge Cases & Error Handling

- Flag off → byte-identical slice-1.5 behavior (all-unverified on non-zero).
- No module detected / unsafe token / install fails / retry fails → honest `UNVERIFIED`, recorded.
- Import-name≠package-name (`cv2`) → install fails → `UNVERIFIED` (M5, honest).
- Stale results file from the failed first run must not be classified (Phase 3 test).
- Bounded: exactly one install + one retry — provable termination (no `while` needed).
