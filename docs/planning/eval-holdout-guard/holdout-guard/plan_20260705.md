## Phase B — Baseline record + pure comparator

**Goal:** Load/save a committed baseline and compute a guard result — all pure, file-free where
possible, so it is fast and deterministic to test.
**Prerequisites:** Phase A module exists.
**Files:** `src/contig/models.py` (add `HoldoutGuardResult`); `src/contig/holdout.py`
(`default_baseline_path`, `load_baseline`, `save_baseline`, `compare_to_baseline`); tests in
`tests/test_eval_holdout.py`.

**Data model — `HoldoutGuardResult` (models.py, Pydantic BaseModel, JSON-serializable):**
```
detector: str
holdout_size: int
accuracy: float
baseline_accuracy: float | None = None
delta: float | None = None          # accuracy - baseline_accuracy
tolerance: float
regressed: bool = False
improved: bool = False
holdout_sha: str
baseline_sha: str | None = None
sha_mismatch: bool = False
detector_mismatch: bool = False     # baseline.detector != detector
has_baseline: bool = True
mismatches: list[DetectorMismatch] = []
```

**Baseline artifact:** a single `EvalSnapshot` (reuse `models.py:397-414`) serialized as one JSON
object (NOT JSONL) to `src/contig/data/holdout_baseline.json`.
- `default_baseline_path() -> Path` → `.../data/holdout_baseline.json`.
- `save_baseline(snapshot: EvalSnapshot, path) -> None`: `Path(path).write_text(
  snapshot.model_dump_json(indent=2) + "\n")` (pretty so the committed file diffs cleanly).
- `load_baseline(path) -> EvalSnapshot | None`: missing file → `None`; else
  `EvalSnapshot.model_validate_json(Path(path).read_text())`.

**`compare_to_baseline` (pure):**
```
def compare_to_baseline(
    report: DetectorEvalReport, *, baseline: EvalSnapshot | None,
    holdout_sha: str, holdout_size: int, detector: str, tolerance: float,
) -> HoldoutGuardResult
```
Semantics:
- `has_baseline = baseline is not None`.
- If no baseline: `regressed=False, improved=False`, `baseline_accuracy=None`, `delta=None`
  (the CLI turns "no baseline + not updating" into an error exit; the comparator stays pure).
- Else: `delta = report.accuracy - baseline.accuracy`;
  `regressed = report.accuracy < baseline.accuracy - tolerance`;
  `improved  = report.accuracy > baseline.accuracy + tolerance`;
  `sha_mismatch = holdout_sha != baseline.corpus_sha`;
  `detector_mismatch = detector != baseline.detector`.
- Carry `report.mismatches` through so the CLI can name newly-missed cases.

**Steps:**
1. **RED** (unit, no files needed — build `DetectorEvalReport`/`EvalSnapshot` inline):
   - `test_compare_pass`: accuracy == baseline → `regressed=False, improved=False`.
   - `test_compare_regression` (AC3 core): accuracy < baseline - tol → `regressed=True`, delta<0.
   - `test_compare_improvement`: accuracy > baseline + tol → `improved=True`.
   - `test_compare_tolerance`: accuracy == baseline - tol/2 → not regressed (float-noise absorbed).
   - `test_compare_no_baseline`: baseline=None → `has_baseline=False`, not regressed.
   - `test_compare_sha_and_detector_mismatch`: differing sha/detector set the two flags True.
   - `test_baseline_roundtrip`: `save_baseline` then `load_baseline` returns an equal snapshot;
     missing path → None.
2. **GREEN:** implement the model + functions.
3. **Also (AC3 at the eval level):** `test_worse_detector_scores_lower`: define an in-test stub
   `def _worse(events, log_text): return Diagnosis(failure_class="unknown", ...)` (a plain
   `Detector` callable — check `Diagnosis` required fields in `models.py:228-231` and fill
   minimal valid values); assert `evaluate_detector(holdout_cases, _worse).accuracy` <
   `evaluate_detector(holdout_cases).accuracy` (rules). This is the "deliberately worse detector"
   the roadmap acceptance names, exercised without touching the registry.

**Validation:** `uv run pytest tests/test_eval_holdout.py` green; whole suite green.
**Commit:** `feat(holdout): baseline record + pure regression comparator`

## Phase C — `eval-guard` CLI command

**Goal:** The user-facing guard: score → compare → warn/nudge → exit 0/1; plus `--update-baseline`.
**Prerequisites:** Phases A + B.
**Files:** `src/contig/cli.py` (add `eval-guard`); CLI tests in `tests/test_eval_holdout.py`
(module-level `runner = CliRunner()`, `from contig.cli import app`, mirror
`tests/test_cli.py:1797-1936`).

**Command (mirror `eval-detector`, `cli.py:1485-1553`):**
```
@app.command(name="eval-guard")
def eval_guard(
    holdout: str = typer.Option(None, "--holdout", help="Held-out corpus JSONL (defaults to the shipped frozen set)."),
    baseline: str = typer.Option(None, "--baseline", help="Baseline JSON (defaults to the shipped one)."),
    detector: str = typer.Option("rules", "--detector", help="Which detector to guard (rules, rules-strict, ...)."),
    tolerance: float = typer.Option(1e-9, "--tolerance", help="Float tolerance; accuracy below (baseline - tolerance) is a regression."),
    update_baseline: bool = typer.Option(False, "--update-baseline", help="(Re)freeze the baseline to the current held-out accuracy. Deliberate, reviewed act."),
    json_out: bool = typer.Option(False, "--json", help="Emit the guard result as JSON."),
) -> None:
```
Flow:
1. Resolve `holdout_path = Path(holdout) if holdout else default_holdout_path()`;
   `baseline_path = Path(baseline) if baseline else default_baseline_path()`.
2. `detector_fn = get_detector(detector)` — on `KeyError`, `typer.echo(str(exc).strip("\"'"),
   err=True); raise typer.Exit(1)` (copy `cli.py:1517-1523`).
3. `load_corpus(holdout_path)` — on `FileNotFoundError`, echo + `Exit(1)` (copy `cli.py:1526-1530`).
4. `report = evaluate_detector(cases, detector_fn)`; `holdout_sha = sha256_file(holdout_path)`.
5. **If `update_baseline`:** build `snapshot_from_report(report, timestamp=datetime.now(
   timezone.utc).isoformat(), corpus_size=len(cases), corpus_sha=holdout_sha,
   contig_version=_pkg_version("contig"), detector=detector)`; `save_baseline(snapshot,
   baseline_path)`; echo `Baseline updated: accuracy X% over N held-out cases (detector=..., sha
   ...)`; **return (exit 0).** Updating never fails.
6. **Else (guard):** `baseline = load_baseline(baseline_path)`;
   `result = compare_to_baseline(report, baseline=baseline, holdout_sha=holdout_sha,
   holdout_size=len(cases), detector=detector, tolerance=tolerance)`.
   - If `json_out`: `typer.echo(result.model_dump_json())`. (Still apply the exit code below.)
   - **No baseline (AC5):** `typer.echo("No held-out baseline at <path>; run 'contig eval-guard
     --update-baseline' to freeze one.", err=True); raise typer.Exit(1)`.
   - **Loud warnings (AC5), non-failing, to stderr:** if `result.sha_mismatch`: warn
     "Held-out set changed (sha ... != baseline ...); the delta crosses different sets — refreeze
     with --update-baseline." If `result.detector_mismatch`: warn "Baseline was measured with
     detector '<baseline.detector>', guarding '<detector>' — comparison crosses detectors."
   - **Text render** (non-json): `Guard: accuracy X% vs baseline Y% (delta +/-Z pp) over N held-out
     cases [detector]`, then `MISS <case_id>: expected .. predicted ..` per mismatch.
   - **Improved nudge (AC5), stdout, exit 0:** if `result.improved`: echo "Held-out accuracy
     improved (X% > baseline Y%); consider --update-baseline to lock it in."
   - **Regression (AC3), stderr, exit 1:** if `result.regressed`: echo "REGRESSION: accuracy X%
     below baseline Y% (delta -Z pp)."; `raise typer.Exit(1)`.
   - Else echo "Guard PASS: accuracy X% ≥ baseline Y%." and return (exit 0).

**Steps:**
1. **RED** CLI tests (use `runner.invoke(app, [...])`, always pass `--baseline
   str(tmp_path/'baseline.json')` and, where writing, `--holdout` pointing at a tmp copy so the
   committed files are never touched — mirror `test_cli.py:1874-1876`):
   - `test_guard_update_then_pass` (AC4): invoke `--update-baseline` (writes tmp baseline), then
     invoke guard → exit 0, "Guard PASS".
   - `test_guard_regression_worse_detector` (AC3): freeze baseline with `--detector rules`, then
     register a worse stub detector and invoke `--detector worse` → exit_code == 1, "REGRESSION"
     in output. **Registering the stub:** `monkeypatch.setitem(contig.detect.DETECTORS, "worse",
     _worse)` — first confirm `get_detector` resolves plain names via `DETECTORS` (detect.py:599-614);
     if it does not, `monkeypatch.setattr(contig.cli, "get_detector", lambda n: _worse if n==
     "worse" else _real)` as a fallback.
   - `test_guard_no_baseline` (AC5): guard with a `--baseline` path that does not exist → exit 1,
     "No held-out baseline".
   - `test_guard_sha_mismatch_warns` (AC5): freeze baseline, then guard against a held-out tmp
     file with an extra case (sha differs) but the good detector → exit 0 AND a sha-mismatch
     warning on stderr (`result.stderr` or combined output; use `CliRunner(mix_stderr=...)` per
     the repo's existing pattern — check `test_cli.py`).
   - `test_guard_json` : `--json` emits parseable `HoldoutGuardResult` with `regressed` present.
2. **GREEN:** implement the command.

**Validation:** `uv run pytest tests/test_eval_holdout.py` green; **whole suite green** (`uv run
pytest`).
**Commit:** `feat(cli): add eval-guard — held-out regression guard (exit≠0 on drop)`

## Testing Strategy

- Unit (fast, no files): `compare_to_baseline` semantics; baseline round-trip; worse-stub scores
  lower. Written first (RED) in Phase B.
- Data/integration: held-out loads, disjoint from training, source-kind, leakage guard (Phase A).
- CLI (CliRunner): update→pass, regression→exit 1, no-baseline→exit 1, sha-mismatch warning,
  json output (Phase C).
- Commands: `uv run pytest` (whole suite must stay green each phase).

## Environment & Reproducibility

- `uv sync` already done in the worktree. No env vars; `llm` detector must never be the guard
  default (determinism). The baseline pins the held-out `corpus_sha` so an accuracy change is
  attributable to detector vs held-out-set change (the existing `eval_history` invariant,
  `eval_history.py:1-7`). No network anywhere in tests.

## Edge Cases & Error Handling

- Missing baseline → exit 1 with guidance (not a crash).
- Missing held-out file / bad detector name → exit 1, mirroring `eval-detector`.
- Held-out set changed but baseline not refreshed → loud warning, still runs (delta flagged as
  cross-set). Detector ≠ baseline detector → loud warning.
- Accuracy exactly at baseline → PASS (only a real drop beyond tolerance fails).
- Improvement → nudge, never a failure.
