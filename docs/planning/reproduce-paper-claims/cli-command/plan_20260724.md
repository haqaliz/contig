## Phase 2 — the command: happy path + round-trip invariant (RED→GREEN)

- **Goal:** `extract-claims` reads text, writes a **load_claims-valid** draft + sidecar.
- **Steps (test-first, `CliRunner` + `tmp_path` + `monkeypatch`):**
  1. RED: a fixture `.md` written to `tmp_path`; invoke `extract-claims <paper> --out
     <tmp>/claims.draft.json`; assert exit 0, the draft **loads via `load_claims`** (the central
     invariant), and `<tmp>/claims.draft.review.md` exists and contains a source sentence.
  2. RED: the draft JSON claims are **locator-less** (`id`/`value`/`tolerance` keys only).
  3. RED: the sidecar path derivation: `.json` → `.review.md`; a non-`.json` out → append
     `.review.md`.
  4. GREEN: implement the command:
     - Resolve `paper`; `stat()` first: missing/dir → Exit(1); size > `_MAX_MATCH_BYTES` → Exit(1)
       naming size+cap; read UTF-8, `UnicodeDecodeError` → Exit(1). Nothing written on any failure.
     - `claims = default_extractor(text, use_llm=not no_llm)`.
     - Build `[{"id","value","tolerance"}]`; serialize; **write to a temp file in the out dir,
       call `load_claims(tmp)`; on `ClaimsError` remove temp + Exit(1) internal-error; on success
       `os.replace` to `--out`.**
     - Write the sidecar (header + per-claim `id`, `value`, `unit or "—"`, `origin`,
       `source_text`; llm/flagged "review first"; locator + %-scale instructions).
     - Echo the summary; exit 0.
- **Validation:** `uv run pytest tests/test_cli_extract_claims.py -q`.
- **Commit:** `feat(cli): contig extract-claims writes a load_claims-valid draft + sidecar`

## Testing strategy & reproducibility

- `CliRunner`, `tmp_path`, `monkeypatch`; on-disk fixture `.md`. The LLM is never real in CI
  (either `--no-llm`, unconfigured env, or a monkeypatched `default_extractor`).
- Determinism: fixed fixture text → identical draft/sidecar bytes across runs (ids deterministic).

## Edge cases & error handling

- Out dir doesn't exist → create parents, or Exit(1) with a clear message (pick one; test it).
- Sidecar write failure after the draft is written → surface the error; the draft (already
  `os.replace`d) stands. Note in the summary.
- `--out` equal to the input path → refuse (would clobber the paper); test it.
- A claim `value` that is an integer-valued float (`87.0`) serializes as `87.0` — fine for
  `load_claims` (numeric). Confirm no int/float surprise.
