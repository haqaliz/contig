- **Extends** `src/contig/verification/claim_extraction.py` (adds `extract_with_llm`,

- **Reuses `detect.py` env-gating** (confirmed symbols):
  - `_LLM_PROVIDER_KEY_ENV` (`detect.py:417`) ‚Äî `{"claude": "ANTHROPIC_API_KEY", "openai":
    "OPENAI_API_KEY"}`.
  - `_selected_provider()` (`detect.py:427`) ‚Äî returns the provider only when
    `CONTIG_LLM_PROVIDER` is set to a known provider **and** its key env is non-empty, else
    `None`. **Import and reuse this** ‚Äî do not re-implement the gate.
  - Pattern precedent: `_llm_complete` (`detect.py:470`, the single lazy-SDK/network touch point),
    `_extract_json_object` (`detect.py:519`, tolerant JSON-from-prose), `_diagnosis_from_reply`
    (swallows all errors).
- **Decision (from spec):** define a **local** `_llm_complete` in `claim_extraction.py` (its own
  monkeypatch point, extraction-specific prompt) rather than calling `detect._llm_complete`, so
  extraction is independently mockable. Reuse `detect._selected_provider` for the gate only.

## Phase 1 ‚Äî the env-gated seam (RED‚ÜíGREEN)

- **Goal:** `extract_with_llm` returns `[]` when unconfigured; calls the seam when configured.
- **Steps (test-first; `tests/test_claim_extraction_llm.py`, mirroring `test_detect_llm.py`):**
  1. RED: with no `CONTIG_LLM_PROVIDER` env ‚Üí `extract_with_llm("...")` returns `[]` and the seam
     is **not** called (spy). Use `monkeypatch.delenv`.
  2. RED: with `CONTIG_LLM_PROVIDER=claude` + `ANTHROPIC_API_KEY=test-key` and
     `monkeypatch.setattr(claim_extraction, "_llm_complete", fake)` returning a canned JSON list
     ‚Üí the seam is called once with `(provider, prompt)`.
  3. GREEN: implement `extract_with_llm(text)`: `provider = detect._selected_provider(); if
     provider is None: return []`; build the extraction prompt; `reply = _llm_complete(provider,
     prompt)`; `return _claims_from_reply(reply)`. Implement `_llm_complete(provider, prompt)` as
     the lazy-import seam (copy `detect._llm_complete`'s shape; SDK imported **inside**).
- **Validation:** `uv run pytest tests/test_claim_extraction_llm.py -q`.
- **Commit:** `feat(claim-extraction): optional env-gated LLM extraction seam`

## Phase 2 ‚Äî defensive reply parse (RED‚ÜíGREEN)

- **Goal:** `_claims_from_reply(reply) -> list[ExtractedClaim]` ‚Äî tolerant and never-raising.
- **Steps (test-first):**
  1. RED: a clean JSON list reply ‚Üí `ExtractedClaim`s with `origin="llm"`, ids via the shared
     `_slug` uniquifier, `unit` carried, `value` finite-float-validated (non-bool).
  2. RED: **prose-wrapped** JSON list (`"Here are the claims: [ ... ] hope this helps"`) ‚Üí parsed
     (list-analogue of `_extract_json_object`; find the first balanced `[...]`).
  3. RED: seam raises / returns `"not json"` / returns a JSON object not a list / an entry missing
     `value` / a non-finite value ‚Üí each contributes **nothing**, returns `[]`, never raises.
  4. RED: **no API key leak** ‚Äî set a distinctive dummy key, run, assert it appears in no returned
     `source_text`/id and (if the impl logs) no log (mirror
     `test_detector_does_not_log_the_api_key`).
  5. GREEN: implement, swallowing every exception into `[]` (mirror `_diagnosis_from_reply`).
- **Validation:** `uv run pytest tests/test_claim_extraction_llm.py -q`.
- **Commit:** `feat(claim-extraction): defensive LLM reply parser (never raises)`

## Testing strategy & reproducibility

- **No real provider in CI, ever** ‚Äî the seam is monkeypatched in every test; the real path is
  shape-asserted only. Env set with dummy values.
- Manual gate (pre-merge, human-run): real provider once; record in PR.

## Edge cases

- Provider configured but SDK not installed ‚Üí the lazy import inside `_llm_complete` raises; the
  caller swallows ‚Üí `[]` (degrade to core-only). Add a test with the import monkeypatched to raise.
- Reply with duplicate claims ‚Üí deduped by the same `(slug, value)` key.
- Circular import risk: `claim_extraction` importing `detect` ‚Äî `detect` does not import
  `claim_extraction`, so it's safe; confirm no import cycle in a test.
