# Aspect spec: llm-assist

Parent PRD: `../prd.md`. Aspect 2 of 3. Depends on `extractor-core`.

## Problem slice & user outcome

An **optional**, env-gated LLM assist that raises extraction recall — catching claims phrased
in ways the regex core misses — without ever being required, and without touching CI. Mirrors
the shipped `llm` detector precedent (`detect.py:410-633`) exactly.

## In scope

- **`extract_with_llm(text: str) -> list[ExtractedClaim]`** in `claim_extraction.py`:
  - **Env gate:** reuse `detect._selected_provider()` (single source of truth: `CONTIG_LLM_PROVIDER`
    + provider→key map `ANTHROPIC_API_KEY`/`OPENAI_API_KEY`). Unconfigured → return `[]` (pure
    no-op; the deterministic core stands alone).
  - **One network/SDK touch point:** a module-local seam `_llm_complete(provider, prompt) -> str`
    (extraction-specific prompt; its **own** monkeypatch point, independent of `detect`'s). The
    provider SDK is imported **lazily inside** it and **never** at module import → importing
    `claim_extraction` never pulls a provider SDK. (tech-plan settles reuse-vs-replicate of
    `detect._llm_complete`; default is a local seam for independent mockability.)
  - **Prompt:** instruct the model to return a JSON list of
    `{"metric": str, "value": number, "unit": str|null, "source_text": str}` and nothing else.
  - **Defensive reply parse** `_claims_from_reply(reply) -> list[ExtractedClaim]`: tolerant of a
    JSON list wrapped in prose (list-analogue of `detect._extract_json_object`); every entry
    validated (finite numeric `value`, non-bool; ids generated by the same slug+uniquify rule as
    the core); `origin = "llm"`. **Swallows every error** (provider/network/parse/shape) →
    returns `[]`, mirroring `_diagnosis_from_reply`. Never raises, never crashes the caller.
  - **No secret leakage:** the key is read only inside the seam and never logged (pinned by a
    test, like `test_detector_does_not_log_the_api_key`).
- **`merge_claims(core: list, llm: list) -> list`** in `claim_extraction.py`:
  - Union deduped by **`(metric_slug, value)`** (same key as the core). **Core wins** on a
    collision (deterministic, tested); LLM-only candidates are appended in order with
    `origin = "llm"`. Ids re-uniquified across the merged set so the final list has unique ids.

## Out of scope (this aspect)

- CLI wiring, file I/O, sidecar (aspect `cli-command`).
- Any real provider call in CI; prompt-engineering beyond a correct, minimal extraction prompt.
- Streaming, retries, multi-provider fan-out, cost controls.

## Acceptance criteria (test-first)

The `test_detect_llm.py` discipline — the real provider is **never** called in CI:

- **Unconfigured env → `extract_with_llm` returns `[]`** (no seam call, no import).
- **Configured (dummy `CONTIG_LLM_PROVIDER=claude` + `ANTHROPIC_API_KEY=test-key`) + seam
  monkeypatched** to return a canned JSON list → parsed into `ExtractedClaim`s with `origin ==
  "llm"`.
- **Prose-wrapped JSON** in the reply is tolerated (the list is extracted from surrounding text).
- **Seam raises / returns garbage / invalid shape → `[]`** (core-only survives; never crashes).
- **No API key** appears in any output/log (key-leak test).
- **`merge_claims`**: a claim present in both core and llm appears **once**, `origin ==
  "heuristic"` (core wins); an llm-only claim appears with `origin == "llm"`; all ids unique.
- The real `_llm_complete` is asserted for **prompt/shape only** (constructs the right argv/
  request), **never executed** in CI. A **manual pre-merge gate** (documented in the plan) runs
  it once against a real provider.

## Manual pre-merge gate (human-run, once)

CI never calls a real provider: the `_llm_complete` seam is monkeypatched in
every test, and the real seam is **shape-asserted only** (a fake `anthropic`/
`openai` module injected into `sys.modules`; the built request's `model`/
`max_tokens`/`messages` are asserted and the key is confirmed absent from the
request — no network). Before merge, a human runs the real path **once** with a
real `CONTIG_LLM_PROVIDER` + key against a fixture paper, confirms claims come
back (`origin=='llm'`), nothing raises, and no key leaks, then records the result
in the PR description. The gate stays manual by design — a real key must never
live in CI. (Full command in the plan's "Manual pre-merge gate" section.)

## Dependencies & sequencing

After `extractor-core` (imports `ExtractedClaim`, the slug/uniquify helper, the dedup key).
Before `cli-command` (which composes core + llm via `default_extractor`).

## Open questions / risks

- Reuse `detect._llm_complete` directly vs. a local seam — local seam preferred for independent
  mocking; tech-plan makes the call.
- LLM value/unit trust: an LLM-reported `value` is validated as a finite float but its
  correctness is the user's review responsibility (draft-for-review contract); the sidecar marks
  `llm`-origin claims so they get extra scrutiny.
