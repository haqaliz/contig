## Phase 1 — Pure path walker `resolve_pointer`

- **Goal:** a pure, stdlib `resolve_pointer(data, expr) -> object | None` (dotted + `[n]`, leading
  `$.`/`$` tolerated) and its tokenizer. No I/O. (AC1, AC2)
- **Prerequisites:** none.
- **Files:**
  - modify `src/contig/verification/reproduce.py` (add walker near the top, after imports)
  - create `tests/test_reproduce_locator.py`
- **RED — write these tests first** (`tests/test_reproduce_locator.py`):
  - AC1: `resolve_pointer({"model":{"auc":0.9}}, "$.model.auc") == 0.9`;
    `resolve_pointer({"samples":[{"n":5}]}, "samples[0].n") == 5`;
    `resolve_pointer([{"name":"x"}], "[0].name") == "x"` (top-level list);
    `$.model.auc` / `model.auc` / `$model` variants where applicable are equivalent for a dict root.
  - AC2 (each returns `None`, never raises): missing key `"model.missing"`; index OOR
    `"samples[9].n"`; index on a dict `"model[0]"`; key on a list `"samples.n"`; malformed
    `"a..b"`, `"a[x]"`, `"a["`, `"a[0]b"`, `"a."`, `""`, `"$"`, `"$."`; a value that is JSON `null`.
- **GREEN — implement** (reference shape; refine under TDD):

  ```python
  import re  # only if the impl uses it; the reference parser below is regex-free

  def _parse_path(expr: str) -> list[str | int] | None:
      """Tokenize a dotted+[n] path into keys (str) and indices (int).
      Leading '$' and one leading '.' are stripped. Returns None on any
      malformed expression -- the caller treats None as 'unresolved'."""
      s = expr.strip()
      if s.startswith("$"):
          s = s[1:]
      if s.startswith("."):
          s = s[1:]
      if not s:
          return None
      tokens: list[str | int] = []
      i, n = 0, len(s)
      first = True
      while i < n:
          c = s[i]
          if c == "[":
              j = s.find("]", i)
              if j == -1:
                  return None
              inner = s[i + 1 : j]
              if not inner.isdigit():   # rejects empty, sign, spaces, non-digit
                  return None
              tokens.append(int(inner))
              i = j + 1
          elif c == ".":
              if first:
                  return None
              i += 1
              if i >= n or s[i] in ".[":
                  return None
              start = i
              while i < n and s[i] not in ".[":
                  i += 1
              tokens.append(s[start:i])
          else:  # a bare key -- only valid as the very first accessor
              if not first:
                  return None
              start = i
              while i < n and s[i] not in ".[":
                  i += 1
              tokens.append(s[start:i])
          first = False
      return tokens or None

  def resolve_pointer(data: object, expr: str) -> object | None:
      """Walk `data` (nested dict/list from parsed JSON) by `expr`.
      Any unresolved step -> None. Never raises. Never guesses."""
      tokens = _parse_path(expr)
      if tokens is None:
          return None
      cur = data
      for tok in tokens:
          if isinstance(tok, int):
              if isinstance(cur, list) and 0 <= tok < len(cur):
                  cur = cur[tok]
              else:
                  return None
          else:
              if isinstance(cur, dict) and tok in cur:
                  cur = cur[tok]
              else:
                  return None
      return cur
  ```

## Edge cases & error handling (all → `UNVERIFIED`, never a false pass / never `DIVERGED`)

- Missing `from` file · unparseable `from` JSON · unresolved `path` · target non-number (incl.
  **numeric string**, strict) · target `bool` · target `NaN`/`inf` · (safety) `from` escapes repo
  reaching the engine. Each carries a specific message.
- Malformed locator *structure* (only one of `from`/`path`, non-string, empty) → `ClaimsError` at
  load (exit non-zero, no record).
- Escaping/absolute `from` at the CLI → refused pre-run (exit non-zero, no record).
- Non-zero script exit → every claim (located or flat) `unverified` (unchanged short-circuit).
- Mixed claims file → each claim resolves by its own path; a located claim never consults
  `--results`, a flat claim never has a locator.
