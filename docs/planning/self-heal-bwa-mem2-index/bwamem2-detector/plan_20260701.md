### Phase 1 — bwa-mem2 detector branch

- **Goal:** classify the bwa-mem2 unreadable-index failure as `missing_index`.
- **Prereq:** none.
- **Files:** `tests/test_detect.py` (RED), `src/contig/detect.py` (GREEN).
- **Steps:**
  1. **RED** — in `tests/test_detect.py`, after the classic-bwa test (`:262-267`), add:
     - `test_bwamem2_unreadable_index_is_missing_index`: events
       `[TaskEvent(process="BWA_MEM2_MEM", status="FAILED", exit=1)]`, log
       `"ERROR! Unable to open the file: /work/idx/genome.fasta.bwt.2bit.64"`; assert
       `d.failure_class == "missing_index"` and `any("bwt.2bit.64" in e for e in d.evidence)`.
     - `test_bwamem2_branch_does_not_swallow_wrong_reference`: reuse a wrong-reference line
       (mirror `:270-276`) and assert `!= "missing_index"` (guards over-match). The
       existing `test_bwa_missing_index_is_missing_index` (`:262-267`) already guards that
       the classic-bwa branch is untouched — no change needed there.
     - Run `uv run pytest tests/test_detect.py -k bwamem2` → the positive test FAILS
       (currently `tool_crash`), confirming RED.
  2. **GREEN** — in `src/contig/detect.py`, insert a branch immediately **after line 244**
     (after the classic-bwa `if bwa_missing_lines:` block, before the `nosuchfile_lines`
     block at `:246`):
     ```python
     # bwa-mem2's index loader (FMI_search.cpp) prints this exact line then
     # exit(EXIT_FAILURE) when a sidecar is missing/truncated/version-incompatible.
     # The message is generic ("Unable to open the file"), so gate on the
     # bwa-mem2-only sidecar token `.bwt.2bit.64` to stay narrow: a wrong-reference
     # line carries neither token, and classic bwa keeps its own branch above.
     bwamem2_missing_lines = [
         line
         for line in _matching_lines(log_text, ("unable to open the file",))
         if _has_any(line, ("bwt.2bit.64",))
     ]
     if bwamem2_missing_lines:
         return Diagnosis(
             failure_class="missing_index",
             root_cause="bwa-mem2's index is missing, incomplete, or incompatible.",
             evidence=bwamem2_missing_lines,
             confidence=0.85,
         )
     ```
     Match the surrounding style (`_matching_lines` is case-insensitive; keep confidence
     0.85 like its siblings).
  3. Run `uv run pytest tests/test_detect.py` → all green.
- **Validation:** `uv run pytest tests/test_detect.py` passes; `uv run pytest` still green.
- **Commit:** `feat(detect): classify bwa-mem2 unreadable-index failure as missing_index`

## Environment & reproducibility

- No env vars, no external tools, no network. Determinism preserved: detection is pure
  string matching over the run's own log; the corpus is a static committed asset.

## Edge cases & error handling

- Generic `Unable to open the file` over-match → prevented by the `bwt.2bit.64` AND-guard.
- Ancillary "not found" line from bwa-mem2 → the generic index branch (`:165-169`) would
  also yield `missing_index` (same class), so no wrong-class risk; the corpus `log_text` is
  the isolated error line to target the new branch.
- JSONL corruption → validate the file parses after the append (build-order step 4).
