## Phase 2 — Pure resolver (`resolve_match`)

- **Goal:** given text + a pattern, return the captured string or an honest `(None, reason)`.
  Crash-proof on every input.
- **Prereqs:** Phase 1 (types only; the function itself is type-agnostic).
- **Files:** `reproduce.py` (`resolve_match` + `_MAX_MATCH_BYTES`); **new**
  `tests/test_reproduce_pattern_locator.py` (mirror `tests/test_reproduce_tsv_locator.py`).
- **RED — tests first**, pure, no engine:
  - Happy: one match with one group → the group-1 string; one match, **no groups** → the whole
    match; a named group `(?P<v>…)` is group 1 and resolves; inline `(?i)`/`(?m)` honored.
  - **Strict ambiguity:** 0 matches → `(None, msg)` where `"0"` is in `msg`; 3 matches →
    `(None, msg)` where `"3"` is in `msg`. (Mirrors `resolve_cell`'s "matched N rows" wording.)
  - **Non-participating group** (`r"(old)?|AUC=([\d.]+)"` against `"AUC=0.9"`, and the simpler
    `r"(?:x)?(y)?z"` shapes) → `(None, msg)`, **never `TypeError`**. *Critique gap #1 — the one
    case that can crash rather than degrade.*
  - Multi-group pattern → **group 1 only** is returned (documents M3 unambiguously).
  - Oversized text (build a string just over `_MAX_MATCH_BYTES`) → `(None, msg)` naming the size;
    do this with a cheap `"x" * (_MAX_MATCH_BYTES + 1)` and assert no match attempt is made.
  - Uncompilable pattern passed directly → `(None, msg)`, never raises (belt-and-braces).
  - `test_resolve_match_never_raises_on_wild_inputs`: a grab-bag of patterns
    (`""`, `"("`, `"*"`, `"(?P<>)"`, `"a{100000,}"`, `"[z-a]"`) × texts (`""`, `"abc"`, a
    multiline blob) — every call returns a 2-tuple of `(str | None, str)`.
- **GREEN — implement:**
  ```python
  def resolve_match(text: str, pattern: str) -> tuple[str | None, str]:
      if len(text) > _MAX_MATCH_BYTES:  # bound BEFORE compiling/scanning
          return None, f"text is {len(text)} chars, over the {_MAX_MATCH_BYTES} limit"
      try:
          compiled = re.compile(pattern)
      except re.error as exc:
          return None, f"pattern {pattern!r} is not a valid regex: {exc}"
      matches = list(compiled.finditer(text))
      if len(matches) != 1:
          return None, f"pattern {pattern!r} matched {len(matches)} times"
      m = matches[0]
      captured = m.group(1) if compiled.groups else m.group(0)
      if captured is None:
          return None, f"pattern {pattern!r} capture group did not participate in the match"
      return captured, ""
  ```
  Docstring must state: never raises; strict 0-or-many; group 1 else whole match; the returned
  value is always a raw **string** (float-parsing is the caller's job, as with `resolve_cell`).
- **Validation:** `uv run pytest tests/test_reproduce_pattern_locator.py -q` green; full green.
- **Commit:** `feat(reproduce): pure never-raising regex capture resolver (C8 slice 4)`

## Phase 3 — Engine branch (`_observe_pattern_located` + explicit dispatch)

- **Goal:** a pattern claim on a completed run binds its observed value and classifies; stdout mode
  reads the (possibly retried) run output; file mode is contained and cached; failures →
  UNVERIFIED.
- **Prereqs:** Phases 1-2.
- **Files:** `reproduce.py` (`_text_cache` at `:590-591`, `_observe_pattern_located` after `:685`,
  dispatch head `:780-785`); `tests/test_reproduce.py`.
- **RED — tests first** (via `run_reproduction` with the `_fake_executor` at `:614` — note its
  `output=` parameter already exists — and `tmp_path` fixture logs):
  - **stdout mode:** executor returns `(0, "Final AUC: 0.91\n")` and writes **no** results file →
    `reproduced`; drifted value → `diverged` (message names observed-vs-stated + delta); near →
    `within_tolerance`.
  - **file mode:** fixture `logs/train.log` in the repo → resolves; missing file → `unverified`;
    a **directory** `from` → `unverified`; non-UTF-8 file → `unverified`, never raises.
  - 0 matches / 3 matches → `unverified`, count in the message; never `diverged`.
  - Unparseable capture (`"Final AUC: NA"`) → `unverified`; non-finite (`"inf"`) → `unverified`.
  - **Numeric-string capture is the normal case** → it *is* the observed value (the slice-3 rule,
    deliberately unlike the JSON locator's strict rule).
  - **Retried-run output** (PRD M5): `_ScriptedExecutor` (copy from
    `tests/test_reproduce_env_resurrection.py:71`) scripted
    `[(1, "No module named 'numpy'"), (0, "Final AUC: 0.91")]` + `_ScriptedInstaller(0)` +
    `allow_install=True` → the claim resolves from the **second** run's output.
  - **Escaping `from` reaching the engine directly** → `unverified`, file never read (mirror
    `:945`/`:1314` style).
  - **A `from`-less claim never touches the filesystem** (assert via a `monkeypatch` spy on
    `Path.read_text`, or simply that it resolves with no such file present).
  - Same file, two pattern claims → read once (spy on the read, mirroring
    `_table_claim_same_file_parsed_once` at `:1253`).
  - Non-zero exit → all `unverified` (unchanged short-circuit).
  - **A `PatternLocator` never reaches `_observe_located`** (PRD R6) — e.g. monkeypatch/assert no
    `AttributeError`, or assert the message is the pattern-shaped one, not a JSON one.
  - Mixed pattern + table + JSON + flat claims resolve independently.
- **GREEN — implement:**
  - Add `_text_cache: dict[str, str | None] = {}` beside the other caches.
  - `_observe_pattern_located(loc)`:
    - `if loc.source is None: text = run_output` — **no path join, no filesystem access**. (The
      closure sees the retried `run_output` by late binding; state this in the docstring.)
    - else: the verbatim containment guard (`(repo_path / loc.source).resolve()` →
      `relative_to(repo_root)` → `ValueError` ⇒ `"locator 'from' {loc.source!r} escapes the
      repo"`), then cache lookup by `str(resolved)`; on a miss, `stat()` the file first — if
      `st_size > _MAX_MATCH_BYTES`, cache/return the oversize message **without reading**;
      else `read_text()` inside `try/except (OSError, ValueError)` → `None` on failure
      (`ValueError` covers `UnicodeDecodeError`, exactly as `_observe_located` documents at
      `:614-619`). `None` → `f"locator file {loc.source!r} is missing or unreadable"`.
    - `cell, reason = resolve_match(text, loc.pattern)`; `None` → `(None, f"locator pattern in
      {…} did not resolve: {reason}")` where `…` is `loc.source!r` or `"the run output"`.
    - `float(cell.strip())` in `try/except ValueError` → `(None, msg)`; `math.isnan/isinf` →
      `(None, msg)`; else `(value, "")`.
  - Dispatch head `:780-785` becomes an explicit chain:
    ```python
    if isinstance(claim.locator, TableLocator):
        observed, fail_msg = _observe_table_located(claim.locator)
    elif isinstance(claim.locator, PatternLocator):
        observed, fail_msg = _observe_pattern_located(claim.locator)
    else:
        observed, fail_msg = _observe_located(claim.locator)
    ```
- **Validation:** `uv run pytest tests/test_reproduce.py -q` green; full suite green.
- **Commit:** `feat(reproduce): route stdout/log pattern locators through run_reproduction (C8 slice 4)`

## Edge cases & error handling (all → UNVERIFIED, never DIVERGED, never a raise)

| Case | Outcome |
|---|---|
| 0 matches / N>1 matches | UNVERIFIED, count named |
| Capture group didn't participate | UNVERIFIED (would otherwise be `TypeError`) |
| Capture not float-parseable (`"NA"`) / non-finite | UNVERIFIED |
| `from` missing / directory / non-UTF-8 / unreadable | UNVERIFIED |
| `from` escapes the repo | CLI refuses pre-run; engine defensively UNVERIFIED, never read |
| Text or file > 8 MiB | UNVERIFIED naming the size; file never read |
| Uncompilable pattern | `ClaimsError` pre-run (nothing written); resolver also degrades defensively |
| Non-zero run exit | all claims UNVERIFIED before any locator runs (existing short-circuit) |
