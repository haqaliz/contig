### The Strelka2 VAF formulas (pin these in fixtures; cite Strelka2's AF definition)

- **SNV** (`*.somatic_snvs*`): FORMAT carries per-base tier counts `AU,CU,GU,TU`, each a
  `tier1,tier2` pair. With single-base `REF`=r, `ALT`=a (both ∈ {A,C,G,T}):
  `refCount = tier1({r}U)`, `altCount = tier1({a}U)`,
  `VAF = altCount / (refCount + altCount)` (skip if denominator 0).
- **Indel** (`*.somatic_indels*`): FORMAT carries `TAR` (ref) and `TIR` (alt), each
  `tier1,tier2`. `VAF = tier1(TIR) / (tier1(TAR) + tier1(TIR))` (skip if denominator 0).
- Read only the **`TUMOR`** sample column. Multiallelic (comma in ALT) excluded. SNV records
  whose REF/ALT aren't single A/C/G/T are skipped (indel file handles those via TAR/TIR).
- Omit-never-guess: any missing/malformed/non-numeric field, absent U/TAR/TIR key, or zero
  denominator → that record contributes **no** VAF.

## Phase 1 — Strelka2 tier-count VAF parser (pure, unit-tested)

**Goal:** a pure, stdlib, streaming parser that turns the Strelka2 SNV+indel VCFs into a
pooled list of per-record tumor VAFs and a pooled median. No verdict/gate coupling yet.

**Prerequisites:** none.

**Files:** create `src/contig/verification/strelka_vaf.py`; create
`tests/verification/test_strelka_vaf.py`.

**RED — write these tests first** (`tests/verification/test_strelka_vaf.py`, real files via
`tmp_path`, mirroring `test_somatic_plausibility.py`'s inline-VCF pattern):
1. `test_snv_vaf_exact` — a one-record SNV VCF (`REF=C, ALT=A`, FORMAT `…:AU:CU:GU:TU`, TUMOR
   field with known tier1 counts, e.g. `AU=6,7 CU=14,15 …`) → the record VAF equals
   `6/(14+6)=0.30`.
2. `test_indel_vaf_exact` — a one-record indel VCF (`TAR=18,20 TIR=2,3`) → `2/(18+2)=0.10`.
3. `test_pooled_median_exact` — SNV file (VAFs 0.30, 0.50) + indel file (VAF 0.10) → pooled
   median of [0.10,0.30,0.50] = 0.30.
4. `test_multiallelic_excluded` — a comma-ALT record contributes nothing.
5. `test_zero_denominator_and_malformed_omitted` — a record with `AU=0,0 CU=0,0` (its
   ref+alt bases) or a non-numeric field contributes nothing (not 0.0).
6. `test_snv_non_acgt_skipped` — a "SNV" record with a multi-base REF in the SNV file is
   skipped, never mis-indexed.
7. `test_no_tumor_column_returns_none` — a header with only `NORMAL` (no `TUMOR`) → the
   `tumor_found` signal is False and the VAF list is empty.

**GREEN — implement in `strelka_vaf.py`:**
1. `_open_text(path)` — gzip-transparent text open (mirror the sibling `_open_text`).
2. `_tumor_column_index(header_lines) -> int | None` — find the `#CHROM` line, split on
   tabs, return the index (≥9) of the column literally named `TUMOR`, else `None`. **No
   positional guess.**
3. `_tier1(field: str) -> int | None` — first comma-token as int; `None` on malformed.
4. `_snv_vaf(fmt_keys, sample_fields, ref, alt) -> float | None` — map base→`{base}U`, read
   tier1 of REF-base and ALT-base fields, guard both present + denominator>0.
5. `_indel_vaf(fmt_keys, sample_fields) -> float | None` — tier1 TIR / (tier1 TAR + tier1
   TIR), guarded.
6. `read_strelka_vafs(snv_path=None, indel_path=None) -> tuple[list[float], bool]` — stream
   each provided file; resolve tumor col at `#CHROM`; for data records build the
   `{key:index}` map from `cols[8]`, read `cols[tumor_idx]`, apply the SNV or indel deriver
   by which file it is; return the pooled VAF list and whether a `TUMOR` column was found in
   any read file. Skip a file whose tumor col is absent (contributes nothing).
7. `strelka_median_vaf(snv_path=None, indel_path=None) -> tuple[float | None, bool]` — median
   via `statistics.median` (None on empty), plus the `tumor_found` flag.

**Validation:** `uv run pytest tests/verification/test_strelka_vaf.py -q` all green.

**Commit:** `feat(verify): Strelka2 tier-count VAF parser (SNV AU/CU/GU/TU, indel TAR/TIR)`

- **Integration** (Phase 3): the `_discover_qc` gate emits both metrics / skips / UNVERIFIED,
  in `test_run_qc.py`.
- **Regression**: full `uv run pytest` stays green; no other assay's verdict changes.
- **No real tool**: all VCFs synthetic inline fixtures; no nf-core/sarek or samtools in CI.

## Environment & reproducibility

- `uv sync` (done). Stdlib only. Deterministic (median of parsed floats). No env vars, no
  external tools, no network. No reproduce/`launch.json` impact (read-time verdict metric).

## Edge cases & error handling

- No Strelka VCF → silent skip (structural QC owns missing outputs).
- No `TUMOR` column / zero derivable VAFs / non-unique layout → one honest UNVERIFIED.
- Multiallelic, non-ACGT SNV, zero denominator, malformed field → record omitted.
- SNV-only or indel-only present → pool what's there; both absent handled by the skip.
- Never FAIL, never change the exit code.
