# Implementation plan — `fail-floor` (2026-07-15)

Sources: [`../prd.md`](../prd.md), [`spec.md`](spec.md), `docs/planning/_card/understanding.md`.

**Shape:** existing codebase, no scaffolding. One data line + tests + a decision record. Strict
TDD (RED → GREEN → REFACTOR). Mirrors shipped commit `69b6385` (germline FAIL bands: 4 files,
2 source + 2 test, no CLI test).

## 1. Project setup checklist

Nothing to set up. No new directory, module, dependency, config, or env var. `uv sync` is already
done in this worktree. **Do not add a `conftest.py`** — the repo has none by design; each test file
defines local helpers.

## 2. Implementation phases

### Phase 1 — RED: the empty-call-set FAIL floor

- **Goal:** a failing test proving a 0-record somatic VCF FAILs. AC1, AC2, AC3, AC5.
- **Prerequisites:** none.
- **Files to modify:** `tests/verification/test_somatic_plausibility.py`

### Phase 4 — The decision record (the durable half)

- **Goal:** convert "FAIL deferred until calibrated" → "declined by design, here's the structural
  reason", in the three user-confirmed places. PRD M4, IS2, IS3.
- **Prerequisites:** Phases 2-3 green.
- **Files to modify:** `src/contig/verification/rule_pack.py`, `src/contig/verification/somatic_plausibility.py`, `docs/technical/CAPABILITY_ROADMAP.md`, `CHANGELOG.md`
- **Implementation steps:**
  1. **Pack docstrings.** Rewrite the WARN-cap headers so they read as a decision, not a delay:
     - `rule_pack.py:301-307` (somatic pack header) and `:309-317` (`median_vaf` comment) +
       `:326-339` (`strelka_median_vaf` comment): state that the VAF metrics are WARN-capped
       **by decision** — tumor VAF's expected value is a function of purity and clonality that the
       code never observes (no purity, ploidy, copy-number, or target type), so no band can
       separate "broken" from "legitimately low"; and `strelka_median_vaf` is arithmetically
       bounded to [0,1] by construction (`strelka_vaf.py:95-98,121-124`), making a `fail_above 1.0`
       provably dead code. **Do not add one.**
     - `rule_pack.py:280-284` (RNA-seq plausibility header) and `:375-378` (composition header):
       WARN-capped **by decision** — every metric has a legitimate protocol at the extreme
       (deep/high-input → 90%+ duplication; total-RNA/ribo-depletion → high rRNA; nuclear/FFPE/3'
       → intron-dominated; non-model annotation → high unassigned), so "extreme" and "unusual
       protocol" are the same number. Note additionally that `percent_duplication`/`percent_rRNA`
       have never resolved against a real nf-core/rnaseq MultiQC report.
     - `somatic_plausibility.py:6,227,230-234`: keep the existing "capped at WARN" language honest
       — the **count** can now FAIL on an empty call set; the **VAF** metrics remain WARN-capped by
       decision.
  2. **`CAPABILITY_ROADMAP.md`.** Update the C3 line (`:369`, `:910`) and the C4 line (`:528`,
     `:911`) plus the somatic VAF slice prose (`:552-556`): record the shipped floor, and change
     "FAIL severity … deferred until calibrated" to **declined by design** with the one-line
     reason. This is the file `/contig-next` ranks from — the wording must stop it being re-picked.
  3. **`CHANGELOG.md`.** New `### Changed` entry under `## [Unreleased]`: the shipped floor (what
     now FAILs and what deliberately still doesn't); the declined-by-design decision + reason; and
     **the signature caveat (PRD R2)** — re-verifying an affected old bundle recomputes `verdict`
     inside the signed canonical payload (`signing.py:63-64`), so its Ed25519 signature no longer
     matches; blast radius is only bundles whose verdict actually flips (empty somatic call sets,
     i.e. broken runs), and this is inherited unchanged from v0.35.0.
- **Validation:** `uv run pytest` (full suite) → green. Re-read each edited comment and confirm it
  states a **reason**, not a **deferral**.
- **Commit:** `docs(verify): decline somatic-VAF/RNA-seq FAIL bands by design, not deferral`

## 5. Environment & reproducibility

No env vars, no external tools, no pins. **Reproducibility impact:** no new model field, no
provenance change, no reproduce-bundle contract change. **One caveat, documented not fixed:**
`verdict` is a `@computed_field` serialized into the signed canonical payload
(`models.py:357-369`, `signing.py:63-64`), so this band change invalidates signatures on bundles
whose verdict flips. Bounded to empty-call-set somatic runs; inherited from v0.35.0; disclosed in
the CHANGELOG (Phase 4).

## 6. Edge cases & error handling

| Edge case | Expected behavior | Covered by |
|---|---|---|
| 0-record VCF | count `0` → **FAIL** (0 rides the band; `count` is always an int per `somatic_plausibility.py:133,155,233-234`) | AC1-AC3 |
| 0-record VCF, `median_vaf` | **UNVERIFIED** — never FAIL, never PASS | AC5 |
| 0-record VCF, `pon_applied` | **UNVERIFIED** (no GATK header); does not block the FAIL reduction | AC3 |
| 2 records | **WARN** (below warn floor, at/above fail floor) — unchanged | AC4 |
| Legitimate low-purity tumor (VAF 0.99 / extreme) | **WARN**, never FAIL | AC8 |
| Legitimate 0-call hotspot panel | **FAILs** — accepted (PRD R1); narrowest possible escalation, opt-in flag | PRD R1 |
| Re-verified old bundle whose verdict flips | signature mismatch | PRD R2, disclosed |

  ("an empty call set is a broken run"), never a biological or clinical claim.
