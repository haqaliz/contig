# Implementation Plan — methylseq-firing (aspect of assay-qc-verdict-fires)

**Date:** 2026-07-11
**PRD:** `../prd.md` · **Spec:** `./spec.md`
**Branch:** `feat/assay-qc-verdict-fires/aliz`
**Execution:** strict TDD (RED → GREEN → REFACTOR), one agent per phase, suite kept

This slice makes the `methylseq` biological verdict FIRE by parsing Bismark's own
on-disk report artifacts behind a dedicated `_discover_qc` gate — a near-exact
mirror of the shipped scrnaseq v0.21.0 slice. ampliseq/mag are out of scope.

---

## Key design reconciliation (read before Phase 2)

M6 (PRD) requires the new gate to be the **single authoritative source** for
methylseq biological metrics — no double-emit with the generic MultiQC pack path
(`runner.py:116-124`). Two facts constrain how:

1. scrnaseq (the precedent) **keeps** `SCRNASEQ_RULE_PACK` registered in
   `_RULE_PACKS` (`rule_pack.py:314`) *and* has a dedicated gate; an existing test
   asserts `rule_pack_for("scrnaseq") is SCRNASEQ_RULE_PACK` (`test_rule_pack.py:205`).
   Its generic path is inert only because the metrics never land in MultiQC.
2. An existing test asserts `rule_pack_for("methylseq") is METHYLSEQ_RULE_PACK`
   (`test_rule_pack.py:311`).

**Decision:** do NOT remove methylseq from `_RULE_PACKS` (keeps the precedent and
the existing contract/test). Instead, make the generic path **explicitly skip
methylseq** so the dedicated gate is the sole emitter. This satisfies M6
structurally (not just by inertness), breaks no existing test, and matches the
scrnaseq registration convention. Add a module-level constant in `runner.py`:

```python
# Assays whose biological metrics come from a dedicated on-disk gate below, NOT
# from MultiQC general-stats. The generic pack path skips them so a metric can
# never be emitted twice if a future MultiQC ever carried a matching slug.
_DEDICATED_METRIC_ASSAYS = {"methylseq"}
```

and change the generic guard to `if pack is not None and assay not in
_DEDICATED_METRIC_ASSAYS:`. (Scope note: we add only `methylseq` now; scrnaseq is
left exactly as-is to stay in scope.)

## 2. Implementation phases

### Phase 1 — Bismark report parsers (`methylseq_metrics.py`)

- **Goal:** pure, stdlib-only parsers that turn Bismark report text into
  `{canonical_slug: float}` for one sample. Canonical slugs MUST match
  `METHYLSEQ_RULE_PACK` (`rule_pack.py:131-153`): `percent_aligned`,
  `percent_duplication`, `percent_bs_conversion`.
- **Prerequisites:** none.
- **Files:**
  - CREATE `src/contig/verification/methylseq_metrics.py`
  - CREATE `tests/verification/test_methylseq_metrics.py`
- **Implementation steps:**
  1. Write failing unit tests first (RED) from realistic Bismark report snippets as
     inline strings written to `tmp_path` (mirror `test_scrnaseq_metrics.py`'s
     `_write` helper):
     - Alignment report (`*_PE_report.txt`) containing a
       `Mapping efficiency:\t78.90%` line → `{"percent_aligned": 78.9}`.
     - Deduplication report containing
       `duplicated alignments removed:\t123456 (12.34%)` → `{"percent_duplication": 12.34}`.
     - Splitting/conversion: a report WITH a recognizable conversion line (e.g.
       `bisulfite conversion rate:\t99.30%`, or a labeled control line) →
       `{"percent_bs_conversion": 99.3}`; a standard splitting report WITHOUT such a
       line → `{}` (omitted, NOT guessed).
     - Non-numeric / missing field → slug omitted.
     - Unrecognized file → `{}`.
  2. Implement (GREEN): define slug constants; `_to_float` (reuse the scrnaseq
     pattern — `None` on non-numeric); regex extractors:
     - `parse_bismark_alignment_report(path)` → `percent_aligned` from
       `Mapping efficiency:` (tab/space tolerant, strip trailing `%`).
     - `parse_bismark_dedup_report(path)` → `percent_duplication` from the
       `duplicated alignments removed:` line's parenthesized percent.
     - `parse_bismark_conversion_report(path)` → `percent_bs_conversion` ONLY when an
       explicit conversion/control rate line matches; else `{}`.
     - Each parser reads only the passed file; a non-matching file yields `{}`.
     - **Floor principle:** absent/non-numeric → omit the slug, never coerce to 0.
  3. REFACTOR: dedupe the regex/float helpers; module docstring stating the
     no-MultiQC rationale (mirror `scrnaseq_metrics.py:1-20`) and that field labels
     are pinned to the committed fixture.
- **Validation:** `uv run pytest tests/verification/test_methylseq_metrics.py -q`
  green.
- **Commit:** `test+feat(verify): Bismark report parsers for methylseq metrics [C3]`

### Phase 2 — Runner gate + locate/sample helpers + M6 skip-guard

- **Goal:** a dedicated `assay == "methylseq"` gate in `_discover_qc` that locates
  and merges Bismark reports per sample, evaluates `METHYLSEQ_RULE_PACK`, emits an
  explicit UNVERIFIED breadcrumb only when a located artifact yields zero metrics,
  and skips silently when no artifact exists. Plus the M6 generic-path skip.
- **Prerequisites:** Phase 1.
- **Files:**
  - MODIFY `src/contig/runner.py`
  - MODIFY `tests/verification/test_run_qc.py` (gate tests, mirror `:531-649`)
- **Implementation steps:**
  1. RED — add gate tests in `test_run_qc.py` using `_write_*` helpers that build
     the real nf-core/methylseq report layout under `tmp_path`, asserting on
     `_discover_qc(run_dir, assay="methylseq")`:
     - **A1** healthy alignment+dedup → non-UNVERIFIED `mapping_efficiency:<s>` +
       `duplication_rate:<s>`.
     - **A2** mapping efficiency `< 30.0` → FAIL `mapping_efficiency:<s>`.
     - **A3a** report present but zero usable metrics → exactly one
       `methylseq_qc:<s>` UNVERIFIED.
     - **A3b** no methylseq artifact at all → no methylseq metric result, no crash.
     - **A4** alignment-only sample (no dedup, no conversion) → PASS/WARN on mapping
       efficiency and **no** `methylseq_qc:<s>` UNVERIFIED.
     - **A5** conversion emitted only with a control line; standard report omits it
       (no `bisulfite_conversion:<s>` result), sample still evaluates the rest.
     - **A6** a methylseq run whose MultiQC carries a matching slug does NOT
       double-emit (build a `multiqc_data.json` with a `percent_aligned` general-stat
       and assert `mapping_efficiency:<s>` appears exactly once — from the gate).
     - **A7** gate not applied to another assay (`_discover_qc(run_dir,
       assay="rnaseq")` over the same dir emits no `methylseq_qc:*`).
  2. GREEN — in `runner.py`:
     - Add `_sample_from_bismark(path)` — derive the sample id by stripping known
       Bismark report suffixes from the filename (e.g.
       `<sample>_bismark_bt2_PE_report.txt`, `..._SE_report.txt`,
       `..deduplication_report.txt`, `.._splitting_report.txt`). Resolve exact
       suffixes against the Phase 3 fixture.
     - Add `_locate_methylseq_qc(run_dir) -> dict[str, dict[str, float]]` mirroring
       `_locate_scrnaseq_qc` (`:90-107`): rglob each report kind, route to its
       parser, and **merge** all report dicts for the same sample into one metric
       dict (M3: alignment + dedup + conversion combine; partial samples keep
       whatever parsed).
     - Add the gate after the scrnaseq block (`:247`), mirroring `:231-246` exactly
       but with `check=f"methylseq_qc:{sample}"` and `METHYLSEQ_RULE_PACK`.
     - Add the `_DEDICATED_METRIC_ASSAYS` constant and the generic-path skip (see the
       reconciliation section).
     - Add imports: `from contig.verification.methylseq_metrics import (…)` and add
       `METHYLSEQ_RULE_PACK` to the existing `rule_pack` import.
  3. REFACTOR — factor the "evaluate-or-explicit-UNVERIFIED" gate body if it cleanly
     generalizes with scrnaseq's, but do NOT change scrnaseq behavior; keep the diff
     surgical.
- **Validation:** `uv run pytest tests/verification/test_run_qc.py -q` green; full
  `uv run pytest -q` green (confirms no existing test — incl.
  `test_rule_pack_for_methylseq_returns_the_methylseq_pack` — regressed).

## 4. Testing strategy

- **Unit (Phase 1):** each parser in isolation — happy path, partial, non-numeric,
  unrecognized-file → `{}`, conversion present vs absent.
- **Gate (Phase 2):** `_discover_qc(..., assay="methylseq")` for A1-A7 above.
- **Integration (Phase 3):** parse the committed realistic fixture end-to-end.
- **Regression:** full `uv run pytest -q` after every phase; specifically confirm
  `test_rule_pack.py` methylseq tests still pass (registration contract preserved).
- No real nf-core/methylseq run in CI; no network; deterministic.

## 5. Environment & reproducibility

Stdlib-only, no new dep, no env var. Parsers read small local report files on the
user's compute (no raw-read egress). Deterministic (no randomness/time). The
committed fixture is the reproducible source of truth for the field labels.

## 6. Edge cases & error handling

- Located-but-zero-metrics → explicit `methylseq_qc:<sample>` UNVERIFIED (never
  silent). No artifact → silent skip (structural QC owns missing outputs).
- Partial reports (alignment only) → evaluate available checks, no false
  sample-wide UNVERIFIED (A4/M3).
- `percent_bs_conversion` absent without a control line → omitted, no result for
  that one check (honest silence at check level).
- Two report kinds for one sample → merged, no double-count.
- A future MultiQC carrying a matching slug → generic path skips methylseq, so no
  double-emit (M6).

## 7. Agent execution notes

- **Checkpoint after Phase 2:** confirm full suite green (esp. the registration
  test) before the fixture phase — that's where a wrong field label surfaces.
- **Likely iteration:** the exact Bismark report field labels and filename suffixes.
  The Phase 3 fixture is authoritative; if Phase 1's inline snippets disagree with
  the real fixture, the fixture wins and the parser regex is adjusted. The
  explicit-UNVERIFIED breadcrumb guarantees a wrong label fails loudly, not silently.
- **Ask before proceeding** only if the real Bismark report format cannot be
  determined confidently for a metric — prefer omitting that metric (→ UNVERIFIED)
  over guessing a label.
- **Do not** touch scrnaseq/rnaseq/variant paths, change slug names or bands, or
  start ampliseq/mag (separate slices).
