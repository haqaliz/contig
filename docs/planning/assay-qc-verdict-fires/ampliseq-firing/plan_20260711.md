# Implementation Plan — ampliseq-firing

**Date:** 2026-07-11 · **Spec:** `./spec.md` · **Template:** the methylseq plan
(`../methylseq-firing/plan_20260711.md`) — same seam, same discipline.
**Execution:** strict TDD, commits per phase, `uv run pytest` green after each.

The one real difference from methylseq: DADA2 artifacts are **multi-sample files**,
so parsers return `{sample: {slug: float}}` directly (not `{slug: float}` for one
sample).

## Phase 1 — DADA2 parsers (`ampliseq_metrics.py`)
- RED: unit tests from inline `overall_summary.tsv` + ASV-table strings.
- GREEN:
  - `parse_dada2_overall_summary(path) -> {sample: {input_reads, percent_retained}}`
    — header-driven TSV; `input_reads` = DADA2-input column; `percent_retained` =
    `nonchim / input * 100` (omit if input is 0/absent). Tolerate column-name
    variants case-insensitively.
  - `parse_asv_table(path) -> {sample: {asv_count}}` — rows=ASVs, columns=samples;
    per-sample count of non-zero cells.
  - Floor: absent/non-numeric omitted; unrecognized file → `{}`.
- Commit: `test+feat(verify): DADA2 stats parsers for ampliseq metrics [C3]`

## Phase 2 — Runner gate + M6
- RED: gate tests (B1–B7) in `test_run_qc.py`, mirroring the methylseq gate tests.
- GREEN in `runner.py`:
  - `_locate_ampliseq_qc(run_dir) -> {sample: {slug: float}}` — rglob
    `*overall_summary*.tsv` and `*ASV_table*` (the structural manifest's required
    glob), route to parsers, **merge by sample** (`setdefault(s, {}).update(...)`).
  - Gate `if assay == "ampliseq":` mirroring the methylseq gate exactly, with
    `check=f"ampliseq_qc:{sample}"`, `AMPLISEQ_RULE_PACK`.
  - Add `"ampliseq"` to `_DEDICATED_METRIC_ASSAYS`.
  - Imports: `ampliseq_metrics` parsers + `AMPLISEQ_RULE_PACK`.
- Validation: `uv run pytest -q` green (incl.
  `test_rule_pack_for_ampliseq_returns_the_ampliseq_pack`).
- Commit: `feat(verify): ampliseq QC gate fires from DADA2 stats; single-source [C3]`

## Phase 3 — Fixture + docstring + CHANGELOG
- Realistic `tests/fixtures/ampliseq/overall_summary.tsv` + an ASV table; one
  integration test parsing them.
- Tighten `AMPLISEQ_RULE_PACK` docstring (name the DADA2 source per metric; no
  slug/band change).
- `[Unreleased]` CHANGELOG line.
- Commit: `test+docs(verify): realistic DADA2 fixture + ampliseq firing changelog [C3]`

## Edge cases
Multi-sample file (B5); partial (summary only, B4) → asv_count absent, no false
UNVERIFIED; zero-usable-metrics sample → explicit `ampliseq_qc:<s>` UNVERIFIED (B3);
no double-emit (B6). Do not touch other assays.
