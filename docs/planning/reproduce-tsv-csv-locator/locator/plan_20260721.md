New symbols (all in `src/contig/verification/reproduce.py` unless noted):
- `TableLocator` (frozen dataclass): `source: str`, `column: str | int`,
  `row: int | dict[str, str]`, `delimiter: str` (resolved 1-char), `header: bool`.
- `Claim.locator: Locator | TableLocator | None`.
- `_resolve_delimiter(source: str, explicit: str | None) -> str | None` (pure; returns the
  1-char delimiter or `None` for an unknown extension with no explicit override — `None`
  signals a `ClaimsError` to the caller).
- `_read_table(path: Path, delimiter: str) -> list[list[str]] | None` (gzip-transparent
  stdlib read; `None` on any OS/decode/parse error — never raises).
- `resolve_cell(rows, column, row, header) -> tuple[str | None, str]` (pure; returns the
  raw cell string or `(None, reason)`; index-safe, never raises).
- `_observe_table_located(locator: TableLocator) -> tuple[float | None, str]` (engine
  reader, sibling of `_observe_located`; uses the table cache; parses the cell to a finite
  float or returns `(None, message)`).

## Phase 1 — Schema & validation (`TableLocator` + `load_claims`)

- **Goal:** a table-locator claim parses into a validated `TableLocator`; every malformed /
  contradictory shape raises `ClaimsError` (exit non-zero, nothing written).
- **Prereqs:** baseline green.
- **Files:** `reproduce.py` (dataclass + `load_claims` + `_resolve_delimiter`);
  `tests/test_reproduce.py` (extend the `load_claims` test cluster around `:200-246`).
- **RED — tests first** (`tests/test_reproduce.py`):
  1. `from` + `column`(str) + `row`(dict) → `Claim.locator` is a `TableLocator` with the
     fields; default `header is True`; delimiter inferred `"\t"` for `.tsv`, `","` for
     `.csv`.
  2. `from` + `column`(int) + `row`(int) + `header:false` → positional `TableLocator`.
  3. `.tsv.gz`/`.csv.gz` → delimiter inferred (strip `.gz` first).
  4. explicit `"delimiter": ";"` overrides extension.
  5. **Reject (ClaimsError):** `path` **and** `column`/`row` together; `column` without
     `row` (and vice-versa); `column` a float/empty-string/negative-int; `row` a
     negative-int / empty-object / multi-key object / object with non-string key or value;
     `delimiter` not a single char; `header` not a bool; table fields **without** `from`;
     `row`-object **+** `header:false`; `column`-str **+** `header:false`; unknown
     extension (`.txt`) **with no** `delimiter`.
  6. Existing JSON-locator + flat-claim tests still pass unchanged.
- **GREEN — implement:** add `TableLocator`; widen `Claim.locator` type; in `load_claims`
  branch after the existing `has_from` block — if `column`/`row` present, validate per the
  PRD M1 rules, resolve the delimiter via `_resolve_delimiter`, build a `TableLocator`. Keep
  JSON-locator path byte-identical. `_resolve_delimiter` lower-cases source, strips one
  trailing `.gz`, maps `.tsv`/`.tab`→`\t`, `.csv`→`,`, else returns `explicit` or `None`.
- **Validation:** `uv run pytest tests/test_reproduce.py -q` green; full suite green.
- **Commit:** `feat(reproduce): validate TSV/CSV table locators in load_claims (C8 slice 3)`

## Phase 2 — Pure cell reader (`_resolve_delimiter` done; `_read_table` + `resolve_cell`)

- **Goal:** given parsed rows + a resolved address, return the target cell string or an
  honest `(None, reason)`; read a table file gzip-transparently. Both crash-proof.
- **Prereqs:** Phase 1 merged (uses `TableLocator` types, but these fns are type-agnostic).
- **Files:** `reproduce.py` (`_read_table`, `resolve_cell`); **new**
  `tests/test_reproduce_tsv_locator.py` (mirror `tests/test_reproduce_locator.py`).
- **RED — tests first** (`tests/test_reproduce_tsv_locator.py`), pure, no engine:
  - `resolve_cell` header mode: column-by-name hit; column-by-int hit; row-by-index hit;
    row-by-key single match; **row-key 0 matches → (None, "…0 rows")**; **>1 matches →
    (None, "…N rows")**; column-name-absent → None; **duplicate header name → None**;
    column-int OOR → None; row-int OOR → None (message names the data-row count);
    **ragged row shorter than column index → None (no IndexError)**; empty rows → None;
    header-only (0 data rows) → None.
  - `resolve_cell` headerless mode: integer row/col hit; OOR → None.
  - key compare is exact on `.strip()`ed cell (a trailing-space cell still matches; a
    different string does not).
  - `_read_table`: reads `.tsv` (tab) and `.csv` (comma) fixtures written to `tmp_path`;
    reads a **gzip** `.tsv.gz` fixture (write with stdlib `gzip`); a **directory** path →
    None; a **non-utf8** file → None; a missing file → None. Never raises.
  - A `test_resolve_cell_never_raises_on_wild_inputs` fuzz-style case (ragged/empty/huge
    indices/weird row objects), mirroring `resolve_pointer`'s never-raises test.
- **GREEN — implement:** `_read_table` opens via `gzip.open(path, "rt", encoding="utf-8",
  newline="")` when the name ends `.gz` else `open(..., newline="")`, feeds `csv.reader(f,
  delimiter=delimiter)`, returns `list(reader)`; wraps everything in `try/except (OSError,
  UnicodeDecodeError, csv.Error)` → `None`. `resolve_cell` splits header vs headerless,
  resolves column index (name→lookup with duplicate guard; int→bounds check), resolves the
  data-row list, resolves the row (int index / single key match), then index-safe cell
  fetch (`col_idx < len(row)` guard). Returns `(cell_str, "")` or `(None, reason)`.
- **Validation:** `uv run pytest tests/test_reproduce_tsv_locator.py -q` green; full green.
- **Commit:** `feat(reproduce): pure gzip-aware TSV/CSV cell reader + resolver (C8 slice 3)`

## Phase 3 — Engine branch (`run_reproduction` dispatch + table cache)

- **Goal:** a located **table** claim on a completed run binds its observed value and
  classifies; a table `from` file is parsed at most once; failures → UNVERIFIED.
- **Prereqs:** Phases 1–2.
- **Files:** `reproduce.py` (`_observe_table_located` + a `_table_cache` + isinstance
  dispatch at the per-claim branch `:457-544`); `tests/test_reproduce.py`.
- **RED — tests first** (`tests/test_reproduce.py`, via `run_reproduction` with a scripted
  executor + on-disk fixture tables in `tmp_path`):
  - named table claim matching → `reproduced`; drifted → `diverged` (message names
    observed-vs-stated + delta); near → `within_tolerance`.
  - positional/headerless claim → resolves.
  - missing file / unresolved column / row-key 0-or-many / ragged / **unparseable cell
    ("NA") → unverified** / **non-finite ("inf") → unverified**; never `diverged`.
  - **numeric-string cell ("30.4") → the observed value** (the deliberate divergence from
    the JSON rule — a cell is a string; this must classify, not go UNVERIFIED).
  - two claims targeting different cells of the **same** file → file parsed once (assert via
    an executor/read counter or a spy on `_read_table`).
  - mixed table + JSON-locator + flat claims resolve independently.
  - **escaping `from` reaching the engine directly → unverified, file never read**
    (defensive; mirrors `reproduce.py:321-325`).
  - non-zero exit → all unverified (unchanged).
- **GREEN — implement:** at the branch, `if isinstance(claim.locator, TableLocator):
  observed, msg = _observe_table_located(...)`; else the existing JSON `_observe_located`;
  else flat. `_observe_table_located` does the engine containment guard
  (`resolved.relative_to(repo_root)` → "escapes the repo" → UNVERIFIED, no read), pulls
  parsed rows from `_table_cache` (keyed by resolved abs path; populated via `_read_table`
  once), calls `resolve_cell`, then applies the shared finite-float parse (`float(cell)` in
  try/except → `(None, msg)` on `ValueError`; nan/inf → `(None, msg)`). Reuse the exact
  UNVERIFIED-plumbing pattern of the JSON branch.
- **Validation:** `uv run pytest tests/test_reproduce.py -q` green; full suite green.
- **Commit:** `feat(reproduce): route TSV/CSV table locators through run_reproduction (C8 slice 3)`

## Edge cases & error handling (all → UNVERIFIED unless noted)

Missing/dir/non-utf8/unparseable file; unknown extension **without** delimiter (→
`ClaimsError` at load, **pre-run**); mixed `path`+table or `row`-object+`header:false` (→
`ClaimsError`); column name absent / **duplicate** / index OOR; row index OOR / key 0-match
/ key >1-match; ragged/empty/header-only table; empty, `"NA"`, `"1,024"`, `"1.5%"`, or
non-finite cell. Never raises; never `DIVERGED` on a resolution failure.
