## Design decision (the seam)

Keep `apply_patch` (`self_heal.py:416-462`) **pure** and keep the ceiling-clamp +
never-shrink math in that one place. Add an optional keyword `observed_target_gb:
int | None = None`; in the memory branch (450-455), when it is not `None`, use it as
`bumped` instead of `int(current * mult)`. The existing `capped = min(bumped, ceiling)`
and `final = max(capped, int(current))` then run unchanged, so **every safety invariant is
preserved automatically**. Default `None` ⇒ all existing callers behave exactly as today.

The sized target is computed at the OOM safe-path site (`self_heal.py:974`) — where
`run_dir` (737) and `exc.record.events` (771) are in scope — by a new **pure helper** in a
new module `src/contig/resource_sizing.py` (isolated, unit-testable without a run).

**Join key (confirmed safe):** `TaskEvent` and `TaskResource` both set
`process = name or ""`, `name = name` from the same trace `name` column
(`events.py:91/95` and `127/131`); in the heal loop both come from the same `trace.txt`.
So joining OOM'd events to resource rows on `process`/`name` is exact.

## 2. Implementation Phases

### Phase 1 — Pure sizing helper (`resource_sizing.py`)

- **Goal:** `(events, resource_usage, factor) → PeakSizing` implementing the full ladder +
  binary-GB formula, with zero I/O.
- **Prerequisites:** none.
- **Files:** create `src/contig/resource_sizing.py`, `tests/test_resource_sizing.py`.
- **Steps:**
  1. Define `PEAK_RSS_SAFETY_FACTOR = 1.5` (module constant, code-overridable).
  2. Define a small result type `PeakSizing` (dataclass or `NamedTuple`):
     `target_gb: int | None`, `tier: str` (`"oom_task"｜"sibling"｜"unavailable"`),
     `observed_peak_mb: float | None`.
  3. `def peak_informed_memory_gb(events: list[TaskEvent], usage: list[TaskResource], *,
     factor: float = PEAK_RSS_SAFETY_FACTOR) -> PeakSizing:`
     - `oom_keys = {(e.process, e.name) for e in events if e.exit == 137}`;
       `oom_procs = {e.process for e in events if e.exit == 137}`.
     - **tier a:** `exact = [r.peak_rss_mb for r in usage if (r.process, r.name) in
       oom_keys and r.peak_rss_mb > 0]`; if `exact` → `peak = max(exact)`, tier
       `"oom_task"`.
     - **tier b:** else `sib = [r.peak_rss_mb for r in usage if r.process in oom_procs
       and r.peak_rss_mb > 0]`; if `sib` → `peak = max(sib)`, tier `"sibling"`.
     - **tier c:** else `return PeakSizing(None, "unavailable", None)`.
     - `target_gb = math.ceil(peak / 1024 * factor)` (binary GB), return
       `PeakSizing(target_gb, tier, peak)`.
  4. Treat `peak_rss_mb == 0.0` / absent as unknown (the `> 0` guards already do this).
- **Validation:** `uv run pytest tests/test_resource_sizing.py`. Tests (write FIRST, RED):
  - tier a: one `exit==137` event + matching row peak 7800 MB, factor 1.5 →
    `target_gb == ceil(7800/1024*1.5) == 12`, tier `oom_task`.
  - multi-task OOM: two `exit==137` rows (5 GB, 9 GB) → sizes to the 9 GB one.
  - tier b: OOM'd row peak `0.0` (dash) but a same-`process` sibling row peak 6144 MB →
    tier `sibling`, sizes to sibling.
  - tier c #1: OOM'd row peak `0.0`, no positive sibling → `(None, "unavailable", None)`.
  - tier c #2: **no `exit==137` event at all** (OOM detected by log only) → unavailable.
  - sub-current sanity: peak 800 MB → `target_gb == 2` (helper returns 2; clamping to
    `current` happens later in apply_patch, NOT here).
- **Commit:** `feat(self-heal): pure peak-RSS memory sizing helper (C2)`

### Phase 2 — `apply_patch` accepts an observed target

- **Goal:** memory branch uses `observed_target_gb` when supplied; else unchanged.
- **Prerequisites:** Phase 1 merged (for the constant only; helper not imported here).
- **Files:** `src/contig/self_heal.py` (`apply_patch`, 416-462); tests in
  `tests/test_self_heal.py` (or a focused `tests/test_apply_patch.py`).
- **Steps:**
  1. Add `observed_target_gb: int | None = None` to `apply_patch`'s keyword-only args
     (after `ceiling`).
  2. In the memory branch (450-455): `bumped = observed_target_gb if observed_target_gb
     is not None else int(current * int(mult["memory"]))`. Leave `capped`/`final`
     (ceiling + never-shrink) untouched. The `time` branch is unchanged.
  3. Update the docstring (427-431) to note the observed-target override.
- **Validation:** `uv run pytest` — tests (RED first):
  - with `observed_target_gb=12`, current 8, ceiling 128 → `memory == "12.GB"`.
  - with `observed_target_gb=None` → unchanged blind `×2` (current 8 → `"16.GB"`).
  - never-shrink: `observed_target_gb=2`, current 8 → `"8.GB"`.
  - ceiling clamp: `observed_target_gb=999`, ceiling 128 → `"128.GB"`.
- **Commit:** `feat(self-heal): apply_patch honors an observed memory target (C2)`

### Phase 3 — Wire at the OOM safe-path site + telemetry

- **Goal:** at `self_heal.py:974`, compute sizing for OOM failures and feed it into
  `apply_patch`; record the tier + observed peak into `RepairStep.detail`.
- **Prerequisites:** Phases 1 & 2 merged.
- **Files:** `src/contig/self_heal.py` (import + the block 965-980); integration tests in
  `tests/test_self_heal.py`.
- **Steps:**
  1. Import `peak_informed_memory_gb`, `PeakSizing` from `contig.resource_sizing`.
  2. Just before line 974, gated on `diagnosis.failure_class == "oom"`:
     - `usage = parse_resource_usage_file(run_dir / "trace.txt") if (run_dir /
       "trace.txt").exists() else []` (reuse the already-imported parser, `self_heal.py:27`).
     - `sizing = peak_informed_memory_gb(exc.record.events, usage)`.
     - else (`time_limit`) `sizing = PeakSizing(None, "n/a", None)`.
  3. `current_target, current_params = apply_patch(current_target, safe, current_params,
     ceiling=resource_ceiling, observed_target_gb=sizing.target_gb)`.
  4. Build `detail`: when `sizing.target_gb is not None`,
     `f"scaled memory to ~{sizing.target_gb} GB from observed peak
     {sizing.observed_peak_mb:.0f} MB (×{PEAK_RSS_SAFETY_FACTOR}, {sizing.tier})"`; else
     for oom, `"no usable observed peak; blind ×2 fallback (unavailable)"`; for time_limit,
     leave `detail=None` (unchanged behavior).
  5. Pass `detail=detail` into the `RepairStep(... outcome="patched_and_retried")` at
     975-979.
- **Validation:** `uv run pytest tests/test_self_heal.py` — integration tests (RED first),
  extending the `test_self_heal_populates_resource_usage_from_trace` fixture pattern
  (`tests/test_self_heal.py:105-125`):
  - **Sized heal:** fake executor fails attempt 1 with an `exit==137` STAR event AND writes
    a partial `trace.txt` whose STAR row has a high `peak_rss` (e.g. 60 GB); attempt 2
    succeeds. Assert the retried target's memory equals the sized value (`ceil(60GB*1.5)` GB
    clamped to ceiling), **not** `×2`; assert the run reaches its verdict; assert the
    `patched_and_retried` `RepairStep.detail` contains `"observed peak"` and `"oom_task"`.
  - **Sibling rescue:** OOM'd STAR row `peak_rss = -`, a completed STAR sibling row carries
    a peak → `detail` names tier `sibling`.
  - **Blind regression guard:** trace with no usable peak (or absent trace) → memory is the
    old blind `×2` and `detail` says `unavailable`. This proves no regression.
  - **time_limit untouched:** a `time_limit` heal still scales time `×2`, `detail=None`.
- **Commit:** `feat(self-heal): size OOM retries to observed peak RSS with blind fallback (C2)`

## 6. Edge Cases & Error Handling

- OOM'd row `peak_rss = -`/`0` → treated as unknown (tier b, then blind). Never "0 MB".
- OOM detected via log only (no `exit==137` event) → `unavailable` → blind. Honest.
- Multiple tasks OOM → size to the **max** peak among them (global `resourceLimits`).
- Sized target < current request → never-shrink holds `current` (expected, tested).
- Sized target > ceiling → clamped; if already at ceiling, `_resource_ceiling_block`
  (965) still gives up with `gave_up_at_ceiling` before this code runs — unchanged.
- Missing/empty `trace.txt` at heal time → `usage = []` → `unavailable` → blind.
- Snakemake (`stats.json`, no `trace.txt`) → `usage = []` → blind. No regression.
