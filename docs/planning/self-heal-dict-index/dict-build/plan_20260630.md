## Reference strings to PIN (critique #1) — do this BEFORE writing tests

GATK4's missing-sequence-dictionary USER ERROR, as emitted under nf-core/sarek:

```
A USER ERROR has occurred: Fasta dict file /path/to/genome.dict for reference /path/to/genome.fasta does not exist. Please build it using e.g. picard CreateSequenceDictionary or samtools dict.
```

- **Wording:** `does not exist` — **NOT** in the current notfound tuple
  (`"not found","missing","no such file"` at `detect.py:160`). This is *why* a targeted
  branch is needed, not just adding `.dict` to `detect.py:168`.
- **Path shape:** sarek passes a **local absolute path**, so the primary fixture uses
  `/path/to/genome.dict` (absolute). The deriver MUST therefore resolve candidates
  relative to that path's own parent (`/path/to/`), not `run_dir` (critique #3).
- **`file://` wrinkle (critique #1):** some GATK builds print `file:///path/to/genome.dict`.
  The token regex would capture the URI verbatim, and `Path("file:///…")` is wrong. The
  deriver must **strip a leading `file://`** (and `file:`) scheme before Path math. Cover
  with a dedicated unit test; keep the primary corpus seed a plain path.
- **Picard variant** (defensive, should-have): `picard CreateSequenceDictionary` /
  "Could not read sequence dictionary" — the branch keys on a `.dict` token + an absence
  phrase, so it tolerates wording; do not hard-code GATK-only text.

## Phase 3 — Generalize the build table + `.dict` source deriver

- **Goal:** the `{ext: (deriver, argv_fn)}` table; `.dict` resolves its source FASTA;
  the 4 existing kinds are byte-identical through the new path (critique #2).
- **Files:** `src/contig/self_heal.py:75-101`; `tests/test_self_heal.py`.
- **Steps (RED→GREEN):**
  1. RED (regression guard, critique #2): keep/adapt the existing per-kind
     `_index_build_command` argv tests (`:1217-1257`) to the new signature
     `_index_build_command(index_path, ext, run_dir)` and assert the SAME argv for
     `.fai/.bai/.tbi/.csi` (e.g. `("reference.fasta.fai", ".fai", tmp_path)` →
     `["samtools","faidx","reference.fasta"]`). These must stay green across the refactor.
  2. RED (`.dict` deriver unit tests): a new `_resolve_dict_source` (or the deriver
     entry) given `/dir/genome.dict` with `/dir/genome.fasta` present on disk → returns
     `/dir/genome.fasta`; candidate **priority** `.fasta` > `.fa` > `.fasta.gz` >
     `.fa.gz`; **absolute-safe** (critique #3) — probes `/dir/` (the `.dict`'s own
     parent), not `run_dir`; **relative** `genome.dict` with `run_dir/genome.fa` present →
     returns `genome.fa`; **`file://`** stripping (critique #1); **none present** →
     `None`.
  3. RED (`.dict` argv): `_index_build_command("/dir/genome.dict", ".dict", run_dir)` with
     `/dir/genome.fasta` present → `["samtools","dict","-o","/dir/genome.dict",
     "/dir/genome.fasta"]` (output = the missing `.dict` path itself; input = resolved
     FASTA). With no FASTA present → returns `None` (signals unresolvable).
  4. GREEN: introduce a deriver type and refactor the table:
     ```python
     SourceDeriver = Callable[[str, str, Path], str | None]  # (index_path, ext, run_dir)

     def _strip_suffix(index_path: str, ext: str, run_dir: Path) -> str:
         return index_path.removesuffix(ext)  # pure; ignores run_dir

     _DICT_FASTA_EXTS = (".fasta", ".fa", ".fasta.gz", ".fa.gz")

     def _resolve_dict_source(index_path: str, ext: str, run_dir: Path) -> str | None:
         raw = index_path.removeprefix("file://")  # tolerate GATK URI
         p = Path(raw)
         base_parent, stem = p.parent, p.name[: -len(ext)]  # ext == ".dict"
         for cand_ext in _DICT_FASTA_EXTS:
             cand = base_parent / f"{stem}{cand_ext}"
             probe = cand if cand.is_absolute() else Path(run_dir) / cand
             if probe.exists():
                 return str(cand)
         return None

     _INDEX_BUILD: dict[str, tuple[SourceDeriver, Callable[[str, str], list[str]]]] = {
         ".fai":  (_strip_suffix, lambda src, idx: ["samtools", "faidx", src]),
         ".bai":  (_strip_suffix, lambda src, idx: ["samtools", "index", src]),
         ".tbi":  (_strip_suffix, lambda src, idx: ["tabix", "-p", "vcf", src]),
         ".csi":  (_strip_suffix, lambda src, idx: ["bcftools", "index", src]),
         ".dict": (_resolve_dict_source,
                   lambda src, idx: ["samtools", "dict", "-o", idx, src]),
     }

     def _index_build_command(index_path: str, ext: str, run_dir: Path) -> list[str] | None:
         entry = _INDEX_BUILD.get(ext)
         if entry is None:
             raise ValueError(f"unsupported index extension: {ext}")
         deriver, argv_fn = entry
         source = deriver(index_path, ext, run_dir)
         if source is None:
             return None
         return argv_fn(source, index_path)
     ```
     Note the argv lambdas take `(src, idx)`; suffix-strip kinds ignore `idx`.
- **Validation:** `uv run pytest tests/test_self_heal.py -k "build_command or resolve"`
  green; the 4 existing-kind argv assertions green; full suite green.
- **Commit:** `feat(self-heal): generalize the index-build table and resolve a .dict source FASTA`

## Edge cases & error handling

- GATK `file://` URI in the path → deriver strips the scheme (P3 test).
- Absolute `.dict` path → probe its own parent dir, not `run_dir` (P3 test).
- No FASTA companion on disk → `index_unresolvable`, never guess (P4 test).
- `samtools dict` rc≠0 → `index_build_failed` with the path (P4 test).
- Wrong-reference masquerade (build ok, run still fails) → one build then honest give-up
  (P5 test).
- Wrong-reference / contig-mismatch line (no absence phrase) → NOT classified as
  `missing_index` (P1 test) — stays the deferred reference-mismatch repair's territory.
