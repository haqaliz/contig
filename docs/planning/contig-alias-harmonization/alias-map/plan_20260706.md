## Architectural decision (locks the PRD's M1/M3/M9 into one mechanism)

Replace the prefix-only transform with an **equivalence-group rename map** driven by a
**lookup against the actual FASTA contig set**:

- For each GTF contig `g`, the **candidates** for what it might mean are
  `{g} ∪ prefix_variants(g) ∪ alias_group(g)`, where `prefix_variants` add/strip `chr` and
  `alias_group` comes from the alias table.
- If exactly one candidate exists in the FASTA set `F`, rename `g → that name`. This makes
  the mito rule FASTA-driven (M9): `chrM` wins if the FASTA has `chrM`, `MT` if it has `MT`.
- **Rename map** = only the entries where the chosen name differs from `g`.
- **Unmatched** = GTF contigs with no candidate in `F` (surfaced by M8).
- **Trigger:** the map is non-empty **and** applying it strictly increases `|F ∩ GTF|`.
- **Refuse (`None`):** after applying the map the two sets are still **disjoint**
  (genuine wrong-assembly), or the map is empty (nothing to do / already consistent).

The prefix rule is thereby just one candidate source; mito + scaffolds are alias-group
sources. This subsumes v0.9.0 behavior (pure-prefix cases map identically) and adds the
residual + pure-alias + scaffold cases.

### Phase 1 — Alias equivalence table (data + loader)

- **Goal:** a data-driven `alias_group(name) -> frozenset[str]` returning all cross-convention
  spellings of a contig; mito universal in code, scaffolds in a bundled TSV.
- **Prerequisites:** none (new files).
- **Files:** create `src/contig/contig_aliases.py`, `src/contig/data/contig_aliases.tsv`,
  `tests/test_contig_aliases.py`.
- **Steps:**
  1. RED: `tests/test_contig_aliases.py` — `alias_group("MT")` ⊇ `{"MT","M"}`;
     `alias_group("M")` ⊇ `{"MT","M"}`; a seeded scaffold `alias_group("GL000191.1")` ⊇

     the TSV loads and every row forms a bidirectional group; loader tolerant of blank/`#`
     lines.
  2. GREEN: `contig_aliases.py` — a module-level `_MITO = frozenset({"M","MT"})` constant;

     handful of **real** GRCh38 chromAlias scaffold pairs (e.g.
     `GL000191.1<TAB>chrUn_GL000191v1`, `KI270711.1<TAB>chr1_KI270711v1`). Keep it small —
     completeness is not a test gate (PRD S1). Add a top comment documenting the source
     (UCSC `hg38.chromAlias.txt`) and how to extend.

### Phase 2 — General rename-map planner

- **Goal:** rewrite `plan_harmonization` to produce a rename map + unmatched set + direction
  label, per the architectural decision.
- **Prerequisites:** Phase 1.
- **Files:** modify `src/contig/reference_harmonize.py`, `tests/test_reference_harmonize.py`.
- **Steps:**
  1. RED: extend `TestPlanHarmonization` — add: UCSC/Ensembl full case → map has `MT:chrM`
     (+ `1:chr1`,`2:chr2`), `unmatched==()`; residual `{chr1,chr2,chrM}` vs
     `{chr1,chr2,MT}` → map `{MT:chrM}`; pure-alias both-`chr` `{chr1,chrM}` vs `{chr1,chrMT}`
     → `{chrMT:chrM}`; a tabled-scaffold case harmonizes; an **un-tabled** GTF contig lands
     in `.unmatched`; wrong-assembly (post-map disjoint) → `None`; full-overlap → `None`;
     strict-subset GTF → `None` (unchanged). Keep every existing assertion, adapting to the
     new return shape.
  2. GREEN: extend the frozen `HarmonizationPlan` to
     `{rename_map: dict[str,str], direction: str, unmatched: tuple[str,...], fasta_sample,
     gtf_sample}`. Implement `plan_harmonization(fasta_path, gtf_path)`:
     - parse `F = fasta_contigs(...)`, `G = gtf_contigs(...)` (reuse `reference_check`
       parsers — gzip-transparent); if either empty → `None`.
     - `prefix_variants(g)` = `{g, "chr"+g, g[3:] if g.startswith("chr")}` minus empties.
     - for each `g in G`: `cands = (prefix_variants(g) ∪ ⋃_{a∈alias_group(g)} prefix_variants(a)) ∩ F`;
       if `cands`: pick deterministically (prefer exact `g` if in `F`, else sorted()[0]);
       if chosen != g record `rename_map[g]=chosen`; else if no cands → add to `unmatched`.
     - compute `overlap_before=|F∩G|`, `overlap_after=|F ∩ {rename_map.get(g,g) for g in G}|`.
     - if `rename_map` empty **or** `overlap_after <= overlap_before` → `None`.
     - if `F ∩ {mapped}` still empty (disjoint) → `None` (redundant with above but explicit).
     - derive `direction`: `"add_chr"` if every rename is a pure `chr`-add, `"strip_chr"` if
       every pure strip, else `"alias"` (mixed prefix+alias). Preserves the v0.9.0 labels for
       the pure-prefix cases the existing tests assert.
  3. REFACTOR: keep `_all_chr_prefixed`, `_sample` reuse; drop the old `transformed` branch.
- **Validation:** `uv run pytest tests/test_reference_harmonize.py`.
- **Commit:** `feat(reference): rewrite plan_harmonization as a FASTA-driven rename map`

## 6. Edge Cases & Error Handling

- **Full overlap / already-consistent** → `plan_harmonization` returns `None` (no-op, run
  proceeds normally). Existing `test_full_overlap_returns_none` guards this.
- **Strict-subset GTF** (partial annotation) → shares contigs → not disjoint → `None` (no
  harmonization, proceeds). Unchanged.
- **Hybrid mito FASTA** (`chrMT`) → FASTA-lookup (M9) resolves GTF `MT`→`chrMT` if that's what
  the FASTA has; convention fallback only if no mito in FASTA.
- **Ambiguous candidate** (both `chrX` and `X` somehow in FASTA) → deterministic pick (prefer
  exact `g`, else sorted first); document; realistically impossible in a single-convention
  FASTA.
- **Un-tabled contig after harmonization** → left as-is and **enumerated in the breadcrumb**
  (M8) — the run proceeds but the honesty signal fires. This is the deliberate
  "don't relocate the silent failure" behavior.
- **Genuine wrong-assembly** → post-map disjoint → `None` → refuse (exit 1) unless
  `--allow-reference-mismatch`.
- **Empty/unparseable FASTA or GTF** → `None` (never a false refusal), matching
  `reference_check` behavior.
