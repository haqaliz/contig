## Phase 1 — Generalize the parser (path + extension)

- **Goal:** `_parse_missing_index(diagnosis) -> tuple[str, str] | None` returns the first
  missing-index token and its extension, for `.fai|.bai|.tbi|.csi`.
- **Prerequisites:** none.
- **Files:** `src/contig/self_heal.py`, `tests/test_self_heal.py`.
- **Steps:**
  1. **RED baseline:** add `test_parse_missing_index_finds_bai` / `_tbi` / `_csi`
     asserting `_parse_missing_index(d) == ("<path>", ".bai"/".tbi"/".csi")` for canonical
     tool lines (see Edge Cases for the lines). These fail today (no such function / `.fai`
     regex misses).
  2. Replace `_FAI_TOKEN_RE` with `_INDEX_TOKEN_RE` matching any supported extension,
     longest-extension-wins within a line, same end-boundary lookahead
     (`(?=\s|$|[:,;])`). Suggested: alternation ordered longest-first, or a single
     `\S+\.(fai|bai|tbi|csi)` with the boundary lookahead.
  3. Implement `_parse_missing_index`: scan `diagnosis.evidence` in order, return the first
     match as `(token, "." + ext)`, else `None`.
  4. **Rename + update existing tests:** rename `_parse_missing_fai`→`_parse_missing_index`
     everywhere; update `test_self_heal.py:937-1033` to the new name and `(path, ext)`
     return (e.g. `== ("reference.fasta.fai", ".fai")`); keep the boundary/colon/backup
     cases. Update the `None` test.
  5. Add AC5 determinism test: a line naming both `aln.bam` and `aln.bam.bai` →
     `("aln.bam.bai", ".bai")`.
- **Validation:** `uv run pytest tests/test_self_heal.py -k "parse_missing_index"` green;
  full `uv run pytest tests/test_self_heal.py` green.
- **Commit:** `feat(self-heal): parse any single-file missing-index path + extension`

## Phase 2 — Table-driven build command

- **Goal:** `_index_build_command(path, ext) -> list[str]` returns the per-kind argv.
- **Prerequisites:** Phase 1 (uses the parsed ext).
- **Files:** `src/contig/self_heal.py`, `tests/test_self_heal.py`.
- **Steps:**
  1. **RED:** add `test_index_build_command_*` asserting AC2 argv for all four kinds,
     including suffix stripping (`aln.bam.bai`→`aln.bam`, `calls.vcf.gz.tbi`→`calls.vcf.gz`)
     and an absolute-path case.

     `.tbi`→`["tabix","-p","vcf", strip(".tbi")]`; `.csi`→`["bcftools","index",
     strip(".csi")]`. Strip exactly the trailing index suffix.
  3. Rename/replace `_fai_build_command`; update `test_self_heal.py:981-1000` to call
     `_index_build_command("reference.fasta.fai", ".fai")`.
- **Validation:** `uv run pytest tests/test_self_heal.py -k "build_command"` green; full
  file green.
- **Commit:** `feat(self-heal): table-driven index-build command for .fai/.bai/.tbi/.csi`

## Edge cases & canonical log lines (for tests + corpus)

- `.bai` (samtools): `[E::idx_find_and_load] Could not retrieve index file for 'aln.bam'`
  — but this names `aln.bam`, not `aln.bam.bai`. For a token-bearing line use:
  `samtools index: failed to open "aln.bam.bai": No such file or directory`.
- `.tbi` (tabix): `[E::idx_load] Could not load the index calls.vcf.gz.tbi: No such file or directory`.
- `.csi` (bcftools): `Failed to open calls.vcf.gz.csi: No such file or directory`.
- Determinism (AC5): `... aln.bam.bai missing for aln.bam ...` → `.bai` wins.
- Bare-sibling (`x.bai` without `.bam`) and BAM-`.csi`: out of scope; the derived source
  won't exist → the build fails honestly (`index_build_failed`), never a false pass.
