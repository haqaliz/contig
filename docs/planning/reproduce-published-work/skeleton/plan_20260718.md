### Phase 2 тАФ Reproduce core (`verification/reproduce.py`)

- **Goal:** load+validate claims, run the repo script via an injected executor, bind `results.json`, compare, classify тАФ the whole verdict engine, pure/seam-driven.
- **Prerequisites:** Phase 1.
- **Files:** new `src/contig/verification/reproduce.py`; new `tests/test_reproduce.py`.
- **Steps (TDD тАФ write the boundary cases RED first, they are the risk):**
  1. RED `load_claims(path) -> list[Claim]`: valid JSON list `{id, value, tolerance?}` (default tol `0.1`); **reject** (raise a typed error) malformed JSON, non-list, missing `id`/`value`, non-numeric `value`, duplicate `id`, `tolerance <= 0`. (AC 6.)
  2. RED `classify(claimed, observed, tolerance) -> (ClaimStatus, delta|None, message)`:
     - `observed is None` or non-finite (`math.isnan/isinf`) claimed/observed тЖТ `unverified`, delta `None`. (AC 4.)
     - else `delta = _relative_delta(observed, claimed)` (reuse benchmark); `abs(observed-claimed) <= 1e-9` тЖТ `reproduced`; elif `delta <= tolerance` тЖТ `within_tolerance`; else `diverged`. (AC 1тАУ3.)
     - zero-claim: claimed `0` & observed `0` тЖТ `reproduced`; claimed `0` & observed nonzero тЖТ `_relative_delta` returns `abs(observed)` (absolute), classified vs tolerance тАФ assert the documented behavior. (AC 7.)
     - message names observed-vs-stated + delta for `within_tolerance`/`diverged`.
  3. RED `run_reproduction(repo, run_command, claims, *, executor, results_path="results.json", created_at, reproduce_id) -> ReproduceRecord`:
     - runs `executor(shlex.split(run_command), repo)`; **non-zero exit тЖТ every claim `unverified`**, `exit_code` recorded, no comparison attempted (AC 5);
     - zero exit тЖТ read `repo/results_path` JSON `{id: number}`; missing file тЖТ all claims `unverified`; per claim bind observed (absent key / non-numeric тЖТ `unverified`) and `classify`;
     - returns a fully-populated `ReproduceRecord` (claims sha256 computed from the claims file bytes by the caller or here тАФ pick one, keep pure).
  4. GREEN minimal implementation; **stdlib only** (`json`, `math`, `shlex`, `hashlib`).
  5. REFACTOR for clarity; keep every uncomputable path landing on `unverified`.
- **Validation:** `uv run pytest tests/test_reproduce.py` (inject a fake executor that writes a canned `results.json`; no real process).
- **Commit:** `feat(reproduce): claims loader + tolerance classifier + run engine (C8 slice 1)`

## Testing Strategy

- **Unit (RED first):** models/reduce (P1), claims-loader + classifier boundary matrix + run engine (P2), bundle sign/verify round-trip (P3).
- **Integration:** CLI via `CliRunner` + a scripted executor writing a canned `results.json` (P4); full-suite green gate after P4.
- **The boundary matrix is the crux** (AC 4 & 7): NaN/inf, missing key, non-numeric observed, claimed-0 vs observed-0, claimed-0 vs nonzero, exact-vs-epsilon, within-vs-outside tolerance. One test each.
- **Manual (post-merge, NOT CI, no network):** clone one small real public repo, hand-write a claims file + a tiny `results.json`-emitting wrapper, run `contig reproduce`, eyeball a sensible verdict. This is the PRD's go/no-go for slice 2 тАФ record the result in the completion note, do not add it to CI.
- **Commands:** `uv run pytest` (core); no dashboard.

## Edge Cases & Error Handling

- Non-zero script exit тЖТ all claims `unverified` (not diverged). ┬╖ Missing/malformed `results.json` тЖТ all `unverified`. ┬╖ Missing claim key / non-numeric / NaN / inf тЖТ that claim `unverified`. ┬╖ Duplicate claim id / bad tolerance / non-numeric claim value / malformed claims file тЖТ hard error, non-zero exit, **no record written**. ┬╖ Empty claims list тЖТ honest "no claims" summary, exit 0. ┬╖ Extra `results.json` keys тЖТ ignored. ┬╖ `shlex.split` on `--run` (never `shell=True`).
