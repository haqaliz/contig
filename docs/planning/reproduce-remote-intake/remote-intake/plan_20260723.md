## The ordering problem — read this before Phase 5

This is the highest-risk part of the slice (PRD RISK-1). Two constraints pull in opposite
directions:

- **R2:** the clone must happen **before** `run_started_at = time.time()` (`cli.py:858`), or
  every author-committed artifact looks fresh and the v0.46.0 freshness guard is silently
  disabled for exactly the runs it matters for.
- **"Nothing is written on a refusal":** every existing refusal (`--results` containment
  `:789-794`, `load_claims` `:796-800`, `--run` shlex `:808-815`, locator containment
  `:836-847`) happens before anything touches disk. A clone writes.

**The resolution:** the containment checks are purely *lexical* — `Path.resolve()` on a
non-existent path is well-defined and does not touch the filesystem. So compute the
**prospective checkout path** early and validate against *that*, then clone. Every cheap
refusal still precedes the expensive, disk-touching clone.

> ⚠️ **Do not** leave the containment loops validating against the raw `repo` argument in
> remote mode. `Path("https://github.com/x/y") / "out.json"` is lexically "contained" in
> `Path("https://github.com/x/y")`, so the guard would silently pass for **every** locator —
> turning a security check into a no-op precisely when the repo is untrusted third-party code.

Target order in `reproduce()`:

```
 1. classify repo argument            → local | remote | refusal        (Phase 1)
 2. remote && not --allow-fetch       → refuse, nothing written
 3. local                             → existing is_dir check (cli.py:782-784)
 4. reproduce_id = _generate_run_id() (MOVED UP from cli.py:850)
 5. repo_path = local dir OR <runs_dir>/<reproduce_id>/source   (prospective; not created yet)
 6. --results containment             (unchanged logic, new repo_path)
 7. load_claims
 8. --run shlex validation
 9. tolerance re-default
10. locator containment               (unchanged logic, new repo_path)
11. remote → fetch_repo(...)          ← FIRST disk write; failure cleans up + exits
12. claims_sha256, created_at
13. run_started_at = time.time()      ← the freshness boundary, AFTER the clone
14. run_reproduction(str(repo_path), ...)
15. attach source_url/source_commit → write_reproduce_bundle
```

Only step 4 moves; steps 6/10 keep their logic and change only what `repo_path` is.

## Phase 1 — Repo-argument classifier (pure)

- **Goal:** one pure function deciding local vs remote vs refusal. No I/O.
- **Files:** create `src/contig/fetch.py`; create `tests/test_reproduce_remote_intake.py`.
- **Steps:**
  1. RED: tests for every shape — `https://…` → remote; a plain path / `./x` / `/abs/x` →
     local; refusals for a leading `-` (argv injection: git would read `--upload-pack=…` as an
     option), `ssh://`, `git://`, `git@host:path`, `file://`, `ext::`, `http://` (not TLS),
     `doi:10.…` and a bare `10.\d+/…` (PRD R8: name DOIs explicitly rather than falling
     through to "No such repo directory").
  2. GREEN: `classify_repo_argument(arg: str) -> RepoArgument` returning a small frozen
     dataclass `(kind: Literal["local","remote"], url: str | None, refusal: str | None)`, or
     equivalent. **Check the leading `-` first**, before any scheme parsing.
  3. Refusal strings name the cause and the remedy (DOI → "pass an https:// git URL").
- **Validation:** `uv run pytest tests/test_reproduce_remote_intake.py -q`
- **Commit:** `feat(reproduce): classify a repo argument as local path, https URL, or refusal`

## Phase 2 — `Fetcher` seam + fixed argv builders

- **Goal:** the injectable seam, mirroring `Installer`.
- **Files:** modify `src/contig/runner.py` (alongside `:567-578` / `:594-647`); test in
  `tests/test_reproduce_remote_intake.py`.
- **Steps:**
  1. RED: assert `_git_clone_argv("https://h/r", Path("/d/s")) == ["git", "clone", "--depth",
     "1", "--", "https://h/r", "/d/s"]` and `_git_rev_parse_argv() == ["git", "rev-parse",
     "HEAD"]`. **The `--` terminator is required**, not cosmetic: it is the second line of
     defence against a URL that survives classification and looks like an option.
  2. GREEN: add `Fetcher = Callable[[list[str], Path], tuple[int, str]]` with a comment in the
     house style ("tests inject a fake, so no real git runs in CI"), plus `default_fetcher`
     using `subprocess.run(argv, cwd=…, stdout=PIPE, stderr=STDOUT, text=True, check=False)`
     returning `(returncode, output)` — matching `default_command_executor` (`:608-619`), not
     the bare-`int` `Installer`, because git's stderr is the only useful clone diagnostic and
     `rev-parse`'s **stdout is how the pin is read**.
  3. `default_fetcher` must convert a missing `git` binary (`FileNotFoundError`) into a
     returned non-zero + message, never an exception (R7).
- **Validation:** `uv run pytest tests/test_reproduce_remote_intake.py -q`. **`default_fetcher`
  is asserted on for argv shape and never executed** (`test_count_quantifier.py:1-8` precedent).
- **Commit:** `feat(runner): add an injectable Fetcher seam with fixed git argv builders`

## Phase 3 — `fetch_repo` orchestration

- **Goal:** clone + resolve + validate + clean up, all through the seam.
- **Prerequisites:** Phase 2.
- **Files:** modify `src/contig/fetch.py`; tests in `tests/test_reproduce_remote_intake.py`.
- **Steps:**
  1. RED, one test per branch: happy path returns `(path, sha)`; non-empty destination →
     refusal (spec AC-8); clone non-zero → refusal naming git's output; `rev-parse` non-zero →
     refusal; `rev-parse` stdout not 40-hex → refusal (PRD RISK-3 — **refuse rather than record
     an unvalidated pin**); every failure leaves **no** `<reproduce_id>/` directory (AC-7).
  2. GREEN: `fetch_repo(url, dest, *, fetcher) -> FetchResult`. Sequence: refuse non-empty
     dest → `shutil.rmtree(dest, ignore_errors=True)` + `mkdir(parents=True)` (the C2 scratch
     convention, `self_heal.py:635-638`) → clone → `rev-parse` in the checkout → `.strip()` and
     validate against `^[0-9a-f]{40}$` → return.
  3. On any failure, remove the created `<reproduce_id>/` tree so R7's "no record, no bundle"
     is literally true on disk (R9).
- **Validation:** `uv run pytest tests/test_reproduce_remote_intake.py -q`
- **Commit:** `feat(reproduce): fetch a remote repo through the seam and resolve its commit`

## Phase 5 — CLI wiring (the load-bearing phase)

- **Goal:** `--allow-fetch`, the ordering above, and the freshness regression.
- **Prerequisites:** Phases 1, 3, 4.
- **Files:** modify `src/contig/cli.py` (`:714-878`); tests in

- **Steps:**
  1. **RED first, before any wiring — the G3 regression (AC-5):** a scripted fetcher populates
     a checkout containing a **committed** `results.json` whose value *exactly equals* the

     `reproduced`. This is the test that catches a clone-after-stamp mistake, and nothing else

  3. GREEN: add the flag beside `allow_install` (`:733-742`), reusing its help-text template

     `reproduce_id` generation, and steps 6/10 keep their logic with a new `repo_path`.
  4. Extend the docstring (`:744-779`) to state the remote form and that the freshness
     requirement applies to a fetched checkout exactly as to a local one.
  5. **Do not** add `allow_fetch`/`fetcher` parameters to `run_reproduction` — the CLI resolves

## Testing strategy

- **Unit:** classifier (Phase 1), argv builders (Phase 2), `fetch_repo` branches (Phase 3),
  model/manifest round-trip + back-compat (Phase 4).
- **Integration (CLI, Phase 5):** refusals write nothing; happy-path remote run records the
  pin; **the G3 freshness regression**; local path unchanged.
- **Scripted fetcher** (the scripted-installer analogue): records the argv it was called with,
  populates a fixture tree on "clone", returns a canned SHA on "rev-parse", and can be
  scripted to fail at either step.
- **Manual, post-merge, NOT CI** (house style —
  `reproduce-published-work/skeleton/plan_20260718.md:118` "do not add it to CI"): clone one
  small real public repo through the real `default_fetcher`; confirm the SHA is recorded and
  that a repo with committed outputs reports `UNVERIFIED`. Record the result in the completion
  note. **This is the only real-network validation the slice gets** (PRD gap 1).

## Edge cases

| Case | Behavior |
|---|---|
| URL, no `--allow-fetch` | Refuse, name the flag, nothing written |
| Leading `-` | Refuse before scheme parsing (argv injection) |
| `http://`, `ssh://`, `git://`, `file://`, `ext::`, `git@h:p` | Refuse, name the accepted form |
| `doi:` / bare `10.x/…` | Refuse naming DOI explicitly (R8) |
| `git` binary absent | Non-zero + message from the seam, not an exception |
| Clone fails / host unreachable | Refuse, name git's output, remove the run dir |
| `rev-parse` non-40-hex | Refuse — never record an unvalidated pin |
| Non-empty destination | Refuse rather than clone into it |
| Cloned repo commits its outputs | `UNVERIFIED` via the freshness guard — **the headline test** |
| Pre-slice-6 bundle | Loads; `source_*` default to `None` |

## Agent execution notes

- **Checkpoint after Phase 3** (the fetch works in isolation) and **after Phase 5** (the
  ordering is right). Phase 5 is where human review is most valuable.
- **Phase 5 step 1 is not optional and not reorderable.** Writing the G3 test after the wiring
  would let a clone-after-stamp bug ship green.
- Phases 1 and 2 are independent — dispatch in parallel. Phase 4 is also independent of 1–3.
- **Ask before deviating on:** the checkout directory name (`source/` — the dig suggested
  `checkout/`; either is fine but pick one and use it everywhere), and whether `--allow-fetch`
  should imply `--allow-install` (PRD OPEN-1 leans no: the flags gate different side effects).
