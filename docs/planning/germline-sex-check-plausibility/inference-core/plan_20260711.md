This aspect is the pure science + verdict. It exposes the public functions the other two
aspects consume: `sex_signals(vcf) -> SexSignals` and
`evaluate_sex_plausibility(vcf, sample=...) -> list[QCResult]`. No `runner.py`, no
`RunRecord`, no methods/HTML here.

## Confirmed facts from the dig (do not re-derive)

- `concordance.parse_vcf(path) -> dict[(CHROM,POS,REF,ALT), gt|None]`; CHROM/POS kept
  verbatim (POS as `str`); GT normalized to sorted `"0/1"`, phased handled, missing → `None`;
  **first sample only**. Reuse it — do NOT write a second site reader.
- `parse_vcf` **skips `#` header lines**, so it does NOT expose `##contig`. Build detection
  needs its own light header scan → add a private `_open_text` (mirror
  `concordance.py:79-84`: `gzip.open(p,"rt")` when the path ends `.gz`, else `open(p)`).
- `QCStatus = Literal["pass","warn","fail","unverified"]` (`models.py:55`) — **no
  "informational" status**. Emit the raw `x_het_ratio` result as `status="pass"` with an
  explicitly-informational message (the `gene_symbol_concordance` always-PASS convention).
  `QCKind` includes `"metric"` (`models.py:64`) — use it.
- WARN-cap = a result that is only ever `pass`/`warn`/`unverified`, never `fail`. The derived
  call is **hand-built in the wrapper** (bimodal → cannot use a single `evaluate()` band),
  exactly like `variant_metrics.py`'s hand-built UNVERIFIED branch (`variant_metrics.py:167-178`).
- Confirmed thresholds: `X_HET_LOW=0.10`, `X_HET_HIGH=0.20`, `MIN_X_SITES=20`,
  `Y_PRESENT_FLOOR=5`.

### PAR / build constants (1-based, inclusive) — VERIFY against an authoritative source in RED

```
GRCh37 (hg19): chrX length 155_270_560
  PAR1 60_001–2_699_520 ; PAR2 154_931_044–155_260_560
GRCh38 (hg38): chrX length 156_040_895
  PAR1 10_001–2_781_479 ; PAR2 155_701_383–156_030_895
```
Y-PAR (for Y-presence masking, M3/N1):
```
GRCh37 chrY: PAR1 60_001–2_699_520 ; PAR2 59_034_050–59_373_566
GRCh38 chrY: PAR1 10_001–2_781_479 ; PAR2 56_887_903–57_217_415
```
Build detected from `##contig=<ID=(chr)?X,length=L>`: `L==155_270_560 → "GRCh37"`,
`L==156_040_895 → "GRCh38"`, else `None`. A wrong length silently disables masking, so the
male-pattern PAR test (Phase 2) is the guard.

## Data model (in `sex_plausibility.py`)

```python
@dataclass(frozen=True)
class SexSignals:
    inferred_sex: str            # "XY" | "XX" | "discordant" | "indeterminate"
    x_het_ratio: float | None    # None when x_sites < MIN_X_SITES or no X contig
    x_sites: int                 # biallelic, non-missing, non-PAR X genotypes counted
    y_variant_count: int         # non-PAR Y variant sites
    par_masked: bool             # False when build undetermined
    reference_build: str | None  # "GRCh37" | "GRCh38" | None
```

## Implementation phases (each: RED → GREEN → REFACTOR, suite stays green)

### Phase 1 — constants + build detection

- **Goal:** thresholds/PAR tables exist; `_detect_build(vcf)` reads the `##contig` X length.
- **Files:** `rule_pack.py` (constants + `SEX_PLAUSIBILITY_PACK`), `sex_plausibility.py`
  (`_open_text`, PAR tables, `_detect_build`), `test_sex_plausibility.py`.
- **RED tests:**
  - assert the four thresholds and the two chrX lengths equal the pinned values.
  - `_detect_build` on a header with `##contig=<ID=chrX,length=156040895>` → `"GRCh38"`;
    `155270560` → `"GRCh37"`; a header with no `##contig` / an unknown length → `None`;
    both `chrX` and bare `X` recognized.
- **GREEN:** add constants + `_detect_build` (gzip-transparent header scan; stop at first
  non-`#` line).
- **Validation:** `uv run pytest tests/verification/test_sex_plausibility.py -k "build or const"`.
- **Commit:** `test+feat(verify): sex-check thresholds + PAR tables + VCF build detection [inference-core]`

### Phase 2 — X-heterozygosity with PAR masking

- **Goal:** compute `x_het_ratio` + `x_sites` over biallelic, non-missing, **non-PAR** X
  genotypes; `None`/`0` when < `MIN_X_SITES` or no X contig.
- **Files:** `sex_plausibility.py` (`_x_signals` helper), tests.
- **RED tests (tmp_path VCFs via `_HEADER`/`_vcf_line`/`_write_vcf`):**
  - a female-pattern chrX block (autosomal-level het) → ratio ≥ `X_HET_HIGH`, `x_sites` correct.
  - a male-pattern chrX block (mostly hom) → ratio ≤ `X_HET_LOW`.
  - **PAR exclusion:** a GRCh38 fixture whose only het chrX sites sit inside PAR1 → those
    sites are excluded from `x_sites`, ratio reads male/low (the load-bearing masking test).
  - < 20 X sites → `x_het_ratio is None`.
  - no chrX contig → `x_het_ratio is None`, `x_sites == 0`.
- **GREEN:** implement; het = alleles differ, denominator = non-missing biallelic X genotypes
  (het + hom-ref + hom-alt). Reuse `parse_vcf`.
- **Validation:** `uv run pytest tests/verification/test_sex_plausibility.py -k xhet`.
- **Commit:** `test+feat(verify): X-heterozygosity ratio with PAR masking [inference-core]`

### Phase 3 — Y presence + `sex_signals` assembly + `inferred_sex`

- **Goal:** `_y_count` (non-PAR Y sites); `sex_signals(vcf)` assembles `SexSignals` and derives
  `inferred_sex`.
- **Files:** `sex_plausibility.py` (`_y_count`, `sex_signals`), tests.
- **Derivation logic:**
  - `x_het_ratio is None` (or no X contig) → `"indeterminate"`.
  - `x_het_ratio <= X_HET_LOW` → `"XY"` (Y present strengthens the message, does not change
    the call).
  - `x_het_ratio >= X_HET_HIGH` and `y_variant_count < Y_PRESENT_FLOOR` → `"XX"`.
  - `x_het_ratio >= X_HET_HIGH` and `y_variant_count >= Y_PRESENT_FLOOR` → `"discordant"`.
  - `X_HET_LOW < x_het_ratio < X_HET_HIGH` (mid-band) → `"discordant"`.
  - **Y-absence never forces discordant** (uninformative asymmetry).
- **RED tests:** male-pattern → `"XY"`; female-pattern (no Y) → `"XX"`; autosomal-X + Y
  variants → `"discordant"`; mid-band ratio → `"discordant"`; too-few-X → `"indeterminate"`;
  Y-PAR sites excluded from `y_variant_count`; `chrY`/`Y` both recognized.
- **GREEN:** implement.
- **Validation:** `uv run pytest tests/verification/test_sex_plausibility.py -k "signals or infer"`.
- **Commit:** `test+feat(verify): Y-presence + karyotype inference (sex_signals) [inference-core]`

### Phase 4 — `evaluate_sex_plausibility` + robustness/refactor

- **Goal:** the public evaluator emitting QCResults; gzip + case-insensitivity + fallback
  hardened.
- **Files:** `sex_plausibility.py` (`evaluate_sex_plausibility`), tests.
- **Result contract:**
  - one `sex_plausibility:{sample}`: `"XY"`/`"XX"` → `status="pass"` (message names the
    karyotype + evidence); `"discordant"` → `status="warn"` (message names the conflict +
    "possible aneuploidy / contamination / sample swap"), **assert `!= "fail"`**;
    `"indeterminate"` → `status="unverified"`, `value=None`.
  - one informational `x_het_ratio:{sample}`: `status="pass"`, `value=` the ratio (or omit /
    `None` when indeterminate), message explicitly "informational". `kind="metric"` on both.
- **RED tests:** the pass/warn/unverified matrix over the Phase-3 fixtures; a **gzip
  round-trip** (`.vcf.gz` == `.vcf`); a **build-undetermined** fixture → `par_masked False`,
  `reference_build None`, unmasked ratio, still at most `warn`.
- **GREEN + REFACTOR:** implement; factor shared bits; ensure the whole new test module and
  `uv run pytest` are green.
- **Validation:** `uv run pytest tests/verification/test_sex_plausibility.py` then
  `uv run pytest` (full suite).
- **Commit:** `test+feat(verify): sex_plausibility evaluator (WARN-capped, UNVERIFIED-when-weak) [inference-core]`

## Edge cases & error handling

- No X contig / no Y contig → `indeterminate` / `y_variant_count 0`; never a crash.
- Build undetermined → unmasked fallback, flagged in `SexSignals`, still WARN-capped.
- Multiallelic / indel X sites → excluded from het (biallelic only), consistent with
  `variant_metrics.py`.
- Missing GT (`./.`) → excluded from numerator and denominator.
- Multi-sample VCF → first sample only (inherited from `parse_vcf`); documented, not an error.
- Non-human / aneuploid karyotype → `indeterminate` or `discordant`+WARN, never confident-wrong.
