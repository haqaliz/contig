## Conventions

- ts_tv: over **biallelic SNV** sites only (REF and ALT each a single base in
  ACGT). Transition if `{REF,ALT}` is `{A,G}` or `{C,T}`; else transversion.
  `ts_tv = transitions / transversions`, `None` if transversions == 0.
- het_hom: from the normalized GT. Heterozygous if the two alleles differ (e.g.
  `0/1`); homozygous-alt if both alleles are the same non-zero (e.g. `1/1`).
  Homozygous-ref (`0/0`) and missing genotypes are excluded. `het_hom =
  het_count / hom_alt_count`, `None` if hom_alt_count == 0.
- Checks are `kind="metric"` (no new kind, no report/dashboard change).
- WARN cap: the activated checks never FAIL in this slice.

## Phase 1: VCF-derived variant metrics

**Goal:** deterministic `ts_tv` and `het_hom` from a VCF.
**Files:** create `src/contig/verification/variant_metrics.py`,
`tests/verification/test_variant_metrics.py`.

**RED (write first):**
1. `test_ts_tv_counts_transitions_over_transversions`, inline VCF with known SNVs
   (e.g. 2 transitions A>G, C>T and 1 transversion A>C) -> ratio 2.0.
2. `test_ts_tv_ignores_indels_and_multiallelic`, indel and multiallelic rows do
   not count toward ts_tv.
3. `test_ts_tv_none_when_no_transversions`, only transitions -> ratio None.
4. `test_het_hom_counts_genotypes`, mix of `0/1` and `1/1` -> het/hom-alt ratio;
   `0/0` and `./.` excluded.
5. `test_het_hom_none_when_no_hom_alt`, only het genotypes -> ratio None.
6. `test_gzip_vcf_supported`, a `.vcf.gz` parses identically.

**GREEN:**
- Reuse `concordance.parse_vcf(path) -> {(CHROM,POS,REF,ALT): gt}`. From its output
  compute ts_tv (REF/ALT from the key) and het_hom (gt). If `parse_vcf` does not
  expose enough (it normalizes GT but the site key has REF/ALT, which is all ts_tv
  needs), reuse it directly; otherwise add a thin parse in `variant_metrics.py`.
- Expose `variant_metrics(vcf_path) -> VariantMetrics` (a small frozen dataclass or
  dict with `ts_tv: float | None`, `het_hom: float | None`, plus the primary
  `sample` name parsed from the `#CHROM` header line, or a stable synthetic key).

**Validation:** `uv run pytest tests/verification/test_variant_metrics.py -q`.
**Commit:** `feat(plausibility): compute germline ts_tv and het_hom from the VCF`.

## Phase 2: germline plausibility evaluator + WARN cap

**Goal:** turn the computed metrics into QCResults using the existing rule bands,
capped at WARN, with explicit `unverified` for uncomputable metrics.
**Prerequisites:** Phase 1.
**Files:** modify `src/contig/verification/variant_metrics.py`,
`src/contig/verification/rule_pack.py`; tests in `test_variant_metrics.py` and a
check in `tests/verification/test_rule_pack.py`.

**RED:**
1. `test_plausibility_in_band_passes`, a VCF whose ts_tv is in band -> PASS
   `ts_tv_ratio` check, kind="metric".
2. `test_plausibility_out_of_band_warns_never_fails`, ts_tv far outside the band
   (e.g. 0.5) -> WARN, never FAIL, with value and expected range in the result.
3. `test_plausibility_uncomputable_is_unverified`, no hom-alt -> `het_hom_ratio`
   check status `unverified` (not skipped, not PASS).
4. `test_variant_pack_ts_tv_is_warn_only`, `VARIANT_RULE_PACK`'s `ts_tv_ratio` and
   `het_hom_ratio` entries have no `fail_below`/`fail_above`.

**GREEN:**
- In `rule_pack.py`: drop `fail_below`/`fail_above` from the `ts_tv_ratio` and
  `het_hom_ratio` entries (keep `warn_below`/`warn_above`). Leave `mean_coverage`.
- In `variant_metrics.py`, add `evaluate_variant_plausibility(vcf_path) ->
  list[QCResult]`:
  - compute metrics (Phase 1); build a `{sample: {metric: value}}` dict with only
    the **computable** metrics; run `evaluate(metrics, [the two rules])` for the
    pass/warn results (reuses existing band logic);
  - for each **uncomputable** metric (None), append a `QCResult(check=
    "<check>:<sample>", status="unverified", message=why, kind="metric")`.
- Run the full suite; fix any existing test that asserted the old FAIL bands on
  `VARIANT_RULE_PACK` (the two-sided fixture tests use a LOCAL `_ts_tv_range_pack`,
  so they should be unaffected; verify).

**Validation:** `uv run pytest tests/verification -q`.
**Commit:** `feat(plausibility): WARN-capped germline plausibility evaluator`.

## Phase 3: wire it into the germline QC path (independent of MultiQC)

**Goal:** germline runs get the plausibility checks even with no MultiQC report.
**Prerequisites:** Phase 2.
**Files:** modify `src/contig/verification/run_qc.py` and/or
`src/contig/runner.py:_discover_qc`; tests in `tests/verification/test_run_qc.py`.

**RED:**
1. `test_discover_qc_includes_plausibility_for_variant_run`, a `variant_calling`
   run dir with a `results/*.vcf.gz` and NO MultiQC yields the plausibility checks
   (proves the independent path).
2. `test_plausibility_not_run_for_non_variant_assay`, an `rnaseq` run does not get
   variant plausibility checks.
3. `test_plausibility_vcf_value_authoritative`, when MultiQC also carries `ts_tv`,
   the VCF-computed value is the one in the emitted check.

**GREEN:**
- In `_discover_qc(run_dir, assay)` (and/or `run_qc`): when `assay ==
  "variant_calling"`, locate the primary VCF via the `variant_calling` manifest
  glob (`manifest_for("variant_calling").required[0]`, i.e. `*.vcf.gz`), and append
  `evaluate_variant_plausibility(vcf)` to the results. This runs **regardless** of
  whether a MultiQC report was found (not inside the `if multiqc is not None`
  block). No VCF found -> skip cleanly (no crash).
- Ensure the merge keeps the VCF value authoritative if the same metric also came
  from MultiQC (compute plausibility from the VCF directly, so MultiQC's value is
  not used for these two checks).

**Validation:** `uv run pytest tests/verification -q` then full `uv run pytest -q`.
**Commit:** `feat(plausibility): evaluate germline plausibility on a MultiQC-independent path`.

## Testing strategy

- Unit: metric math (ts_tv SNV-only, het_hom genotype classes), None cases, gzip.
- Integration: `_discover_qc` for a variant run with/without MultiQC; non-variant
  skip; VCF-authoritative.
- Determinism: tiny inline VCF fixtures in `tmp_path`; no tool execution, no
  network (mirrors `test_concordance.py`, `test_structural.py`).
- Regression: full `uv run pytest` stays green; no dashboard/report change needed
  (kind="metric" already renders).

## Edge cases & error handling

- No VCF for a variant run -> no plausibility checks (clean skip), not a crash.
- Zero SNVs -> `ts_tv_ratio` unverified. Zero hom-alt -> `het_hom_ratio` unverified.
- Multiallelic / indels -> excluded from ts_tv; documented.
- Multi-sample VCF -> primary sample only (slice 1); documented.
