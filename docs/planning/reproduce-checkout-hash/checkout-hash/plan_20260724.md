## Decisions locked (from the review gate)

- **Signed** field on `ReproduceRecord` (attested) ‚Äî accept the disclosed **third**
  signature break; bounded to opt-in `CONTIG_SIGNING_KEY` signers; pinned by a test.
- **Remote/fetched-only** scope this slice; local-path hashing deferred.
- **Pre-run** hash ‚Äî computed right after `fetch_repo` succeeds, **before** the
  `run_started_at` stamp, so an `--allow-install` retry mutation never changes it.
- **Symlinks:** do **not** follow ‚Äî walk with `os.walk(followlinks=False)`, skip
  symlinked files, prune symlinked dirs (containment, determinism).
- **Algorithm (published so third parties can recompute byte-for-byte):** walk the
  tree with `os.walk(followlinks=False)`; **prune any directory component named
  `.git`** and any symlinked dir; for each regular (non-symlink) file, key =
  POSIX-relative path, value = `sha256_file(path)`; **sort by key**; fold the sorted
  lines `f"{relpath}\0{hexdigest}\n"` (NUL delimiter ‚Äî illegal in POSIX paths) and
  `sha256` the UTF-8 encoding of the concatenation; return the hex digest. Any
  `OSError` while reading a file, or a non-directory / missing root, ‚Üí **`None`**
  (never a fabricated or partial digest).

## Phase 1 ‚Äî `compute_tree_sha256` helper (pure, fully CI-observable)

- **Goal:** a deterministic, honest, stdlib tree digest ‚Äî the load-bearing, fully
  testable core (no git/network).
- **Prerequisites:** none.
- **Files:**
  - `tests/test_reproduce_checkout_hash.py` (new) ‚Äî Phase-1 tests.
  - `src/contig/bundle.py` ‚Äî add `compute_tree_sha256` next to
    `compute_output_checksums` (`bundle.py:303`).
- **RED ‚Äî write these tests first (all build fixture trees on disk under
  `tmp_path`, no git):**
  1. `test_compute_tree_sha256_is_deterministic` ‚Äî same tree hashed twice ‚Üí equal;
     a 64-char lowercase hex string.
  2. `test_compute_tree_sha256_order_independent` ‚Äî two trees built by writing the
     same files in different creation order ‚Üí equal digests (proves the sort).
  3. `test_compute_tree_sha256_changes_when_a_file_changes` ‚Äî mutate one byte in one
     file ‚Üí digest differs. **This is the tamper signal (PRD success #3).**
  4. `test_compute_tree_sha256_changes_when_a_file_is_added_or_removed`.
  5. `test_compute_tree_sha256_excludes_dot_git_anywhere` ‚Äî a `.git/` dir at root and
     a nested `sub/.git/` are both ignored: adding/altering files under either does
     **not** change the digest; a real file named `git` (no dot) is **not** excluded.
  6. `test_compute_tree_sha256_empty_dir_does_not_affect_digest` ‚Äî a tree with an
     extra empty subdir equals the tree without it (documented equivalence).
  7. `test_compute_tree_sha256_skips_symlinks` ‚Äî a symlink file inside the tree is
     not hashed and is not followed; a symlink pointing outside the tree never causes
     a read outside root (containment). (Guard with `pytest`'s symlink availability;
     skip on platforms without symlink support.)
  8. `test_compute_tree_sha256_unreadable_file_returns_none` ‚Äî a file made unreadable
     (chmod `0o000`, skip if running as root where perms are bypassed) ‚Üí whole result
     `None`. Never partial.
  9. `test_compute_tree_sha256_missing_or_nondir_root_returns_none` ‚Äî a path that does
     not exist, and a path that is a file, both ‚Üí `None`.
  10. `test_compute_tree_sha256_matches_documented_algorithm` ‚Äî recompute the digest
      in-test by the published fold (independent of the helper's internals) over a
      tiny fixture and assert equality; this pins the algorithm so the CHANGELOG spec
      stays honest.
- **GREEN ‚Äî implement `compute_tree_sha256(root: str | Path) -> str | None`:**
  ```python
  def compute_tree_sha256(root: str | Path) -> str | None:
      base = Path(root)
      if not base.is_dir():
          return None
      lines: list[str] = []
      try:
          for dirpath, dirnames, filenames in os.walk(base, followlinks=False):
              # prune .git dirs and symlinked dirs in place (os.walk honors edits)
              dirnames[:] = [
                  d for d in dirnames
                  if d != ".git" and not Path(dirpath, d).is_symlink()
              ]
              for name in filenames:
                  p = Path(dirpath, name)
                  if p.is_symlink() or not p.is_file():
                      continue
                  rel = p.relative_to(base).as_posix()
                  lines.append(f"{rel}\0{sha256_file(p)}\n")
      except OSError:
          return None
      blob = "".join(sorted(lines)).encode("utf-8")
      return hashlib.sha256(blob).hexdigest()
  ```
  - Add `import os` / confirm `hashlib` import at top of `bundle.py`.
  - Note: pruning by `d != ".git"` removes `.git` at any depth (os.walk visits every
    dir); a nested `sub/.git` never yields files because it is pruned before descent.
- **Validation:** `uv run pytest tests/test_reproduce_checkout_hash.py`.
- **Commit:** `feat(reproduce): compute_tree_sha256 ‚Äî deterministic stdlib checkout-tree digest`

## Environment & Reproducibility

- No env vars needed for the feature. `CONTIG_SIGNING_KEY` only exercises the
  signing tests (already used by `test_reproduce_bundle.py`).
- Determinism is the whole point: sorted POSIX-relative keys, `os.walk(followlinks=
  False)`, NUL fold delimiter, `.git` pruned. The digest must be reproducible across
  platforms for the same tree content (unicode/case are FS-inherited limits,
  documented, not solved).

## Edge Cases & Error Handling

- Missing / non-directory root ‚Üí `None`. Unreadable file ‚Üí whole digest `None`
  (never partial/fabricated). Symlinks never followed. `.git` at any depth excluded.
  Empty dirs are digest-invisible (documented). Local runs ‚Üí field stays `None`.
- Post-run tree divergence (run outputs added to `source/`) is **expected** and is
  why the hash is taken pre-run; documented in the CHANGELOG (R3).
