## Nature of the change

**Data + tests + docs only.** The Phase-2 dig confirmed the scorer (`_status_for`,
`rule_pack.py:435-454`), evaluator (`evaluate_variant_plausibility`), verdict reducer
(`overall_verdict`, `models.py:78-96`), report, and CLI are all unchanged — adding
`fail_*` keys to the three germline rule dicts is sufficient to produce `status="fail"`,
which flows to `record.verdict`. No CLI exit-code wiring (verdict-only, per the PRD).

This is one cohesive unit that touches `rule_pack.py` + two test files. It is **not**
parallelizable across agents (they would collide on the same files), so it is executed as
**two sequential agent tasks** under strict TDD: Task A (code + tests), then Task B (docs).
