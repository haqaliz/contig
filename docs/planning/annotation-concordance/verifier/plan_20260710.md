## Phase 2 — Concordance core: discovery, site-keyed join, `consequence_concordance` (M-2, M-3, M-4)

- **Goal:** a pure module that, given a VEP source and a SnpEff source of per-variant
  most-severe consequences keyed on (CHROM,POS,REF,ALT), computes a WARN-capable
  consequence-agreement fraction, honest UNVERIFIED below floor, handling both the
  two-file and single-VCF-both-fields layouts and recording which it used.
- **Prerequisites:** Phase 1 (not strictly — can proceed in parallel).
- **Files:** create `src/contig/verification/annotation_concordance.py`,
  `tests/test_annotation_concordance.py`.
- **Design (mirror `somatic_concordance.py`):**
  - Constants: `_WARN_BELOW = 0.90`, `_MIN_SHARED_VARIANTS = 10`,
    `SiteKey = tuple[str,str,str,str]`.
  - `_concordance(check, status, message, value=None, expected_range=None) -> QCResult`
    factory tagging `kind="concordance"` (copy verbatim).
  - **Reuse from `annotation_plausibility`** (import, do not fork):
    `_variant_terms`, `_resolve_consequence_index`, `_most_severe_rank`, `_SEVERITY_RANK`,
    `_UNKNOWN_RANK`; and from `annotation_structural`: `_open_text`, `_declared_key`.
    For the **most-severe TERM** (not just rank) add a tiny local inverse:
    `_most_severe_term(terms)` returning the term whose `_SEVERITY_RANK` is max (stable
    tie-break by the severity order), or None.
  - `parse_consequences(vcf_path, key) -> dict[SiteKey, str]`: stream the VCF
    (gzip-transparent via `_open_text`), resolve the consequence index for `key`
    (`_resolve_consequence_index`), and for each data record with ≥8 cols and the key in
    INFO, map `(cols[0],cols[1],cols[3],cols[4])` → `_most_severe_term(_variant_terms(...))`.
    Skips records without the key. **Dual-key safe:** called once per key; the single-VCF
    layout calls it twice on the same path with `"CSQ"` then `"ANN"` — this is M4's own
    dual-key path, `annotation_plausibility` is never touched.
  - `_layout(vep_map, snpeff_map, detected: str)`: internal helper to build the two maps
    depending on layout (see discovery below) and remember `detected ∈ {"two-file",
    "single-vcf-both"}`.
  - `evaluate_consequence_concordance(vep_map, snpeff_map, *, layout) -> list[QCResult]`:
    shared = keys in both; if `len(shared) < _MIN_SHARED_VARIANTS` → one UNVERIFIED
    (value None, message names the floor + that concordance isn't ground truth). Else
    fraction = matches/shared (rounded 4dp), `warn` if `< _WARN_BELOW` else `pass`,
    message names both tools, the shared count, and the detected layout.
- **TDD steps (write fixtures first):**
  1. RED: `test_consequence_concordant_two_maps` — ≥10 shared, all matching → PASS,
     value 1.0, `kind=="concordance"`. Fails (module absent).
  2. Implement constants + factory + `parse_consequences` + `_most_severe_term` +
     `evaluate_consequence_concordance` → GREEN.
  3. Add: divergent (some mismatch, ≥10 shared, fraction < 0.90) → WARN naming both tools;
     exactly-at-0.90 boundary → PASS (>=); below-floor (<10 shared) → UNVERIFIED value
     None; empty maps → UNVERIFIED.
  4. Add the **single-VCF-both** fixture: one VCF string with both
     `##INFO=<ID=CSQ,...Format:...|Consequence|...>` and `##INFO=<ID=ANN,...>` and records
     carrying both — assert `parse_consequences(path,"CSQ")` and `(path,"ANN")` each yield
     the right map, and the combined evaluate reports `layout="single-vcf-both"`.
- **Validation:** `uv run pytest tests/test_annotation_concordance.py` green; run the full
  suite to confirm `annotation_plausibility`/`annotation_structural` tests untouched (G5).
- **Commit:** `feat(verify): consequence concordance core for annotation (VEP vs SnpEff) [C7 M4]`

## Phase 3 — `gene_symbol_concordance` (informational-only) (M-5)

- **Goal:** a second, always-PASS metric reporting gene-symbol agreement, with fixed
  minimal normalization; never WARNs.
- **Prerequisites:** Phase 2 (shares the parse/join scaffolding).
- **Files:** extend `annotation_concordance.py` + `tests/test_annotation_concordance.py`.
- **Design:**
  - `_symbol_index_csq(header_lines) -> int | None`: mirror `_consequence_index_csq` but
    `fields.index("SYMBOL")`. SnpEff ANN gene symbol is fixed index 3 (`_ANN_SYMBOL_INDEX
    = 3`).
  - `_normalize_symbol(s) -> str | None`: strip + case-fold; `""`/`"."` → None
    (unresolvable). **No alias table** (gap-fix 1, documented in a comment).
  - `parse_symbols(vcf_path, key) -> dict[SiteKey, str | None]`: like `parse_consequences`
    but extracts the (normalized) most-severe transcript's symbol. Keep it simple: take the
    symbol from the first transcript entry (documented choice) — or the transcript that
    produced the most-severe consequence if cheaply available; first-entry is acceptable
    and testable. Record None when unresolvable.
  - `evaluate_gene_symbol_concordance(vep_syms, snpeff_syms) -> list[QCResult]`: denominator
    = shared keys where BOTH symbols resolvable; if `< _MIN_SHARED_VARIANTS` → UNVERIFIED.
    Else fraction = matches/resolvable, status ALWAYS `"pass"` (informational), message
    labels it "informational" and names both tools + the fraction. Never warn/fail.
- **TDD steps:**
  1. RED: `test_gene_symbol_informational_pass` — divergent symbols but ≥10 resolvable →
     status PASS (not WARN), value = the (low) fraction. Fails.
  2. Implement → GREEN.
  3. Add: unresolvable-heavy fixture (< 10 resolvable pairs) → UNVERIFIED; case-only
     difference (`BRCA1` vs `brca1`) → counts as a MATCH (normalization); `.`/empty on one
     side → excluded from denominator.
- **Validation:** `uv run pytest tests/test_annotation_concordance.py` green.
- **Commit:** `feat(verify): informational gene-symbol concordance for annotation [C7 M4]`

## Phase 4 — Discovery + auto-wire into `_discover_qc` (M-3 discovery, M-6, M-8)

- **Goal:** a run-level entry that discovers the annotation source(s) under the run dir,
  runs both metrics, and is auto-wired for both variant assays with no CLI flag.
- **Prerequisites:** Phases 2–3.
- **Files:** `annotation_concordance.py` (run-level entry), `src/contig/runner.py:150-163`,
  `tests/test_annotation_concordance_wiring.py`.
- **Design:**
  - `evaluate_annotation_concordance_from_run(run_dir, vcfs=None) -> list[QCResult]`:
    - Gather annotated VCFs (`run_dir.rglob("*.vcf.gz")` or the passed `vcfs`), keep those
      whose header declares CSQ or ANN.
    - **Two-file layout:** select a VEP VCF (a `vep` path component OR header declares CSQ)
      and a SnpEff VCF (`snpeff` component OR header declares ANN). If exactly one of each
      → `parse_consequences`/`parse_symbols` per file, `layout="two-file"`.
    - **Single-VCF-both layout:** if a single VCF declares BOTH CSQ and ANN → parse both
      keys off it, `layout="single-vcf-both"`.
    - **Skips/UNVERIFIED (never false pass):** zero annotated VCFs → `[]` (clean skip);
      only one annotator resolvable → both metrics UNVERIFIED with a clear message; an
      ambiguous multi-file layout (>1 candidate per annotator across different dirs) → one
      UNVERIFIED (mirror `select_caller_vcfs`).
    - Return `consequence` + `gene_symbol` results, each carrying the detected layout in
      the message (breadcrumb).
  - Wire in `runner.py` inside the existing `if assay in VARIANT_ASSAYS:` block, after the
    structural/plausibility loop:
    `results.extend(evaluate_annotation_concordance_from_run(run_dir))`. Import at top
    (near `runner.py:29`).
- **TDD steps:**
  1. RED: `test_wiring_two_file_run` — build a tmp run dir with `.../vep/x.vcf.gz` and
     `.../snpeff/x.vcf.gz` (concordant, ≥10 sites), call `_discover_qc(run_dir,
     "variant_calling")`, assert both concordance checks present with `kind=="concordance"`.

     -both run → checks present with `layout="single-vcf-both"`.
- **Validation:** `uv run pytest tests/test_annotation_concordance_wiring.py` + full suite.

## Edge cases & error handling

- Only one annotator present / annotation absent (live-cache) → UNVERIFIED, never PASS.
- Ambiguous multi-file layout → one UNVERIFIED (mirror `select_caller_vcfs`).
- Malformed CSQ `Format:` / missing SYMBOL subfield → consequence UNVERIFIED / symbol
  unresolvable, never a crash.
- Multi-transcript & `&`-joined terms → handled by reused `_variant_terms`.
- Non-injective/odd records (<8 cols) → skipped by the parser.
- **Never mutate `annotation_plausibility.py`'s single-key driver** — M4 owns its dual-key
  parse (hard constraint; a phase-2/3 checkpoint verifies M3 tests unchanged).
