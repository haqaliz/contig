### RED — write these tests first (`tests/verification/test_count_quantifier.py`)
1. `test_kallisto_command_builds_expected_argv` — `kallisto_command(fastqs, index, out_dir)`
   returns argv containing `_KALLISTO`, `quant`, `-i <index>`, `-o <out_dir>`, and the
   fastq paths. Pure; no execution.
2. `test_tx2gene_path_resolves_under_index` — `tx2gene_path(index)` returns
   `<index>/t2g.txt` (the documented kb-ref convention). **This is the review-gate
   must-have:** it asserts the collapse input (the t2g path) is explicit and anchored to the
   index, so the gene-collapse cannot be silently dropped.
3. `test_collapse_to_gene_sums_transcripts` — `collapse_to_gene(abundance_rows, t2g)` sums
   transcript `est_counts` to gene level given synthetic rows + a `{transcript: gene}` map;
   two transcripts of one gene sum; unknown transcript is dropped (documented). **Pure,
   runs for real in CI** — the correctness of the scientific substance.
4. `test_missing_binary_raises` — monkeypatch `_KALLISTO` to a bogus name; a fake `reads`
   sheet + `index` exist; `run_kallisto_quantifier` raises `SecondQuantifierError` (mirrors
   `test_second_caller.py` missing-binary path).
5. `test_missing_reads_raises` — nonexistent `reads` → `SecondQuantifierError`.
6. `test_missing_index_raises` — nonexistent `index` → `SecondQuantifierError`.

## Testing Strategy

- **Unit (Phase 1):** argv builder shape; t2g resolver; **collapse correctness (real)**;
  three error paths. kallisto never executed.
- **Integration (Phase 2):** injected fake quantifier; PASS/WARN, at-most-WARN exit 0, JSON,
  three skip paths, mutual exclusion. Reuse the rnaseq primary-matrix fixture from the
  existing `--concordance-counts` tests.
- **Regression:** full `uv run pytest` after each phase; assert no change to existing
  verify/exit/concordance behavior.

## Environment & Reproducibility

- No env vars; no external services in CI. kallisto is a manual-gate-only dependency.
- Determinism: collapse + parse are pure/stdlib; the seam writes to a `TemporaryDirectory`;
  no run-record or reproduce-contract change (concordance is verify-time, attach/echo only).

## Edge Cases & Error Handling

- `--reads`/`--index` passed **without** `--concordance-counts-auto`: silently unused
  (matches germline `--bam`/`--ref`); no special handling. Note in the docstring.
- Empty/one-column produced matrix or <10 shared genes → existing
  `evaluate_count_concordance` returns UNVERIFIED (never a false pass) — no new code.
- Transcript in abundance but absent from t2g → dropped by `collapse_to_gene` (documented);
  a t2g file missing entirely → `SecondQuantifierError` (skip note), never a silent
  transcript-level matrix.
- Quantifier writes nothing / no `abundance.tsv` → `SecondQuantifierError` (skip note).
