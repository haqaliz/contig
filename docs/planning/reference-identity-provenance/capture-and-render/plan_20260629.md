## 0. Context recap (grounded in code, not assumed)

- `RunRecord` (`src/contig/models.py:235-270`) has **no** reference fields today; it
  does carry `parameters: dict[str, object]`.
- The resolved reference lands in `params`: `cli.py:380`
  `params.update(resolve_reference(genome=..., fasta=..., gtf=...))`, and
  `runner.py:293` sets `record.parameters = params or {}`. So **at finalize the
  reference keys are already on `record.parameters`** — `genome` (iGenomes mode) or
  `fasta`+`gtf` (explicit mode), or neither (Snakemake / no-ref).
- `resolve_reference` (`reference.py:21-41`) absolutizes `fasta`/`gtf`.
- Hash helper: `sha256_file(path)` (`models.py:17-23`), streaming, returns a hex
  digest. Checksums elsewhere are plain `dict[str, str]`; there is **no** `FileHash`
  model.
- Finalize seam: `_finalize` (`self_heal.py:703-735`) already mutates the record
  (`output_checksums`, `resource_usage`) then `write_bundle`. **This is where capture
  goes.** No new arguments needed — read `record.parameters`.
- Renderers: `render_methods` (`methods.py:69-95`, helper `_provenance_clause`
  36-49); HTML provenance panel (`report.py:304-320`, helper `_provenance_rows`).
- Test conventions: `tests/test_models.py`, `tests/test_bundle.py` (fixtures
  `_minimal_record`/`_full_record`, checksum tests 82-159), `tests/test_methods.py`,
  `tests/test_report.py`, `tests/test_reference.py`. Construct `RunRecord(...)`
  directly with mocked `"a"*64` hashes.

**No new dependencies.** No `uv add`. No env vars. nf-core-only is inherent (Snakemake
has no reference keys → `None`, which is correct).

### Phase 2 — Capture helper: `compute_reference_identity(params)`
- **Goal:** Pure function deriving identity from a run's parameters, hashing local
  files, degrading gracefully.
- **Prerequisites:** Phase 1. **Blocks:** Phase 3.
- **Files:** `src/contig/bundle.py`, `tests/test_bundle.py`.
- **Steps:**
  1. RED — in `test_bundle.py`, mirror the existing checksum tests:
     - explicit: write two `tmp_path` files; `params={"fasta": str(fa), "gtf":
       str(gtf)}` → returns `mode="explicit"`, `fasta_sha256 == sha256_file(fa)`,
       `gtf_sha256 == sha256_file(gtf)`, `fasta`/`gtf` set.
     - igenomes: `params={"genome": "GRCh38"}` → `mode="igenomes"`, `genome=="GRCh38"`,
       both checksums `None`; **does not read any file** (use a non-existent path is
       irrelevant here — assert no exception and checksums None).
     - no-ref: `params={"outdir": "..."}` (no reference keys) → returns `None`.
     - missing file: `params={"fasta": str(tmp_path/"nope.fa"), "gtf": str(gtf)}` →
       `fasta_sha256 is None`, `gtf_sha256 == sha256_file(gtf)`, no crash.
     - determinism: calling twice on the same file yields equal digests (covers
       re-run byte-stability, success criterion 5/3).
  2. GREEN — implement in `bundle.py` (import `ReferenceIdentity`, reuse
     `sha256_file`):
     ```python
     def compute_reference_identity(params):
         if not params:
             return None
         genome = params.get("genome")
         fasta = params.get("fasta")
         gtf = params.get("gtf")
         if genome:
             return ReferenceIdentity(mode="igenomes", genome=str(genome))
         if not fasta and not gtf:
             return None
         def _h(p):
             try:
                 return sha256_file(p) if p and Path(p).is_file() else None
             except OSError:
                 return None
         return ReferenceIdentity(
             mode="explicit",
             fasta=str(fasta) if fasta else None,
             gtf=str(gtf) if gtf else None,
             fasta_sha256=_h(fasta),
             gtf_sha256=_h(gtf),
         )
     ```
     (Guard `OSError` so an unreadable/permission-denied path degrades, not crashes.)
  3. REFACTOR — keep alongside `compute_input_checksums`/`compute_output_checksums`
     for discoverability.
- **Validation:** `uv run pytest tests/test_bundle.py -q`.
- **Commit:** `feat(bundle): compute_reference_identity from run params (explicit hashes, igenomes key-only)`

## 5. Environment & reproducibility

- `uv sync`; no new deps, no env vars, no external tools.
- **Determinism:** capture is a pure function of file bytes (`sha256_file`), so the
  identity reproduces byte-for-byte on re-run of the same files (PRD success
  criterion 3) — asserted by the determinism test in P2.
- **No raw-read egress:** only hashes/metadata are recorded; references are hashed
  locally. No network.

## 6. Edge cases & error handling

- **iGenomes key:** record key, checksums `None`; never touch the filesystem.
- **Missing/unreadable explicit path:** checksum `None` (guard `OSError`); never a
  crash, never a fabricated/zero hash, never a false pass.
- **Snakemake / no reference:** no reference keys in params → `reference_identity =
  None`; renderers omit the section cleanly.
- **Partial reference** (only one of fasta/gtf present — shouldn't happen because
  `resolve_reference` enforces both, but defensively): hash whichever exists, other
  `None`.
- **No verdict impact:** this slice adds no `QCResult` and must not change
  `overall_verdict` or any exit code — guard by not touching `qc_results`.
