## Summary

Add two WARN-capped RNA-seq biological-plausibility checks ‚Äî `duplication_rate`
(`percent_duplication`) and `rrna_contamination` (`percent_rRNA`) ‚Äî in a new
`RNASEQ_PLAUSIBILITY_PACK`, evaluated by a new `evaluate_rnaseq_plausibility(metrics)`
that emits **`unverified` (never PASS/FAIL)** for any plausibility metric absent from a
sample's ingested MultiQC, wired into `runner._discover_qc` under `assay == "rnaseq"`.
Mirrors the germline C3 pattern (`variant_metrics.evaluate_variant_plausibility`).
Tests-only (no detector-corpus seeding ‚Äî plausibility is not a `FailureClass`).

This is a **pure addition on the verify path**: no schema change, no new QC kind
(`kind="metric"` already renders), no metric-compute path (all values from ingested
MultiQC), no Layer-1 surface, no raw-read egress.

## Project setup

Nothing new. Existing deps suffice (`uv sync` already done in the worktree). Files
touched live under `src/contig/verification/`, `src/contig/runner.py`, and `tests/`.

## Phase 2 ‚Äî `evaluate_rnaseq_plausibility` (honest UNVERIFIED-when-absent)

- **Goal:** Turn a parsed per-sample MultiQC metrics dict into plausibility
  `QCResult`s, emitting explicit `unverified` for any plausibility metric a sample
  lacks ‚Äî mirroring `evaluate_variant_plausibility` (`variant_metrics.py:137-179`).
- **Prerequisites:** Phase 1.
- **Files:**
  - `src/contig/verification/rnaseq_plausibility.py` (new).
  - `tests/verification/test_rnaseq_plausibility.py` (new) ‚Äî RED first.
- **Design (the contract):**
  ```python
  def evaluate_rnaseq_plausibility(
      metrics: dict[str, dict[str, float]]
  ) -> list[QCResult]:
      ...
  ```
  Takes the already-parsed `{sample: {metric: value}}` (same shape
  `qc_ingest.parse_multiqc_general_stats_file` returns and `evaluate()` consumes).
  Taking a dict (not a path) keeps it pure and trivially testable with synthetic
  inputs ‚Äî the path parse happens once at the call site (Phase 3).
- **Steps (TDD):**
  1. **RED** ‚Äî tests:
     - in-band `percent_duplication` (e.g. 30.0) ‚Üí one `duplication_rate:<sample>`
       result, `status="pass"`, `kind="metric"`, `value==30.0`.
     - out-of-band (e.g. 95.0) ‚Üí `status="warn"`; **assert it is never `fail`**.
     - same two cases for `percent_rRNA` / `rrna_contamination`.
     - a sample dict **missing** `percent_duplication` ‚Üí a `duplication_rate:<sample>`
       result with `status="unverified"`, `value is None`, `kind="metric"`; likewise
       for a missing `percent_rRNA`.
     - a sample with **both** metrics absent ‚Üí two `unverified` results, no PASS.
     - **multi-sample**: two samples, one in-band and one missing the metric ‚Üí the
       right per-sample mix (asserts iteration over all samples).
     - empty metrics `{}` ‚Üí empty list (no crash).
  2. **GREEN** ‚Äî implement, mirroring germline:
     ```python
     from contig.models import QCResult
     from contig.verification.rule_pack import RNASEQ_PLAUSIBILITY_PACK, evaluate

     _CHECKS = ("duplication_rate", "rrna_contamination")

     def _rule_by_check(name: str) -> dict:
         for rule in RNASEQ_PLAUSIBILITY_PACK:
             if rule["check"] == name:
                 return rule
         raise KeyError(name)

     def evaluate_rnaseq_plausibility(metrics):
         rules = [_rule_by_check(n) for n in _CHECKS]
         results: list[QCResult] = []
         for sample, sample_metrics in metrics.items():
             computable = {
                 r["metric"]: sample_metrics[r["metric"]]
                 for r in rules if r["metric"] in sample_metrics
             }
             results.extend(evaluate({sample: computable}, rules))
             for r in rules:
                 if r["metric"] not in sample_metrics:
                     results.append(QCResult(
                         check=f"{r['check']}:{sample}",
                         status="unverified",
                         message=f"{sample}: {r['metric']} not reported by MultiQC",
                         value=None,
                         kind="metric",
                     ))
         return results
     ```
     Note `evaluate()` already names results `"<check>:<sample>"` and sets
     `kind` default ‚Äî confirm the unverified branch sets `kind="metric"` explicitly
     to match (germline does).
  3. **REFACTOR** ‚Äî if the `_rule_by_check` helper duplicates germline's, leave both
     (small, local); do not over-abstract across modules this slice.
- **Validation:** `uv run pytest tests/verification/test_rnaseq_plausibility.py -q`.
- **Commit:** `feat(verify): evaluate_rnaseq_plausibility with honest UNVERIFIED-when-absent`

## Edge cases & error handling

- **Metric absent** ‚Üí explicit `unverified` (the core guarantee), never PASS/FAIL.
- **No MultiQC at all** ‚Üí no plausibility checks (the metric source is MultiQC; we
  cannot know sample names). Honest: nothing claimed. Gated by `multiqc is not None`.
- **Empty metrics dict** ‚Üí empty list, no crash (Phase 2 test).
- **Value far above band** ‚Üí `warn`, never `fail` (no `fail_*` in the pack).
- **Wrong/missing slug (R1)** ‚Üí behaves exactly as "metric absent" ‚Üí `unverified`.

## Reproducibility / determinism

Pure function of ingested metrics; deterministic, no tool exec, no network, no
randomness. No run-record schema change. No raw-read egress.
