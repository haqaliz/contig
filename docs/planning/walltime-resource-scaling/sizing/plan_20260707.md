## Phase 1 — Pure walltime sizer  *(parallelizable with Phase 2)*

**Goal:** `realtime_informed_time_h` returning a raw observed-based hour target, mirroring
`peak_informed_memory_gb`.

**Files:** `src/contig/resource_sizing.py`, `tests/test_resource_sizing.py`.

**RED — write these tests first** (mirror the memory cases in `test_resource_sizing.py`):
1. tail: one usage row `realtime_sec=36000` (10h) → `target_h == ceil(10*1.5)=15`,
   `tier=="realtime"`, `observed_realtime_sec==36000`.
2. censored/common: `realtime_sec=14400` (4h) → raw `target_h==6` (floor-to-blind is
   Phase 2's job, NOT the sizer's — the sizer returns the raw 6).
3. multi-row: takes the **max** realtime across rows.
4. unavailable: all rows `realtime_sec==0` (dash) → `(None,"unavailable",None)`.
5. empty inputs → `(None,"unavailable",None)`.
6. `ceil` boundary (e.g. `realtime_sec` s.t. `/3600*factor` is just over an int).
7. default factor constant `WALLTIME_SAFETY_FACTOR == 1.5`.

**GREEN — implement:**
```python
WALLTIME_SAFETY_FACTOR = 1.5

class TimeSizing(NamedTuple):
    target_h: int | None
    tier: str
    observed_realtime_sec: float | None

def realtime_informed_time_h(events, usage, *, factor=WALLTIME_SAFETY_FACTOR) -> TimeSizing:
    observed = [r.realtime_sec for r in usage if r.realtime_sec > 0]
    if not observed:
        return TimeSizing(None, "unavailable", None)
    longest = max(observed)
    return TimeSizing(math.ceil(longest / 3600 * factor), "realtime", longest)
```
- `events` kept in the signature for symmetry with the memory sizer and so the wiring layer
  can later flag failed-row coincidence (R3); the pure sizer keys on `usage` only, because
  `time_limit` has no `exit==137` join key. Add a one-line docstring note on the censoring
  (this is a lower bound; floor-at-blind lives in `apply_patch`).

**Validation:** `uv run pytest tests/test_resource_sizing.py -q` green.
**Commit:** `feat(self-heal): pure walltime sizer realtime_informed_time_h (C2)`

## Phase 2 — `apply_patch` observed_target_h with floor-at-blind  *(parallelizable with Phase 1)*

**Goal:** the time branch accepts an observed override, floored at blind `× 2`.

**Files:** `src/contig/self_heal.py`, `tests/test_self_heal.py`.

**RED — unit tests (mirror the `observed_target_gb` block `test_self_heal.py:630-738`):**
1. `observed_target_h=15`, current `time="4.h"`, mult 2 → applied `"15.h"` (beats blind 8).
2. **floor:** `observed_target_h=6`, current `4.h` → applied `"8.h"` (blind floor wins).
   *(This is the walltime-specific behavior the memory branch does NOT have.)*
3. `observed_target_h=None`, current `4.h` → applied `"8.h"` (blind unchanged).
4. ceiling: `observed_target_h=200`, current `40.h`, ceiling 72 → applied `"72.h"`.
5. never-shrink: current `50.h`, `observed_target_h=10` → `max(max(10,100),50)`… →
   blind 100 clamped to 72 → `"72.h"` (never below current; pick numbers that make the
   intent unambiguous — verify against `_resource_ceiling_block` which would have blocked a
   current already ≥72).
6. memory branch unaffected by `observed_target_h` (invert
   `test_apply_patch_observed_target_leaves_time_branch_unaffected`, :719-738).
7. **both overrides:** `observed_target_gb=90` AND `observed_target_h=15` with a patch
   multiplying both → memory `"90.GB"` and time `"15.h"` each honored (R5 invariant).

**GREEN — implement** (time branch at `self_heal.py:490-495`):
```python
def apply_patch(..., observed_target_gb=None, observed_target_h=None):
    ...
    if "time" in mult:
        current = _lead_number(limits.get("time"), _DEFAULT_TIME_HOURS)
        blind = int(current * int(mult["time"]))
        bumped = max(observed_target_h, blind) if observed_target_h is not None else blind
        capped = min(bumped, ceiling["time"])
        final = max(capped, int(current))
        limits["time"] = f"{final}.h"
```
- Add `observed_target_h: int | None = None` after `observed_target_gb` (keyword-only).
- Update the docstring: note the time override is **floored at the blind bump** (unlike
  memory), because `realtime` is a censored lower bound.

**Validation:** `uv run pytest tests/test_self_heal.py -q` green (incl. untouched memory tests).
**Commit:** `feat(self-heal): apply_patch observed_target_h, floored at blind x2 (C2)`

## Edge cases & error handling

- `realtime` dash/blank → `0.0` → excluded by `>0` (unknown, never "0h").
- Sized value above ceiling → clamped to 72h (but ceiling-block already gave up if *current*
  ≥72, so this only bites when the sized jump overshoots from below).
- Both-overrides-present can't happen (one failure class/attempt) but is asserted/tested (R5).
- snakemake / trace-less → `unavailable` → blind fallback (no regression).
