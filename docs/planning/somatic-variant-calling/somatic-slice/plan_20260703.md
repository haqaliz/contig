## Phase 1 — Persist assay on the run + backward-compat (M2, M9) 🔴 foundational

- **Goal:** Fix the `nf-core/sarek` pipeline-string collision at its root by making the
  run's `assay` an explicit, persisted input that **wins over** the legacy
  `assay_for_pipeline(...) or "rnaseq"` derivation, with the legacy path preserved as the
  fallback so all existing runs and the ~12 shipped bundles are unaffected.
- **Prerequisites:** green baseline.
- **Files:** `src/contig/cli.py`, `src/contig/models.py` (`LaunchManifest`; optionally
  `RunRecord`), `tests/` (new `test_run_assay_override.py` or extend an existing run test).
- **RED — write these tests first:**
  1. `_dispatch_run`/`run` with an explicit `assay="somatic_variant_calling"` passes that
     assay to `self_heal_run` (assert via a spy/monkeypatch on `self_heal_run`, or assert
     it lands in the RunRecord/manifest), **not** the pipeline-derived one.
  2. **Germline-unchanged regression:** `run --pipeline nf-core/sarek` with **no**
     `--assay` still resolves `variant_calling` (legacy `assay_for_pipeline`), and
     `--pipeline nf-core/rnaseq` still resolves `rnaseq`.
  3. **Backward-compat:** a legacy `LaunchManifest` / RunRecord with no `assay` field
     loads and resolves via the fallback (no error).
  4. `rerun` replays a manifest that carries `assay` and re-applies it.
- **GREEN — implementation steps:**
  1. Add `--assay` option to `run` (`cli.py:203-230`), `str | None = None`, help: "Assay
     key override; needed when two assays share a pipeline (e.g. somatic vs germline
     sarek). Defaults to the pipeline's registered assay." Thread it into `_dispatch_run`.
  2. Add `assay: str | None = None` param to `_dispatch_run` (`cli.py:267-295`).
  3. At `cli.py:505` change to:
     `assay=assay or assay_for_pipeline(effective_pipeline) or "rnaseq"` (explicit wins;
     legacy fallback intact = M9).
  4. Add `assay: str | None = None` to `LaunchManifest` (`models.py`), write it in the
     manifest block (`cli.py:470-490`), and have `rerun` pass it back into `_dispatch_run`.
  5. Add `assay: str | None = None` to `RunRecord` (`models.py:255-276`) and set it from
     the resolved assay where the record is assembled, so `methods.py`/benchmark can read
     `record.assay` instead of re-deriving (used in P6). Keep it optional (backward-compat).
- **Validation:** `uv run pytest tests/ -k "assay or rerun or run_"` green; full suite green.
- **Commit:** `feat(run): carry an explicit --assay that overrides pipeline-derived assay (M2/M9)`

## Phase 2 — Registry entry + somatic routing (M1)

- **Goal:** A somatic goal routes to `somatic_variant_calling` via `plan`/`build_plan`
  and the dashboard; germline routing is unchanged.
- **Prerequisites:** P1.
- **Files:** `src/contig/registry.py`, `tests/test_registry.py` (or the registry
  round-trip test), datashape test.
- **RED:**
  1. `match_assay("somatic tumor/normal variant calling") == "somatic_variant_calling"`.
  2. `match_assay("somatic variant calling") == "somatic_variant_calling"` and
     `match_assay("tumour normal") == "somatic_variant_calling"`.
  3. **Non-collision:** germline goals ("germline variant calling", "call variants",
     "snv") still return `variant_calling`; the somatic goals do **not** return
     `variant_calling`.
  4. `select_pipeline("somatic_variant_calling").pipeline == "nf-core/sarek"` and
     `.revision == "3.5.1"`.
  5. datashape: somatic is **not** in `_REPLICATE_ASSAYS` (a single pair raises no
     replicate warning).
- **GREEN:**
  1. Add a `PipelineEntry(assay="somatic_variant_calling", pipeline="nf-core/sarek",
     revision="3.5.1", description="Somatic tumor–normal short-variant calling
     (nf-core/sarek), research use.")` to `REGISTRY` (`registry.py:12`).
  2. Add a `"somatic_variant_calling"` group to `_ASSAY_KEYWORDS` **before**
     `variant_calling` in iteration order (first-hit-wins, `registry.py:158-160`), needles:
     `"somatic"`, `"tumor"`, `"tumour"`, `"tumor-normal"`, `"tumor normal"`,
     `"tumour-normal"`. Rationale: germline needles don't contain these, and putting
     somatic first means a "somatic variant calling" goal can't fall to the generic
     `variant calling` needle. Add the ordering comment.
  3. Do **not** touch `_REPLICATE_ASSAYS`.
- **Note on `_ASSAY_BY_PIPELINE`:** it now maps `nf-core/sarek` → whichever sarek entry
  is last. That's acceptable **because P1 made the run path stop depending on it**;
  `assay_for_pipeline` remains only the legacy fallback for germline (no `--assay`). Add a
  test pinning that `assay_for_pipeline("nf-core/sarek")` is deterministic (document which
  entry wins) so the fallback behavior is explicit.
- **Validation:** `uv run pytest tests/test_registry.py` green; full suite green.
- **Commit:** `feat(registry): route somatic tumor–normal goals to a sarek somatic assay (M1)`

## Phase 3 — Sarek tumor/normal sample sheet + pre-flight (M3)  ⟂ parallel

- **Goal:** A somatic run's sample sheet is validated for the sarek tumor/normal paired
  shape at the `_dispatch_run` chokepoint, somatic-gated; germline/RNA-seq validation is
  untouched.
- **Prerequisites:** P1 (needs the resolved `assay` in `_dispatch_run` to gate),
  P2 (assay key exists).
- **Files:** `src/contig/samplesheet.py` (new sarek row model + validator),
  `src/contig/cli.py` (gate at `:382-388`), `tests/test_samplesheet.py`.
- **RED:**
  1. A valid sheet (patient P1: sample N status 0 + sample T status 1, same patient) →
     no issues.
  2. Missing `status` column → refuse with the specific missing-column message.
  3. `status ∉ {0,1}` → refuse naming the offending row.
  4. **Unpaired tumor** (a `status:1` patient with no `status:0`) → refuse with a message
     pointing at germline.
  5. **Multi-tumor / relapse** (patient with N + two distinct T rows) → allowed.
  6. **Tumor-only** (no normal at all) → refuse.
  7. Gate is **somatic-gated**: a germline run (`assay="variant_calling"`) still uses the
     generic `validate_samplesheet` (assert germline sheet path unchanged).
- **GREEN:**
  1. Add a sarek-shaped parser/validator to `samplesheet.py` — a `SarekSampleRow`
     (`patient, sample, status:int, lane, fastq_1, fastq_2|None`) and
     `validate_somatic_samplesheet(path) -> list[str]` mirroring `validate_samplesheet`'s
     return contract (`samplesheet.py:40-67`), implementing rules 1–6. Group rows by
     `patient`; a pair = same patient, distinct sample, statuses {0,1}.
  2. In `_dispatch_run` sample-sheet block (`cli.py:382-388`), branch on the resolved
     assay: `if assay == "somatic_variant_calling": issues =
     validate_somatic_samplesheet(input)` else the existing `validate_samplesheet(input)`.
     Reuse the identical refuse/exit block.
  3. Leave `fastq_paths` working for provenance (extend if the sarek columns need it, but
     keep germline/RNA-seq behavior identical).
- **Validation:** `uv run pytest tests/test_samplesheet.py` green; full suite green.
- **Commit:** `feat(samplesheet): validate sarek tumor–normal pairing for somatic pre-flight (M3)`

  3. The injected `--tools` is captured in `params` so it flows into `launch.json`
     (reproduce carries it).
- **GREEN:**
  1. Add `default_params: dict[str, object] = {}` to `PipelineEntry` (`models.py:152-158`).
  2. Set `default_params={"tools": "strelka,mutect2"}` on the somatic registry entry

  3. In `_dispatch_run` param assembly (`cli.py:375-464`), before `--input`/`--outdir`,

## Testing strategy (summary)

- Unit tests land **first** in each phase (RED). Integration test is Phase 7.
- All fixtures synthetic; injected executor/builder pattern (no real
  nf-core/samtools/sarek). Determinism preserved (no network, no `Date.now`-style
  nondeterminism).
- Regression guard: the germline-unchanged assertions in P1/P2/P3 are the safety net for
  the collision fix's blast radius (R6).

## Environment & reproducibility

- `uv sync` already done in the worktree. Run everything via `uv run`.
- Reproducibility contract: the persisted `assay` (P1) must round-trip through
  `launch.json` (`rerun`/`resume`) and `--tools` (P4) rides in `params` → `launch.json`.
  P1's backward-compat test protects the ~12 shipped bundles.
- Sarek revision pinned to the real released `3.5.1` (same tag germline uses — no invented
  version).

## Edge cases & error handling

- Unpaired tumor / tumor-only / germline-sheet-to-somatic-goal → honest refusal at
  pre-flight (P3), never a silent germline run.
- Legacy records/manifests without `assay` → legacy `assay_for_pipeline` fallback (P1/M9).
- Unknown assay anywhere → the existing hard `ValueError` (`manifest_for`,
  `rule_pack_for`) is preserved; somatic is registered everywhere it must be.
