## Pinned decisions (resolve the PRD's three đźźˇ gaps â€” do NOT re-open during impl)

### D1 â€” Minimal SO-severity ordering for "most-severe consequence per variant"

Ship a small fixed ordering, leastâ†’most severe; `intergenic_variant` is the unique
**rank 0**. An unknown, non-empty term ranks **above** intergenic (so it counts as a *real*
consequence, never as intergenic â€” the conservative choice).

```python
# annotation_plausibility.py
_SEVERITY_ORDER = (            # index = severity rank; 0 = least severe
    "intergenic_variant",     # rank 0 â€” the ONLY term that marks a variant "intergenic"
    "downstream_gene_variant",
    "upstream_gene_variant",
    "intron_variant",
    "non_coding_transcript_exon_variant",
    "non_coding_transcript_variant",
    "3_prime_UTR_variant",
    "5_prime_UTR_variant",
    "synonymous_variant",
    "stop_retained_variant",
    "start_retained_variant",
    "splice_region_variant",
    "protein_altering_variant",
    "inframe_deletion",
    "inframe_insertion",
    "missense_variant",
    "start_lost",
    "stop_lost",
    "frameshift_variant",
    "stop_gained",
    "splice_donor_variant",
    "splice_acceptor_variant",
    "transcript_ablation",
)
_SEVERITY_RANK = {term: i for i, term in enumerate(_SEVERITY_ORDER)}
_UNKNOWN_RANK = 1             # unknown non-empty term > intergenic â‡’ treated as "real"
```

Per variant: collect every consequence term across all CSQ/ANN entries (comma-split) and
their `&`-joined sub-terms; the variant's **most-severe rank** =
`max(_SEVERITY_RANK.get(term, _UNKNOWN_RANK) for term in terms)`. A variant is:
- **intergenic** iff it has â‰Ą1 parseable term and its most-severe rank == 0,
- **real** iff most-severe rank â‰Ą 1,
- **empty** iff it carries the field but no parseable term.

### D2 â€” Illustrative, LOOSE band edges (WARN-capped, no FAIL; documented as uncalibrated)

Over the set of records that carry the annotation field (`annotated_records`):

| Metric | Check name | Band | Rationale |
|---|---|---|---|
| `real_consequence_fraction` = real / annotated | `annotation_real_fraction` | `warn_below: 0.10` | Only a *degenerate* run (â‰Ą90% empty or intergenic) trips it â€” won't cry wolf on a legit high-intron/UTR run |
| `intergenic_fraction` = intergenic / annotated | `annotation_consequence_distribution` | `warn_above: 0.95` | The PRD's "~100%-intergenic" smell; loose ceiling so a real high-intergenic WGS/off-target run passes |

These are **engineering defaults, not calibrated** â€” say so in the changelog exactly as the
prior plausibility slices did. The two checks overlap (real + intergenic + empty = 1) but are
not identical: `real_fraction` also catches an all-**empty** (parse-degenerate) annotation
that `intergenic_fraction` misses. Ship both â€” a floor + a specific-smell ceiling.

### D3 â€” Somatic annotated-VCF location (fixture mirrors sarek; detection is path-agnostic)

The gate uses `rglob("*.vcf.gz")` + "first VCF whose header declares CSQ/ANN," so the exact
path does not affect detection. Fixtures mirror sarek's somatic annotation output shape:
`results/annotation/<tumor>_vs_<normal>/<caller>/*_VEP.ann.vcf.gz`. If absent in a live run,
the verifier reports UNVERIFIED (honest) â€” tolerated, not a blocker.

## Edge Cases & Error Handling

- **CSQ `Format:` lacks `Consequence`** â†’ metrics `None` â†’ UNVERIFIED (never guess an index).
- **Field declared but no data records / no record carries it** â†’ `None` â†’ UNVERIFIED (mirror
  M1's `info_key is None` path).
- **Empty consequence subfield** â†’ counts as *empty*, lowers `real_fraction`, not intergenic.
- **Unknown consequence term** â†’ `_UNKNOWN_RANK` (1) â†’ treated as *real*, never intergenic
  (conservative; won't inflate the intergenic smell on novel terms).
- **Somatic multi-VCF** â†’ reuse M1's "first CSQ/ANN VCF wins"; if disambiguation is needed,
  prefer the `mutect2` path component (consistent with `runner.py:169-176`).
- **Legacy record with no persisted assay at `_finalize`** â†’ do not silently drop provenance
  for a real variant run; resolve assay with the `assay_for_pipeline` fallback and cover it.
