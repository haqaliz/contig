## Phase 1 — `IndexBuilder` seam + pure parse helpers

**Goal:** A new injectable build seam and the pure functions that turn a
`missing_index` diagnosis into a build command — all unit-testable without the loop.

**Prerequisites:** baseline green.

**Files:** `src/contig/runner.py`, `src/contig/self_heal.py`, `tests/test_runner.py`,
`tests/test_self_heal.py` (helper tests).

**Implementation steps:**

1. **(RED)** In `tests/test_self_heal.py`, add unit tests for a new pure helper
   `_parse_missing_fai(diagnosis) -> str | None`:
   - evidence line `"[E::fai_load] Failed to open the index reference.fasta.fai: No such file or directory"` → returns `"reference.fasta.fai"`.
   - an absolute path token (e.g. `/data/ref.fa.fai`) → returns it verbatim.
   - evidence with no `.fai` token → returns `None`.
   And a helper `_fai_build_command(fai_path) -> list[str]` returning
   `["samtools", "faidx", "<fasta>"]` where `<fasta>` is `fai_path` minus the
   trailing `.fai`.
2. **(RED)** In `tests/test_runner.py`, add a test that `default_index_builder`
   has the `Executor`-style contract: call it with a harmless command
   (e.g. `["true"]` or `[sys.executable, "-c", ""]`) and a `cwd` `tmp_path`; assert it
   returns `0`. (Mirrors how `default_executor` is exercised.)
3. **(GREEN)** In `runner.py`, beside `Executor`, add:
   ```python
   # An index builder runs an auxiliary build command (e.g. `samtools faidx ref`)
   # in the given cwd and returns its exit code. The default shells out; tests
   # inject a fake that creates the index file, so no real tool runs in CI.
   IndexBuilder = Callable[[list[str], Path], int]

   def default_index_builder(cmd: list[str], cwd: Path) -> int:
       proc = subprocess.run(cmd, cwd=cwd, check=False)
       return proc.returncode
   ```
4. **(GREEN)** In `self_heal.py`, add the two pure helpers (`_parse_missing_fai`,
   `_fai_build_command`). Use a regex that finds a whitespace-free token ending in
   `.fai` (handles absolute/relative). Strip exactly the trailing `.fai`.
5. **(REFACTOR)** Keep helpers small and pure; no I/O.

**Validation:**
`uv run pytest tests/test_self_heal.py -q -k "parse or build_command"` and
`uv run pytest tests/test_runner.py -q -k index_builder` pass.

**Commit:** `feat(self-heal): add IndexBuilder seam and missing-.fai parse helpers`

## Phase 2 — Wire the build into the loop (shared helper)

**Goal:** When a `build_index` patch is applied, build the `.fai` and retry; branch
the recorded outcome honestly. One helper used at every gated apply site.

**Prerequisites:** Phase 1.

**Files:** `src/contig/self_heal.py`, `tests/test_self_heal.py`.

**Implementation steps:**

1. **(RED)** Add the acceptance tests (spec AC1–AC6), each with an injected fake
   executor and fake index_builder. Mirror `_failing_then_capturing` and the
   `test_gives_up_at_memory_ceiling` patterns. Tests:
   - **AC1/AC5 heal:** executor fails attempt 1 with the `fai_load` log, succeeds
     attempt 2; fake builder writes the `.fai` into its cwd and returns 0;
     `auto_approve=True`. Assert run succeeded, `record.repair_history[-1].outcome ==
     "built_index_and_retried"`, patch op `{"build_index": True}`, executor called
     twice (re-run happened).
   - **AC2 argv:** fake builder records its `cmd`; assert
     `cmd == ["samtools", "faidx", "<fasta>"]` derived from the evidence path.
   - **AC3 failed build:** fake builder returns `1`; assert finalize with
     `outcome == "index_build_failed"`, `detail` contains the `.fai` path,
     `record.verdict == "fail"`, builder called once, executor **not** called a
     second time.
   - **AC4 unparseable:** craft a `missing_index` diagnosis whose evidence has no
     `.fai` token (e.g. monkeypatch `propose`/`diagnose` or feed a log that triggers
     `missing_index` via the word "index" but no path). Assert
     `outcome == "index_unresolvable"`, `verdict == "fail"`, builder never called.
   - **AC6:** assert `test_apply_patch_reference_build_index_is_rerun_only` still
     passes unchanged (run the existing test).
2. **(GREEN)** Add `index_builder: IndexBuilder = default_index_builder` to
   `self_heal_run`'s signature (`self_heal.py:333-355`) and import the seam.
3. **(GREEN)** Add the shared helper:
   ```python
   def _apply_patch_and_maybe_build(target, patch, params, *, diagnosis, run_dir,
                                    index_builder, ceiling, default_outcome):
       """apply_patch, then if it's a build_index reference patch, run the build.
       Returns (target, params, outcome, detail, continue_)."""
       target, params = apply_patch(target, patch, params, ceiling=ceiling)
       if not (patch.kind == "reference" and patch.operation.get("build_index")):
           return target, params, default_outcome, None, True
       fai = _parse_missing_fai(diagnosis)
       if fai is None:
           return target, params, "index_unresolvable", \
               "Could not parse a missing index path from the failure.", False
       rc = index_builder(_fai_build_command(fai), run_dir)
       if rc != 0:
           return target, params, "index_build_failed", \
               f"Building the index for {fai} failed (exit {rc}).", False
       return target, params, "built_index_and_retried", None, True
   ```
4. **(GREEN)** At each gated apply site, replace the inline
   `apply_patch(...)` + `_record_attempt(... outcome="approved_and_retried"/...)`
   with a call to the helper, then:
   ```python
   current_target, current_params, outcome, detail, cont = _apply_patch_and_maybe_build(
       current_target, <patch>, current_params, diagnosis=diagnosis, run_dir=run_dir,
       index_builder=index_builder, ceiling=resource_ceiling,
       default_outcome="approved_and_retried")  # or "chose_and_retried" for the ambiguous site
   _record_attempt(run_dir, repair_history,
       RepairStep(attempt=attempt, diagnosis=diagnosis, patch=<patch>, outcome=outcome, detail=detail))
   if not cont:
       return _finalize(exc.record, repair_history, run_dir,
           runs_dir=runs_dir, run_id=run_id, webhook=notify_webhook)
   _write_status(run_dir, "running")  # only where the original site wrote it
   attempt += 1
   continue
   ```
   Apply at the three gated sites: `auto_approve` (~443-451, default_outcome
   `approved_and_retried`), ambiguous-chosen (~470-480, `chose_and_retried`), and
   unambiguous-approve (~507-516, `approved_and_retried`). Preserve each site's
   surrounding `_write_status`/`_clear_pending_approval` calls.
5. **(GREEN)** Update the stale comment in `apply_patch` (`self_heal.py:293-295`) to
   note that a `build_index` reference patch is now handled by
   `_apply_patch_and_maybe_build` at the loop level (the build is the fix), while
   `apply_patch` itself remains a no-op for it.
6. **(REFACTOR)** De-dup the three call sites if cleanly possible; ensure no
   behavior change for non-`build_index` gated patches (existing gated tests stay
   green).

## Environment & reproducibility

- `uv sync` already done in the worktree. No env vars. The build seam is injectable,
  so CI is deterministic and tool-free; real runs shell out to `samtools` on the
  user's compute (no raw-read egress — only the index path is touched).

## Edge cases & error handling

- Unparseable index path → `index_unresolvable`, honest FAIL, builder never called.
- Build command non-zero exit → `index_build_failed` with `detail`, honest FAIL, no
  extra retry.
- Multiple `.fai` tokens in evidence → take the first; acceptable for this slice.
- Bounded by existing `max_attempts`; one build per applied patch, so no builder loop.
- Never promotes UNVERIFIED/UNRECOVERED to PASS (verdict reduction untouched).
