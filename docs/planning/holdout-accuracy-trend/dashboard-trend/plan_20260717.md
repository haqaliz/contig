## 4. Testing strategy

Playwright e2e over the swapped-in fixtures (≥2 points → deltas render); assert **structure
and the presence of a `pp` delta**, not exact percentages (matches the existing
`eval-history.spec.ts` philosophy so appended real points never break the test). `npm run
build` is the type/compile gate. No Python change in this aspect.

## 5. Environment & reproducibility

Env overrides `CONTIG_HOLDOUT_HISTORY` / `CONTIG_HEAL_HISTORY` mirror `CONTIG_EVAL_HISTORY`
(dashboard reads the committed engine data by default). The e2e fixture install backs up and
restores the real files so a developer's committed history is never clobbered by the suite.

## 6. Edge cases

- Absent/empty history file → empty-state card (each component's `history.length === 0`
  branch), never a thrown error. Assert at least the default-render path; an explicit
  absent-file spec is a should-have.
- Single-point history (the committed seed) → sparkline shows one dot, per-class/delta shows
  the "only one snapshot" note. Ensure the components handle `length === 1` (EvalHistory
  already does; mirror it in HealHistory).
- Malformed line in a history file → reader skips it (mirrors `getEvalHistory`).
