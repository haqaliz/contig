## Design decisions locked before coding (from the PRD, do not re-litigate)

- **`run_started_at: float` (epoch seconds)** is a new keyword param on `run_reproduction`,
  defaulting to `None`. The notebook observer **raises `ValueError`** (a programming error,
  not `UNVERIFIED`) if it is dispatched with `run_started_at is None` — the PRD's
  non-bypassable guard, made loud. Every existing caller/test passes no notebook claim, so
  the observer is never reached and `None` is harmless: **no existing test changes.**
- Run-start is captured **once, before the first run**, in the CLI via `time.time()`; not
  re-stamped on an `--allow-install` retry (PRD M6a).
- **Two new pure functions**, kept apart (PRD "two responsibilities"):
  - `resolve_notebook_cell_text(doc, cell) -> tuple[str | None, str]` — notebook JSON +
    cell address → extracted output text. All never-raise surface lives here.
  - the observer `_observe_notebook_located` composes `resolve_notebook_cell_text` →
    `resolve_match` (shipped, unchanged) → strip/float (shipped rule).
- `cell` address semantics: `int` indexes the full `cells` array; `{"contains": s}` selects
  the unique cell whose joined `source` contains `s`; 0-or-many → `(None, "...N cells...")`.
- Output text = concatenation, **in `outputs` order**, of: `stream` outputs with
  `name == "stdout"` (`text` field), and `execute_result`/`display_data` outputs'
  `data["text/plain"]`. `str` or `list[str]` both joined with `""`. Nothing else.

## Phase 1 — Pure extractor `resolve_notebook_cell_text`

**Goal:** notebook document (already-parsed JSON `object`) + `cell` address → `(text, "")`
or `(None, reason)`, never raising.

**Files:** `tests/test_reproduce_notebook_locator.py` (NEW, RED first);
`src/contig/verification/reproduce.py` (GREEN).

**Signature:**
```python
def resolve_notebook_cell_text(
    doc: object, cell: int | dict[str, str]
) -> tuple[str | None, str]:
```

**RED — write these tests first** (mirror `test_reproduce_pattern_locator.py`'s layout):
- Happy: int cell with one stdout stream → that text; with `text` as `list[str]` → joined.
- Happy: `execute_result` `text/plain` (str and list) → that text.
- Happy: cell with stdout stream **and** execute_result → concatenated in output order.
- Happy: `{"contains": "print(auc)"}` selects the matching cell.
- Excluded: a `stderr` stream and an `error` output are **not** in the extracted text.
- Ambiguity: `{"contains": s}` matching 0 cells → `(None, reason)` with `"0"` in reason;
  matching 2 cells → `(None, reason)` with `"2"` in reason. Never an arbitrary pick.
- Range: int cell out of range → `(None, reason)` naming the cell count.
- Empty output: a code cell with `outputs: []`, and a markdown cell with no `outputs` key →
  `(None, reason)` (distinct-but-cosmetic messages; PRD R6 — pick "cell N has no textual
  output").
- Never-raises fuzz: `doc` ∈ `[None, 42, [], {}, {"cells": 3}, {"cells": [7]},
  {"cells": [{"source": None, "outputs": "x"}]}, {"cells": [{"outputs": [{"output_type":
  "stream", "text": 5}]}]}]` × cell ∈ `[0, {"contains": "x"}, {"contains": ""}]` → every
  result is a 2-tuple `(None|str, str)`, no exception.
- `source` as `str` and as `list[str]` both work for `{"contains": …}`.

**GREEN — implementation notes:**
- Guard `isinstance(doc, dict)`; `cells = doc.get("cells")`; guard `isinstance(cells, list)`.
- Resolve the target cell index:
  - `int` (reject `bool`): bounds-check against `len(cells)`.
  - `dict`: it is `{"contains": s}` by construction (schema already validated), but
    defensively re-check; scan cells whose `_join(cell.get("source"))` contains `s`;
    enforce exactly one match, count named on 0/≥2.
- `target = cells[idx]`; guard `isinstance(target, dict)`.
- Extract: iterate `target.get("outputs")` if it is a list; per output dict, dispatch on
  `output_type`; collect stdout `text` and `text/plain` `data`; `_join` each (str →itself,
  list[str] → `"".join(str-items)`, else skip). Concatenate collected pieces in order.
- If no pieces collected → `(None, "cell … produced no textual output")`.
- A private `_join_source_or_text(v) -> str` helper shared by source-scan and output-extract.
- **Never raise:** every access is `.get` + `isinstance`; no indexing without a bounds check.

**Validation:** `uv run pytest tests/test_reproduce_notebook_locator.py -q` green.
**Commit:** `feat(reproduce): pure notebook cell-output extractor (C8 slice 5, phase 1)`

## Phase 2 — `NotebookLocator` + `load_claims` schema (four-way exclusion)

**Goal:** parse & validate a notebook claim pre-run; keep slice-1.5/3/4 byte-identical.

**Files:** `tests/test_reproduce.py` (RED, add a `load_claims` notebook section);
`src/contig/verification/reproduce.py` (GREEN).

**Dataclass** (after `PatternLocator`):
```python
@dataclass(frozen=True)
class NotebookLocator:
    source: str                     # the `from` .ipynb, repo-relative
    cell: int | dict[str, str]      # index, or {"contains": <source substring>}
    pattern: str                    # required; capture via resolve_match
```
Extend the `Claim.locator` union and the local `locator:` annotation in `load_claims` to
include `NotebookLocator`.

**RED — `load_claims` tests (both accept and reject):**
- Accept: `{"id","value","from":"x.ipynb","cell":7,"pattern":"AUC: ([0-9.]+)"}` →
  `NotebookLocator(source="x.ipynb", cell=7, pattern=…)`.
- Accept: `cell` as `{"contains":"print(auc)"}`.
- Reject (`ClaimsError`): `cell` without `from`; `cell` without `pattern`; `cell` with
  `path`; `cell` with any table field (`column`/`row`/`header`/`delimiter`); `cell` a
  negative int; `cell` a bool; `cell` a float; `cell` a dict without `contains`; `cell` a
  `{"contains": ""}`; `cell` a `{"contains": 5}`; `cell` a dict with an extra key.
- Reject: `pattern` that does not compile (already covered — assert it still fires with
  `cell` present).
- **Back-compat:** `{"from":"log.txt","pattern":"…"}` with **no** `cell` → still a
  `PatternLocator` (unchanged); `{"from":"x.json","path":"$.a"}` → `Locator`;
  `{"from":"t.tsv","column":0,"row":0,"header":false}` → `TableLocator`.

**GREEN — where the logic goes in `load_claims` (currently `:456-503`):**
- Add `has_cell = "cell" in item`.
- **Branch order matters.** Put the notebook branch **before** the existing `has_pattern`
  branch, because a notebook claim has both `cell` and `pattern`:
  ```
  if has_cell:
      # requires from + pattern; rejects path + all table fields
      ...
      locator = NotebookLocator(source=raw_from, cell=cell, pattern=raw_pattern)
  elif has_pattern:
      ...(unchanged slice-4)...
  elif has_from:
      ...(unchanged slice-1.5 / slice-3)...
  ```
- In the notebook branch: require `has_from` (else ClaimsError "cell requires from"); require
  `has_pattern` (else "cell requires pattern"); reject `has_path`; reject `has_table_field`.
  Validate `cell` int (non-neg, not bool) or single-key `{"contains": non-empty str}`.
  Reuse the shipped `pattern` non-empty + `re.compile` validation (extract to a tiny helper
  or duplicate the 6 lines — keep it obvious).
- The orphan guard at `:465` currently allows `pattern` without `from`. Notebook `cell`
  **must not** be orphan-legal: since `has_cell` requires `has_from` explicitly inside its
  branch, a bare `cell` with no `from` still errors — but confirm the `:465` precheck
  (`not has_from and not has_pattern and (has_path or has_table_field)`) does not
  accidentally pass a `cell`-only claim through to a wrong branch. Add `has_cell` to that
  precheck's file-requiring set so `cell` without `from`/`pattern` is caught early with a
  clear message.

**Validation:** `uv run pytest tests/test_reproduce.py -q` green; slice-1.5/3/4 tests
unchanged and passing.
**Commit:** `feat(reproduce): NotebookLocator schema + four-way claim exclusion (phase 2)`

## Phase 3 — Engine: observer, dispatch, freshness guard

**Goal:** `run_reproduction` binds a notebook claim, enforces the mtime guard, dispatches
correctly.

**Files:** `tests/test_reproduce.py` (RED, engine section);
`src/contig/verification/reproduce.py` (GREEN).

**Signature change:** add `run_started_at: float | None = None` as a keyword param to
`run_reproduction` (place it near `created_at`).

**Observer** `_observe_notebook_located(loc: NotebookLocator)` — a closure inside
`run_reproduction`, sibling of `_observe_pattern_located`:
1. `if run_started_at is None: raise ValueError("run_started_at is required to verify a "
   "notebook locator")` — loud, non-bypassable (PRD).
2. Containment: `resolved = (repo_path / loc.source).resolve()`; `relative_to(repo_root)` →
   escape → `(None, "... escapes the repo")`.
3. Size bound: `resolved.stat().st_size > _MAX_MATCH_BYTES` (via a `stat()` that also gives
   mtime — call `os.stat`/`Path.stat` once) → `(None, "... over the … limit")`. Missing file
   (`OSError` on stat) → `(None, "notebook … is missing or unreadable")`.
4. **Freshness guard:** `st_mtime < run_started_at` → `(None, "notebook {source!r} was not "
   "rewritten by this run (mtime predates run start)")`. This check is **before** parsing —
   a stale notebook is never even read for content.
5. Parse (cached in a new `_notebook_cache` keyed by resolved path): `json.loads(read_text)`;
   `(ValueError, OSError)` → cache `None` → `(None, "notebook … is not valid JSON")`.
6. `text, reason = resolve_notebook_cell_text(doc, loc.cell)`; `None` → `(None, "notebook
   cell in {source!r} did not resolve: {reason}")`.
7. `captured, reason = resolve_match(text, loc.pattern)`; `None` → `(None, "notebook pattern
   … did not resolve: {reason}")`.
8. `float(captured.strip())`; `ValueError`/non-finite → `(None, …)` (slice-4 wording).

**Dispatch (`:970-979`):** add a `NotebookLocator` branch to the explicit `isinstance`
chain, **before** the `PatternLocator` branch is fine (order among isinstance branches is
irrelevant since types are disjoint), and **before** the final `else` (which assumes
`Locator`):
```python
if isinstance(claim.locator, NotebookLocator):
    observed, fail_msg = _observe_notebook_located(claim.locator)
elif isinstance(claim.locator, TableLocator):
    ...
elif isinstance(claim.locator, PatternLocator):
    ...
else:
    observed, fail_msg = _observe_located(claim.locator)
```
Add `_notebook_cache: dict[str, object | None] = {}` alongside the other caches (`:694-696`).

**RED — engine tests** (build fixture notebooks on disk in `tmp_path`, set mtimes with
`os.utime`, use a scripted executor that returns `(0, "")`):
- Reproduced: fresh notebook (mtime ≥ run_started_at), stdout cell prints the number → status
  `reproduced`.
- Within-tolerance and diverged: same, values off by < / > tolerance.
- **Stale = UNVERIFIED even on exact match:** notebook mtime set to `run_started_at - 10`,
  content matches the claim exactly → `unverified`, message mentions "rewritten"/"run start".
  *(The single most important assertion in the slice.)*
- Missing notebook → unverified. Oversized notebook (> `_MAX_MATCH_BYTES`, via a real large
  file or by monkeypatching the constant low) → unverified naming the size.
- Non-JSON `.ipynb` (fresh) → unverified "not valid JSON".
- Cell/pattern unresolved (fresh, valid) → unverified with the extractor/pattern reason.
- Programming-error: calling `run_reproduction` with a notebook claim and
  `run_started_at=None` raises `ValueError` (guard is non-bypassable).
- `--allow-install` interaction (engine-level, scripted installer): first run fails, retry
  writes a fresh notebook; with `run_started_at` stamped before the first run, the notebook
  resolves (M6a). *(Can be a lighter test — the key is run_started_at is not re-stamped.)*

**Validation:** `uv run pytest tests/test_reproduce.py -q` green.
**Commit:** `feat(reproduce): notebook observer, dispatch, mtime freshness guard (phase 3)`

## Environment & reproducibility

No env vars, no external tools, no network, no notebook execution. Determinism comes from
on-disk fixture `.ipynb` files, `os.utime`-set mtimes, explicit injected `run_started_at`,
and scripted executors/installers. The freshness guard itself is a reproducibility feature:
it ties a bound value to the artifact *this run produced*.

## Edge cases & error handling (all → UNVERIFIED unless noted)

- Stale notebook (mtime < run-start) — the headline case. Committed-output false pass
  prevented.
- Missing / non-JSON / oversized notebook; cell out of range; `{"contains"}` 0-or-many;
  cell with no textual output; pattern 0-or-many; non-numeric/non-finite capture.
- Escaping/absolute `from` → refused pre-run at the CLI (exit 1, no record); engine
  `relative_to` is defense-in-depth.
- Malformed claim shape → `ClaimsError`, exit 1, nothing written.
- `run_started_at is None` with a notebook claim → **`ValueError`** (programming error, not a
  silent UNVERIFIED) — this is intended and tested.
