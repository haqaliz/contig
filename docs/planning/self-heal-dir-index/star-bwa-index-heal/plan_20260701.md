## 2. Implementation Phases

### Phase 0 — Determinations spike (no production code; findings note only)

- **Goal:** lock the two facts the redirect + BWA depth depend on, so later phases don't
  stall. Output is a short findings note appended to this plan (or `findings.md`), not code.
- **Steps:**
  1. Confirm the nf-core/rnaseq **STAR index param name** used to point a run at a
     pre-built index (expected `star_index`). Grep nf-core schema knowledge / existing
     param handling; if not determinable from the repo, record `star_index` as the
     assumption and have Phase 4 give up honestly if the param can't be set.
  2. Determine whether any **currently-supported** pipeline invokes **classic `bwa index`**
     with a redirectable param. rnaseq→STAR; sarek default→bwa-mem2 (out of scope);
     methyl-seq→bwa-meth (different tool). **Decision rule:** if none does, BWA is
     **detector+corpus-only** this slice — Phases 3-5 implement STAR fully and BWA's
     build/redirect are dropped (documented), while Phase 1 still adds the BWA detector
     signature + corpus case (cheap, future-proofs the detector).
  3. Confirm `params` carries `fasta`/`gtf` at heal time (already confirmed at `cli.py:386`)
     and pick the access path for the deriver (thread `params` into the build step).
- **Validation:** findings recorded; STAR param name + BWA decision stated. **Checkpoint:
  surface the BWA decision to the integrator before Phase 3.**
- **Commit:** `docs(plan): self-heal-dir-index Phase 0 determinations (STAR param, BWA scope)`

### Phase 1 — Detector signatures + corpus (RED first)

- **Goal:** STAR-missing, STAR-version-incompatible, and BWA-missing logs classify as
  `missing_index`; a wrong-reference control does not.
- **Files:** `tests/test_detect.py` (or the detector test file), `src/contig/detect.py`,
  `src/contig/data/detector_corpus.jsonl`, `tests/test_corpus.py`.
- **Steps (TDD):**
  1. RED: add `diagnose_failure` tests with the verified real log strings:
     - STAR missing: `EXITING because of FATAL ERROR: could not open genome file /idx/genomeParameters.txt`
     - STAR version: `EXITING because of FATAL ERROR: Genome version: 20201 is INCOMPATIBLE with running STAR version: 2.7.5a` … `please re-generate genome from scratch`
     - BWA missing: `[E::bwa_idx_load_from_disk] fail to locate the index files`
     - Negative control: a wrong-reference/contig-mismatch line must NOT become `missing_index`.
  2. GREEN: add narrow branches in `diagnose_failure` (alongside the `.dict` branch,
     `detect.py:180-198`). Gate each on a tool-specific token **plus** an
     absence/incompat phrase (STAR: `genomeParameters.txt`+absence, or `Genome version`+`INCOMPATIBLE`;
     BWA: `bwa_idx_load_from_disk`+`fail to locate the index`). Keep `missing_index`
     ordered before `missing_reference` so a STAR "genome file" line is not swallowed as a
     reference.
  3. Add one golden `FailureCase` per signature to `detector_corpus.jsonl`
     (`expected_class:"missing_index"`, realistic `events` process names: `STAR_ALIGN`,
     `BWA_MEM`).
- **Validation:** `uv run pytest tests/test_detect.py tests/test_corpus.py`;
  `uv run contig eval-detector` accuracy == 100%.
- **Commit:** `feat(detect): classify STAR/BWA missing & version-incompatible index as missing_index`

### Phase 2 — Parser + kind discriminator (RED first)

- **Goal:** `_parse_missing_index` recognizes STAR/BWA evidence and returns a **kind**
  (not just a file ext), normalizing a STAR inner-file token to its parent **directory** so
  the build-once guard key matches the build target.
- **Files:** `tests/test_self_heal.py`, `src/contig/self_heal.py`.
- **Steps (TDD):**
  1. RED: tests that `_parse_missing_index` returns, e.g., `(genomeDir, "star")` from a STAR

     {`.fai`,…,`.dict`,`star`,`bwa`}; add regex/branches for the STAR dir (derive the dir

- **Validation:** `uv run pytest tests/test_self_heal.py -k parse`.
- **Commit:** `refactor(self-heal): parse missing-index kind (incl. STAR dir / BWA) not just extension`

### Phase 3 — Scratch build seam for STAR (+ BWA if Phase 0 says live) (RED first)

- **Goal:** dispatch off the `kind`; resolve the source FASTA(+GTF) from `params`; build
  into a run-scoped scratch dir via the injected `IndexBuilder`; verify the output dir/
  sidecars are non-empty; honest give-up otherwise.
- **Files:** `tests/test_self_heal.py`, `src/contig/self_heal.py`.
- **Steps (TDD):**
  1. RED: with an **injected builder** (records argv, creates a fake non-empty scratch dir,
     returns 0), assert a STAR `missing_index` heal calls the builder with
     `["STAR","--runMode","genomeGenerate","--genomeDir",<scratch>,"--genomeFastaFiles",<fasta>,
     "--sjdbGTFfile",<gtf>]` (GTF arg present only when `params["gtf"]` set), with `<scratch>`
     under `run_dir/healed_index/star/`. Assert give-up `index_unresolvable` when
     `params["fasta"]` is absent; `index_build_failed` when the builder returns non-zero or
     the scratch dir is empty after build.
  2. GREEN: generalize `_INDEX_BUILD`/`_index_build_command` to dispatch on `kind`; add a
     STAR entry whose deriver reads `params["fasta"]`/`params["gtf"]` (thread `params` into

     `_apply_patch_and_maybe_build` where `params` is in scope) and whose argv builder
     `mkdir`s the scratch genomeDir. Add a post-build **non-empty dir** check
     (`is_dir()` + core files / non-empty) — new vs the file-only `.exists()`/`rc==0`

  3. Keep the build-once guard keyed on the normalized (dir) path.
- **Validation:** `uv run pytest tests/test_self_heal.py -k "build or index"`.

## 5. Environment & Reproducibility

- No env vars, no external services in CI (all injected). `uv sync` already done.
- **Reproducibility requirements:** scratch path NOT persisted in `launch.json` (M13);
  STAR version recorded (S1) so a reproduce can't silently re-hit the version-mismatch
  class; heal is deterministic given the same inputs.

## 6. Edge Cases & Error Handling

- Unresolvable reference (`params["fasta"]` absent) → `index_unresolvable` (honest FAIL).
- Build exit≠0 or empty scratch dir → `index_build_failed`.
- Redirect param name undeterminable → `index_unresolvable` (R4).
- Wrong-reference masquerading as a missing index → NOT classified `missing_index`
  (Phase 1 negative control); never triggers a build.
- Retry fails for a new reason after a good build → surfaced honestly, no rebuild (M14).
- STAR inner-file token vs parent dir → normalized in Phase 2 so the guard key == target.
