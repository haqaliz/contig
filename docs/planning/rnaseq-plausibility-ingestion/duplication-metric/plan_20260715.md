## Decisions taken in this plan

**D1 ‚Äî Do NOT rename `duplication_rate` ‚Üí `duplication_fraction`.** (PRD open question.)
- "Rate" already reads as a fraction; if anything **methylseq's** `duplication_rate = 12.34` is
  the anomalous usage, not rnaseq's `0.707`.
- The check name is **user-visible in every report**. This check currently emits
  `duplication_rate:<sample> unverified` on every rnaseq run, so a rename churns existing output
  for no verification gain.
- The two never co-occur (each is assay-gated; methylseq is in `_DEDICATED_METRIC_ASSAYS`,
  rnaseq is not), so there is no ambiguity *within* a run.
- **Mitigation instead of rename:** the unit goes in the rule's `message` (M8), because
  `expected_range` is `None` for a band-less rule and cannot carry it.

**D2 ‚Äî The guard is data-driven, not hard-coded.** Add `"unit": "fraction"` to the duplication
rule; `evaluate_rnaseq_plausibility` enforces `0.0 <= value <= 1.0` for any rule carrying it.
Keeps `rule_pack.py` as **data** (its stated premise, `rule_pack.py:3-5`) and keeps the honesty
branch in the wrapper that already owns it. Do **not** hard-code the metric name in the wrapper.

**D3 ‚Äî M4 lands first, alone.** `duplication_rate` becomes the **repo's first band-less rule**
(verified: an AST scan of every `*_PACK` found no existing rule lacking `warn_below`/`warn_above`,
so `">= None"` has never fired). Fixing the renderer before creating the rule that triggers it
keeps every phase green and isolates the shared-machinery change for review.

## Phase 1 ‚Äî M4: `_expected_range` supports band-less rules

- **Goal:** a rule with no `warn_*` bounds renders `expected_range = None`, never `">= None"`.
- **Prerequisites:** none.
- **Files:** `src/contig/verification/rule_pack.py`, `tests/verification/test_rule_pack.py`
- **Blast radius:** shared by all seven assays. **Verified zero impact on existing rules** ‚Äî every
  current rule declares at least one `warn_*` bound, so no existing `expected_range` string
  changes. Regression-test anyway.

**Steps (TDD):**
1. **RED** ‚Äî in `test_rule_pack.py`, add a test that `evaluate()` over a synthetic band-less rule
   (`{"check": "x", "metric": "m", "message": "..."}`) yields `status == "pass"` and
   `expected_range is None`. It must fail today with `">= None"`.
2. **RED** ‚Äî add a regression test pinning the two existing render shapes:
   `warn_below` only ‚Üí `">= 60.0"`; `warn_above` only ‚Üí `"<= 10.0"`; both ‚Üí `"[a, b]"`.
3. **GREEN** ‚Äî change `_expected_range` (`rule_pack.py:526-535`): return type ‚Üí `str | None`;
   final branch becomes `if warn_below is not None: return f">= {warn_below}"` then `return None`.
   `_status_for` already returns `"pass"` with no bounds (`rule_pack.py:504-523`) ‚Äî **no change**.
4. Confirm `QCResult.expected_range` is already `str | None` (`models.py:74`) ‚Äî **no model change**.

**Validation:** `uv run pytest tests/verification/test_rule_pack.py` green; full suite ‚â• baseline.

**no `conftest.py` exists** ‚Äî helpers are local per file and deliberately duplicated; docstrings

- **Unit (evaluator):** Phase 2 list above ‚Äî synthetic dicts, no files.

- **Integration (gate):** Phase 3 ‚Äî `_discover_qc` with a realistic `multiqc_data.json`.

**Manual verification:** none available ‚Äî no real rnaseq MultiQC report exists in-repo
(`demo/sample-run/...` is synthetic, `make_sample_run.py:59,105`). This is the accepted
observed-vs-reasoned limit (PRD R1/R2).

## Edge cases

| Case | Expected |
|---|---|
| `PERCENT_DUPLICATION: 1.0` | **valid** ‚Äî reported (a fully duplicated library). Guard is strict `> 1.0`. |
| `PERCENT_DUPLICATION: 0.0` | valid ‚Äî reported. |
| `PERCENT_DUPLICATION: 95.0` | **guard** ‚Üí `unverified` (contract violated). Never rescaled to `0.95`. |
| Negative value | guard ‚Üí `unverified`. |
| Key absent (`--with_umi` / `--skip_markduplicates` auto-skip MarkDuplicates) | exactly one `unverified` ‚Äî a **real production path**. |
| Non-numeric value | already dropped by `qc_ingest.py:19-22` ‚Üí key absent ‚Üí `unverified`. |
| No MultiQC report at all | checks **vanish** (`runner.py:412`) ‚Äî pre-existing bug, **out of scope**, documented. |
| Old key `percent_duplication` present | `unverified` ‚Äî regression lock. |
| Non-rnaseq assay | no rnaseq plausibility checks (strict gate, unchanged). |
