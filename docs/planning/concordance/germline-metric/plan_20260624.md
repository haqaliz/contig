## Conventions

- Site key: `(CHROM, POS, REF, ALT)`. Genotype: the `GT` subfield of the first
  sample column, normalized so `0/1` and `0|1` compare equal (split on `/` or `|`,
  sort alleles). Slice 1 does no further normalization (literal site key).
- Thresholds (documented, tunable engineering defaults, NOT clinical):
  `genotype_concordance` WARN below `0.90`; `site_overlap` WARN below `0.90`. No
  FAIL band in slice 1.
- Concordance `QCResult`s always carry `kind="concordance"`, a numeric `value`,
  an `expected_range`, and a message naming both call sets (basename is fine).

## Phase 1: QCKind + pure metric

**Goal:** the deterministic metric and its QCResult emission, no wiring.
**Prerequisites:** none.
**Files:** create `tests/verification/test_concordance.py`,
`src/contig/verification/concordance.py`; modify `src/contig/models.py`.

**RED — write these tests first:**
1. `test_genotype_concordance_identical_calls_is_1` — two inline VCFs with the
   same sites/genotypes -> rate `1.0`, overlap `1.0`.
2. `test_genotype_concordance_one_mismatch` — same sites, one differing GT ->
   rate `< 1.0` computed correctly over shared sites.
3. `test_site_overlap_partial` — call sets sharing some sites -> overlap is the
   shared fraction; concordance computed only over shared sites.
4. `test_no_shared_sites_is_unverified` — disjoint sites -> `genotype_concordance`
   check `status="unverified"` (no 0/0), `site_overlap` `status="warn"` value
   `0.0`.
5. `test_gzipped_vcf_parses` — a `.vcf.gz` fixture parses identically.
6. `test_concordance_results_tagged_kind` — emitted `QCResult`s have
   `kind="concordance"`.

**GREEN — implement:**
1. `models.py`: `QCKind = Literal["metric", "structural", "concordance"]` (keep
   `kind` default `"metric"`).
2. `concordance.py`:
   - `_concordance(check, status, message, value=None, expected_range=None)` ->
     `QCResult(..., kind="concordance")`.
   - `parse_vcf(path) -> dict[site_key, genotype]`: stream lines (gzip if
     `.gz`), skip `#` headers, parse CHROM/POS/REF/ALT and the first sample GT;
     tolerate a missing GT (store `None`).
   - `genotype_concordance(vcf_a, vcf_b) -> ConcordanceStats` (a small dataclass or
     dict: `shared`, `concordant`, `rate | None`, `overlap`). `rate` is `None`
     when `shared == 0`.
   - `concordance_results(vcf_a, vcf_b) -> list[QCResult]`: build the two checks
     from the stats, applying the thresholds and the empty-intersection rule.

**Validation:** `uv run pytest tests/verification/test_concordance.py -q`.
**Commit:** `feat(concordance): deterministic germline genotype-concordance metric`.

## Testing strategy

- Unit: the metric, parsing (plain + gzip), thresholds, empty intersection.
- Integration: `run_qc` includes concordance and the verdict reflects it at most
  as WARN; `contig verify --concordance-vcf` end to end on a run fixture.
- Determinism: all fixtures are tiny inline VCFs in `tmp_path`; no tool execution,
  no network (matches `test_structural.py` / `test_run_qc.py`).
- Full regression: `uv run pytest` stays green; dashboard `npm run build`.

## Edge cases & error handling

- No shared sites -> `genotype_concordance` `unverified`, `site_overlap` WARN 0.0.
- A primary VCF not found for the run -> `verify` prints a clear note and skips
  concordance (no crash, no false PASS).
- Non-germline assay -> concordance skipped with an explicit message.
- Malformed GT or missing sample column -> treat the site's genotype as unknown
  and exclude it from the concordance denominator (do not crash); count it toward
  overlap only if the site key is present.
- Multiallelic / unnormalized representations -> known limitation; literal site
  key in slice 1 (documented; do not silently "fix").

## Reproducibility

- The two compared call sets are identified (basename, and ideally a checksum) in
  the result message / JSON so the concordance verdict is auditable and a re-run
  reproduces it. No randomness anywhere in the metric.
